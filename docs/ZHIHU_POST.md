# Spring：给你的大模型加一个审计层，让"我们这个模型最好"变成可证明的事实

我做了一套开源框架。不是新模型——是审计层。

一句话说清楚：**以后任何大模型公司，再想说"我们的模型最好"，不用写 50 页 benchmark 了——跑一遍 Spring，拿 Cercis 分数出来，横向对比。**

---

## 为什么需要这个

现在所有大模型（GPT、Claude、Gemini、DeepSeek……）都有一个共同问题：**M=1。**

什么意思？一个模型自己说自己好。没有独立验证。没有多专家交叉审计。幻觉检测靠人工抽检。数据集质量靠"我们清洗过了"。

这叫什么？在数学上叫**不可审计的声称**。在你的框架里叫**M=1 UNDECLARED**。

Spring 改的就是这个。它把 M 从 1 变成大于 1——多专家独立训练、交叉验证、噪声自动剔除。训练跑完的那一刻，你的模型带着一个不可伪造的审计分数出来。

---

## 怎么用（三步）

```bash
git clone https://github.com/shuchaoxi/SCX
cd SCX
python -m scx.spring --data ./your_training_data --experts 5
```

第一步：数据喂进去。第二步：Spring 自动多专家训练 + Yajie 审计 + 噪声剔除。第三步：拿到 M_t 参数和 Cercis 分数。贴在模型卡上。完事。

---

## 你得到什么

1. **M_t 参数：** 不是你自己报的——是从训练数据哈希算出来的，改不了。证明你的模型确实被审计过。
2. **Cercis 分数：** Q（精度）+ N（覆盖度），一个数字。Transformer 的 0.85 和 MoE 的 0.82 可以直接排。
3. **审计日志：** 每一轮训练、哪些数据被标记为噪声、哪些专家一致通过——全程可追溯。

---

## 对搞大模型的人来说，这意味着什么

- **数据清洗：** 不需要单独洗数据。Spring 每一轮迭代 = 训练 + 审计 + 剔除噪声。数据越训越干净。
- **蒸馏：** Teacher 跑 Yajie 共识评分 → CLEAN 数据 → 训 Student。Student 不需要 M>1——训练数据已经被审计过了。
- **横向对比：** 不需要再看各家的 benchmark 表格了。跑 Cercis。一个数字定胜负。
- **开源：** 全部代码在 GitHub，MIT 协议。定理证明在 arXiv/SCX 论文里。
