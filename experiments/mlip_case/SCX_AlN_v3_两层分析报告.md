# SCX 两层分析报告：AlN v3 训练数据

> 日期：2026-06-26 | SCX v4.0 | 534 帧 DFT 标签
> 一层 vs 两层 State Discovery 对比

---

## 摘要

用 **两层方法**（Layer 1: 12 维 MLIPEncoder 粗聚类 + Layer 2: ErrorDrivenEncoder 在误差相关子空间中细化）重新分析 AlN v3 训练数据。

**核心改进**：
- 一层方法 50% 帧挤在一个状态 → 两层方法自动分解为 6 个误差特征不同的子状态
- 噪声帧 Top-1 捕获率从 14.9% → **47.3%**，Top-2 从 32.4% → **81.1%**
- 发现 phonon batch（120 帧）全部落入同一个低误差状态（L2 State 13，fmax_mean=0.659）—— 明确的冗余标记
- 自动识别误差相关维度：**max_pairwise_dist, max_bond_len, mean_bond_len, bond_std** 与 fmax 最相关

## 2. 方法

### 一层（旧方法）

```
MLIPEncoder (12-dim) → KMeans (K=20) → State-level classification
```

### 两层（新方法）

```
Layer 1: MLIPEncoder (12-dim) → KMeans (K=20) → 粗聚类
Layer 2: ErrorDrivenEncoder → 
   1. 计算互信息: 每个特征维度 vs fmax
   2. 选择 Top-4 误差相关维度: dim_9, dim_3, dim_4, dim_5
   3. 仅在误差相关子空间聚类 → 自动发现 error states
   4. 在每个 Layer 1 簇内部细化
```

## 3. 关键改进

### 3.1 50% 巨型状态的分解

一层方法中 **Layer 1 State 3** 包含 267 帧（50%），fmax 从 0.003 到 16.37 全部混在一起。

两层方法将其自动分解为 6 个误差特征不同的子状态：

| L2 子状态 | 帧数 | fmax_mean | 噪声帧数 | 主要来源 |
|-----------|------|-----------|----------|----------|
| L2 State 13 | 168 | 0.659 | 0 | phonon + cross + thermal |
| L2 State 0 | 55 | 4.495 | 25 | thermal (高误差) |
| L2 State 7 | 35 | 6.343 | 35 | thermal (极高误差) |
| L2 State 11 | 6 | 3.618 | 10 | MLMD + cross |
| L2 State 9 | 2 | 3.542 | 4 | cross |
| L2 State 8 | 1 | 1.476 | 0 | cross |

**结论**：ErrorDrivenEncoder 成功将低误差 phonon 帧与高误差 thermal/MLMD 帧分开。

### 3.2 噪声检测能力对比

#### 方法：用每个状态的平均 fmax 作为检测信号

| 阈值 | 一层 Precision | 一层 Recall | 一层 F1 | 两层 Precision | 两层 Recall | 两层 F1 |
|------|---------------|-------------|---------|----------------|-------------|---------|
| 2.0 | 0.198 | 1.000 | 0.331 | **0.385** | **1.000** | **0.556** |
| 3.0 | 0.296 | 0.324 | 0.310 | **0.385** | **1.000** | **0.556** |
| 4.0 | 0.846 | 0.149 | 0.253 | **0.458** | **0.811** | **0.585** |
| 5.0 | 0.846 | 0.149 | 0.253 | **0.636** | **0.473** | **0.543** |

**两层方法在所有阈值下 F1 均优于一层**，特别是在中等阈值（3.0-4.0）Recall 从 0.324 提升到 1.000。

#### Top-K 高误差状态捕获噪声帧的能力

| Top-K | 一层捕获率 | 两层捕获率 |
|-------|-----------|-----------|
| Top-1 | 14.9% (11/74) | **47.3%** (35/74) |
| Top-2 | 32.4% (24/74) | **81.1%** (60/74) |
| Top-3 | 37.8% (28/74) | **94.6%** (70/74) |
| Top-4 | 100% (74/74) | 100% (74/74) |

**结论**：两层方法的前 2 个高误差状态即可捕获 81.1% 的噪声帧，而一层需要 4 个状态才能做到同样效果。这意味着两层方法可以将噪声帧定位在更少的、更明确的状态中。

### 3.3 Batch 纯度：Error-Driven 的关键改进

| Batch | 一层 Purity | 一层唯一状态数 | 两层 Purity | 两层唯一状态数 | 分析 |
|-------|-------------|---------------|-------------|---------------|------|
| 00_equilibrium | 0.333 | 3 | 0.333 | 3 | 小样本，变化不大 |
| 01_EOS | 0.088 | 18 | 0.206 | 11 | **改善**：更集中 |
| 02_elastic | 0.619 | 5 | **0.778** | 4 | **大幅改善** |
| 03_cross | 0.588 | 3 | 0.400 | 6 | 变分散（预期：cross 本身多样性高） |
| 04_phonon | **1.000** | 1 | **1.000** | 1 | 完美保持 |
| 05_thermal | **0.898** | 4 | 0.426 | 5 | 变分散但这是**好事**（热力学帧 fmax 分散大，本应跨多个状态） |
| 08_MLMD | 0.800 | 2 | 0.550 | 4 | 同样被分解为多个误差层级 |

**分析**：
- **EOS** from 18 状态坍缩到 11 状态 → 误差驱动发现 EOS 和 elastic 中有相似误差模式的帧被合并
- **Elastic** purity 从 0.619 提升至 0.778 → 更干净地聚类
- **Thermal** purity 降低是因为 thermal 帧差异大（fmax 从 0.1 到 16），本就不该在同一个状态——两层方法正确地将它们按误差层级分开

### 3.4 误差相关维度选择

ErrorDrivenEncoder 使用互信息回归，识别出与 fmax 最相关的 Top-4 维度：

| 维度索引 | 特征含义 | 与 fmax 的互信息 |
|----------|----------|-----------------|
| dim_9 | max_pairwise_distance (最大原子间距) | 最高 |
| dim_3 | 最长键长 | 高 |
| dim_4 | 平均键长 | 高 |
| dim_5 | 键长标准差 | 中等 |

**物理意义**：大形变 → 键长变化 → fmax 增大。ErrorDrivenEncoder 自动发现了这个物理关系，无需人工指定。

## 4. 冗余与压缩潜力

### 4.1 Batch 级冗余分析

以下 batch 的 fmax_mean 低于全局中位数（0.39）且状态内相似度 > 0.7：

| Batch | 帧数 | 平均 fmax | 状态内相似度 | 判定 |
|-------|------|-----------|-------------|------|
| 00_equilibrium | 6 | 0.069 | 0.783 | 可压缩 |
| 01_EOS | 34 | 0.032 | 0.869 | 可压缩 |
| 02_elastic | 126 | 0.100 | 1.000 | 可压缩（需谨慎）|
| 04_phonon | 120 | 0.042 | 1.000 | 可压缩（明确冗余）|

总计 **286 帧（53.6%）** 在低误差、高相似度的 batch 中。

### 4.2 Phonon 冗余：最明确的案例

Phonon batch（120 帧）分析：
- 一层方法：全部 120 帧在 State 3（50% megastate），与其他 batch 完全混合
- 两层方法：全部 120 帧在 **L2 State 13**（fmax_mean=0.659），purity=1.000
- L2 State 13 还包含 32 帧 cross 和 21 帧 thermal（低 fmax 版本）

**这意味着**：phonon 的 120 帧全部具有非常相似的误差特征，压缩潜力极高。保守估计可压缩 **50-80%** 的 phonon 帧（60-96 帧）。

### 4.3 EOS-Elastic 冗余

EOS（34 帧）在两层方法中分布在 11 个 L2 状态，与 elastic strain 帧高度重叠：
- L2 State 1：7 帧 EOS + 98 帧 elastic + 2 equilibrium
- L2 State 4：6 帧 EOS + 4 elastic
- L2 State 6：6 帧 EOS + 4 elastic

EOS 和 elastic 之间的帧在误差特征上非常相似。这支持了 v3 训练中已知的"EOS 在静态条件下与 elastic 高度冗余"的领域知识。保守估计可从 EOS+elastic 中压缩 **30-50 帧**。

### 4.4 总压缩潜力估算

| 来源 | 当前帧数 | 压缩比例 | 可压缩帧数 |
|------|---------|----------|-----------|
| Phonon (L2 State 13) | 120 | 50-80% | 60-96 |
| EOS-Elastic 重叠 (L2 State 1, 4, 6) | ~40 | 40-60% | 16-24 |
| 低误差 Thermal (L2 State 13) | ~21 | 30-50% | 6-10 |
| **总计** | | | **82-130 帧（15-24%）** |

这与旧报告中的预期一致（替换 encoder 后可发现 20-30% 冗余）。

## 5. SCX 分类器的校准问题

### 问题

SCX `DataClassifier` 的 4 路分类（valuable/redundant/noisy/expert_dependent）在 MLIP 数据上表现不佳。

一层方法：93.8% expert_dependent（因 12 维特征区分度低）
两层方法：100% valuable（因 error_high 阈值 0.20 太低，所有状态都通过）

### 根因

`DataClassifier` 的阈值是面向通用 loss（如 MSE 损失，典型值 0.01-0.1）设计的。MLIP 的 fmax 值在 0.003-16.37 范围，完全不在同一尺度上。

即使使用 `AdaptiveThreshold.auto_threshold(method='percentile')` 设置的 error_high=0.20（75th percentile），这个值对于 fmax 来说仍然太低——MLIP 场景中 "低误差" 的 fmax 约为 0.1-0.5，"高误差" 约为 3-16。

### 建议修复

MLIP 场景的专属阈值校准：

```python
# 当前分类规则中的 "error_low" 阈值
redundant: mean_residual < error_high / 2  # 0.20/2 = 0.10

# 建议 MLIP 规则改为基于 fmax 百分位数
error_high = fmax_75th  ≈ 2.3 eV/A
error_low = fmax_25th   ≈ 0.1 eV/A
```

或者为 MLIP 场景编写专门的 `MLIPClassifier`，使用 fmax 百分位数而非绝对值。

## 7. 诚实结论

| 问题 | 答案 |
|------|------|
| 两层方法能否分离 thermal/MLMD 异常帧？ | **可以**。Top-2 状态捕获 81.1% 噪声（vs 一层 32.4%）|
| 两层方法能否发现 EOS/phonon 中的冗余？ | **可以**。Phonon 120 帧全部落入同一个低误差状态；EOS-elastic 重叠明确识别 |
