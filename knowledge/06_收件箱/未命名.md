



## 算法分层（按精度排序）

你手里现在有 **5 个 tier** 的噪声检测算法：

|Tier|方法|计算量|精度|
|---|---|---|---|
|0|Loss 基线（直接阈值 loss）|1×|F1_base|
|1|SCX + Hoeffding 阈值 (Thm 1)|M× + 聚类|指数保证，常数次优|
|2|SCX + KL 阈值 (Thm 1 Chernoff)|M× + 聚类|同 Tier 1，界紧 2-5×|
|3|**SCX + 自适应阈值 (Thm 4')**|M× + 聚类 + $\hat{\eta}$|**精确常数最优**|
|4|+ Bootstrap 稳定性诊断 (Prop 6)|M× + B× 重采样|同上 + 特征诊断|

## 精度 vs 算力的王者

- **最精准**：Tier 3（自适应阈值）。Thm 4' 证明了它在精确常数意义下不可被击败。
- **算力最优**：Tier 1，$M=8$。实践中 $M$ 超过 12-15 后收益递减，$M=8$ 对大多数场景已经够用。
- **实际性价比之王**：Tier 1 + 先跑 Tier 4 的 $\varepsilon_\phi$ 诊断。花 10% 额外算力确认 SCX 是否值得跑，比盲目上 $M=20$ 划算得多。

## 工况速查

|工况|用哪个|原因|
|---|---|---|
|特征很强 ($\varepsilon_\phi < 0.1$)，噪声适中 (5-20%)|Tier 1, M=10|自适应增益小，Hoeffding 够用|
|特征中等 ($0.1 < \varepsilon_\phi < 0.3$)，噪声很稀 ($\eta < 5%$)|**Tier 3**|自适应阈值在这里显著优于固定阈值|
|特征弱 ($\varepsilon_\phi > 0.5$)|**Tier 0**|SCX 退化到基线，别浪费算力|
|不知道特征强弱|Tier 4 先诊断 → 再决定|Bootstrap ARI 告诉你真相|
|CPU only, 小数据 (<1000 样本)|Tier 1, M=5|样本量限制专家数|
|GPU 集群, 大数据 (>100K)|Tier 3, M=15-20|算力充足，追求极致|

### 工程实践速查

**精度排序**：Tier 3 (自适应) > Tier 2 (KL) > Tier 1 (Hoeffding) > Tier 0 (Loss基线)

**算力排序**：Tier 0 (1×) < Tier 1/2/3 (M×) < Tier 4 (M× + B×)

**性价比之王**：Tier 1, M=8-12。M 超过 15 收益递减严重。

**决策树一句话版**：

```
ε_φ > 0.5  →  别用 SCX，Loss 基线就够了
ε_φ ≤ 0.2, η > 5%  →  Tier 1, M=10（够用，简单）
ε_φ ≤ 0.2, η < 5%  →  Tier 3（自适应阈值在稀疏噪声下显著更好）
不知道 ε_φ        →  Tier 4 先诊断（Bootstrap ARI）
```

**四个典型场景**：

|场景|方法|M|原因|
|---|---|---|---|
|学术 CIFAR, 1 GPU|Tier 2|10|KL 界免费比 Hoeffding 好，不增加算力|
|MLIP CPU 集群, 500 帧|Tier 3|8|自适应阈值 + bootstrap 采样补偿小 M|
|医学影像, FDA 监管|Tier 3|12-15|需要最精准 + 完整验证协议|
|大厂 10M 样本|Tier 4→3|20-24|先诊断后裁剪，离线预计算|
