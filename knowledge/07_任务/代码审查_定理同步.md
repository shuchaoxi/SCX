## 2. Theorem 1 与代码：主要差异

Theorem 1 (`01_noise_detection_guarantee.md`) 证明了在特定假设下 F1 的指数收敛下界：

```
F1 ≥ 1 - (1/η)·Σρ_s·exp(-2M·Δ_s²)
```

### 🔴 关键问题 1：代码未使用多专家一致性方案 【⏳ 等 GPU】

**定理要求**：
- M 个专家在 **不相交数据子集** 上独立训练（假设 A1）
- 一致性得分 C(x) = (1/M)·Σe_m(x,y)，基于多专家错误指示变量的均值
- 需要一个阈值 θ 将 C(x) 二值化为噪声判断

**代码现状**：
- 所有实验（CIFAR-10 的 `run_cifar10_noise.py`、scx-health 的 `run_noise.py`）**只训练一个模型**，然后从这个模型的 loss/confidence 做噪声检测
- `scx.valuation.noise_score.NoiseScore` 是 **单模型的残差 * 密度项 * 不一致项**，不是多专家一致性得分
- CIFAR-10 中的 `detect_noise_scx()` 使用 z-score 检测异常残差（z > 2），与定理的 C(x) > θ 无关

**结果**：当前实验不是 Theorem 1 的实现。Theorem 1 的保证在当前代码中**完全不适用**。

### 🔴 关键问题 2：代码中无 Hoeffding/Chernoff 界 【✅ 已修复】

**定理要求**：
- Lemma 2 使用 Hoeffding 不等式推导 FPR 上界
- 附录 A 给出 Chernoff 界的更紧形式
- 这些界是 Theorem 1 的核心证明工具

**代码现状**：
- 搜索 `hoeffding|chernoff|Hoeffding|Chernoff` 在 `src/` 下 **0 个匹配**
- 没有任何概率不等式被实现或引用

**结果**：Theorem 1 的证明工具未转化为任何实用代码（如 F1 下界计算器、最少专家数计算器、最优阈值选择器）。

### 🟡 重要问题 3：C(s) 定义不一致 【✅ 已修复 — docstring 澄清】
- C(x) = 样本级多专家一致性，基于 e_m(x,y)（错误指示变量）
- C(s) = 状态 s 内 C(x) 的期望

**代码定义**（`learnability.py:LearnabilityScore.consistency`）：
- C(s) = 标签纯度（分类）或方差压缩（回归），或专家共识方差
- 是纯状态级度量，不基于 e_m(x,y)

**结果**：符号重载但定义不同。代码的 C(s) 是一个有用度量，但与 Theorem 1 的 C(x) 无直接关系。

### 🟡 重要问题 4：假设 A1 被违反 【⏳ 等 GPU — 多专家实验】

**定理假设 A1**：M 个专家在不相交的 i.i.d. 子集上训练。

**代码现状**：
- 实验只训练一个模型用于特征提取和残差计算
- 多专家训练方案从未被实现
- 即使是 scx-health 中的 `ErrorDriven` 检测方法（模型扰动），也不是独立训练

### 🔴 关键问题 5：噪声检测实验方法学问题 【✅ 已修复 — z>2.0→θ*】

CIFAR-10 实验中的 `detect_noise_scx()` (第 140-171 行)：
```python
def detect_noise_scx(features, residuals, state_labels, n_states):
    for s in range(n_states):
        z_scores = (state_residuals - mean_r) / std_r
        flagged = z_scores > 2.0  # 硬编码 z=2 阈值
```
- 使用状态内 z-score，与 Theorem 1 无关
- 硬编码 z=2 阈值，非定理的最优 θ*

## 4. V(s) 循环定义问题

### 状态：V(s) 仍在代码中 【✅ 已修复 — deprecated + 定理方法替代】

`src/scx/valuation/state_value.py` 实现了：

```
V_add(s) = r_bar(s) * rho(s) * L(s) * [1 - D(s)] * max_m SCX_m(s)
V_remove(s) = rho(s) * (1 - r_bar(s)) * Sim(s) * (1 - Boundary(s))
```

两者均在 `valuation/__init__.py` 中导出，是公共 API 的一部分。

### 依赖性分析

| 组件 | V_add 依赖 | V_remove 依赖 |
|------|-----------|--------------|
| r_bar(s) | 直接 | 直接 (1 - r_bar) |
| rho(s) | 直接 | 直接 |
| L(s) | 直接 | -- |
| D(s) | 直接 (1 - D) | -- |
| max_m SCX_m(s) | 直接 | -- |
| Sim(s) | -- | 直接 |
| Boundary(s) | -- | 直接 (1 - Boundary) |

**调用方**（18 个测试，5 个源文件引用）：
- `action/policy.py` — decide actions based on V_add/V_remove
- `utils/visualization.py` — plot state value bars
- `core/framework.py` — stores/loads _state_values
- `valuation/influence.py` — uses scx_state_values for selection
- `valuation/__init__.py` — exports StateValue

### 循环性评估

实际上**不存在真正的循环计算**——V(s) 的输入（r_bar, rho, L, D, SCX）都是预先独立计算的。循环性是指**概念层面**：
- L(s) 需要知道状态的可学习性（需要 ground truth）
- D(s) 需要 r_bar(s) 的正确估计（需要 ground truth）
- max_m SCX_m(s) 需要知道谁是好专家（需要 ground truth）

在**实际代码执行路径**中，这些值被顺序计算（先 learnability -> 再 redundancy -> 再 state_value），没有循环。循环性是**理论上的**：要正确计算所有这些值，你需要已经知道 ground truth。

### 操作建议

如果论文计划要求**删除** V(s)：需要同时修改 5 个文件和 `__init__.py`，并移除 18 个测试用例。这可行但影响面广。如果要保留，建议添加文档字符串声明为"oracle version"。

### DataClassifier 的硬编码阈值 【✅ 已修复 — 定理注释 + AdaptiveThreshold】

| 阈值 | 默认值 | 理论依据 |
|------|-------|---------|
| error_high | 0.05 | 无 — 经验值 |
| density_high | 0.05 | 无 — 经验值 |
| consistency_high | 0.7 | 无 — 经验值 |
| redundancy_high | 0.8 | 无 — 经验值 |
| noise_high | 0.5 | 无 — 经验值 |
| expert_gap | 0.3 | 无 — 经验值 |

**Theorem 1 给出的最优阈值**：
```
θ* = ½(1 + μ_max · (K-2)/(K-1))
```
其中 μ_max = 最大状态清洁错误率，K = 类别数。这**未被使用**。

### 其他数值稳定性

- `eps = 1e-8` 在所有地方一致使用（除法保护）
- `max(0.0, ...)` 和 `np.clip` 保护数值范围 — **良好实践**
- 指数项 `exp(-2M*Δ²)` 在 Theorem 1 中可能出现下溢（当 MΔ² 很大时），但实际场景中 M 通常较小，问题不大
