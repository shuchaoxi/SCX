# 决策日志

> 关键决策及其原因。原文备份：[决策日志.md](../../CodexKnowledge/决策日志.md)

## 决策 1：SCX 从 EGP 独立（6/25）

**决策**：将第 4 篇论文（SCX）从 EGP 项目中独立出来，建立 `SCX/` 独立项目和 GitHub 私密仓库。

**Why**：
- SCX 本质是数学理论/ML 方法论文，不依赖 DFT 计算
- 独立项目便于专注理论开发、合成实验和 ML benchmark 验证
- EGP 的 Paper 1-3 继续在材料势函数主线迭代
- 两线并行推进，互不阻塞

## 决策 2：SCX 定位为数学框架 + 轻量验证（6/25）

**决策**：不写成大模型工程论文，定位为"数学框架 + 轻量验证"。

**Why**：
- 博士阶段资源有限，无法负担 LLM 标注/训练实验
- 数学框架本身是核心贡献，合成实验足以验证理论
- 大模型数据治理是应用前景，不是实证主战场

## 决策 3：命名 SCX（6/25）

**决策**：框架命名为 **SCX: State-Conditioned eXpertise**。

**Why**：
- 短、好记、像算法名
- State-Conditioned = 核心思想（状态条件）
- eXpertise = 专家能力/可靠性/专业性
- 可扩展：SCX reliability、SCX data value、SCX expert routing

## 决策 4：EGP-first 发表策略（6/26）

**决策**：论文序列为 EGP Paper 1 → SCX-Theory → SCX-MLIP → SCX-Sim → SCX-Health。

**Why**：
- 先 concrete（EGP 有完整 DFT 数据）打根据地
- 再 abstract（SCX-Theory）建理论
- 再 apply back（SCX-MLIP）验证
- 再跨领域扩展（SCX-Sim, SCX-Health）
- 降低风险：EGP 先发确保有 publication，SCX arXiv 再占坑

## 决策 5：v4.0 通用模块化重构（6/26）

**决策**：SCX v4 进行通用模块化重构，核心是"抽象接口 + Encoder 插件 + YAML 配置"三层分离。

**触发**：用户在与 GPT 对话中指出"框架还是需要更通用一些，避免一直加模块迭代麻烦。"

**Why**：
- 每加一个领域写一个 adapter，代码重复严重
- 重构后新增领域的适配成本从"2-3 周"降到"2-3 天"
- 商业上可以快速生成"试点 → 审计报告"的标准流程

## 决策 6：AI 代码生成分工（6/25 确立）

**决策**：用户负责数学框架 + 架构设计，AI 负责 Python 实现。

**Why**：
- 用户的核心竞争力在理论创新和架构设计
- AI 在代码实现和测试编写上效率极高
- 30 个文件、259 个测试在 24 小时内完成就是证明
- 这是一种有意为之的分工策略

## 决策 7：IP 保护策略（6/26）

**决策**：论文确权 + 最小开源 + 核心工程保留 + 可商业授权。

详见 [[IP策略]]。

## 决策 8：定理放置方案 D（6/27）

**决策**：定理 1+2+3 的论文放置采用方案 D——Paper 1 (Nature Comput. Sci.) 正文 ~1000 words 轻量版 + SI ~8 页完整版，Paper 3 (JMLR) 深度理论放松假设版。

**Why**：
- 方案 A（全在 Paper 1）因 Nature 正文篇幅限制不可行（~1000 words 只能容纳直观陈述）
- 方案 B（独立理论论文）经五审查员全票否决——理论需要实证支撑
- 方案 C（拆分两篇）逻辑割裂
- 方案 D 兼顾了 Nature 的叙事流畅性和 JMLR 的理论深度

**备选方案**：方案 A（全在 Paper 1）被篇幅否决；方案 B（独立理论 paper）被评审策略否决；方案 C（Thm 1 在 Paper 1，Thm 2+3 在 Paper 3）被认为逻辑割裂。

## 决策 9：A6 假设追加（6/27）

**决策**：在定理 1 的 5 个假设基础上追加假设 A6（Expert Calibration）——专家提供的置信度/一致性分数经过校准。

**Why**：
- 独立验证中发现，专家置信度未校准时，Chernoff bound 的实证表现低于理论值
- A6 确保了专家 p_s 的估计是无偏的
- 校准假设在 Dawid-Skene 和 crowd-sourcing 文献中是标准做法

**影响**：A6 追加后，定理 1 的 5→6 假设。Paper 3（JMLR 深度版）中计划放松此假设。

## 决策 10：5-paper vs 4-paper 路线图（6/27）

**决策**：从 4-paper 路线图扩展为 5-paper，新增 Paper 5（SCX 应用部署路线图/商业化）。

**Why**：
- 学术影响力需要落地验证——只有实证论文不足以驱动采用
- Paper 5 提供商业化路径：开源库 → 行业试点 → 评估基准 → 商业授权
- 整合「博士论文故事线」：理论（Paper 3）→ 验证（Paper 1）→ 应用（Paper 5）
- 不额外增加时间成本——Paper 5 内容可在 Paper 1+3 投稿间隙撰写

**反对意见**：5 篇可能分散精力。回应：Paper 5 不是新增工作量，而是现有工作（部署架构、领域适配模板）的整合。

## 决策 11：V(s) 循环定义废弃与 StateValue 重构（6/27）

**决策**：废弃原 V(s) 状态价值函数（标记为循环定义），重构为 `StateValue` 类，引入"外部参考分布"打破循环。

**Why**：
- 原 V(s) 定义存在循环依赖：状态价值依赖标注价值，标注价值依赖状态划分
- 重构后的 StateValue 将状态价值定义为相对于参考分布的期望提升
- 新架构天然支持 theorem-based 方法（detect_noise / detect_weak_features / analyze_unidentifiability）

**影响**：V(s) 相关接口全部标记为 `@deprecated`，`StateValue` 成为新标准。

**决策**：新增模块命名为 `yajie/`（SCX 前向 + Spring plan 配置）。

## 决策 13：Arrow analogy 移除（6/27）

**决策**：从理论描述中移除 Arrow 类比（"SCX 是噪声版的 Arrow 不可能定理"）。

**Why**：
- Arrow 类比适用于 social choice（偏好聚合），与 SCX（噪声检测）数学结构不同
- 审稿人可能质疑类比的有效性并分散注意力
- 定理 3（噪声不可辨识性）自身已提供足够的必要性论证
- 移除后理论叙事更清晰：Thm 1（正向）→ Thm 2（反向）→ Thm 3（必要性）

**存档位置**：类比内容移入 `02_概念/已归档/`，不影响主线叙事。

## 决策 14：独立定理验证（6/27）

**决策**：定理证明完成后，由独立审查者（AI reviewer）进行白盒审查 + 对抗性测试。

**Why**：
- 作者/实现者存在认知盲点——容易默认"显然成立"的步骤实际有漏洞
- 该决策被实践验证极其正确：发现 Thm 1 的 2 个 bug + Thm 2 的 3 个 issue
- 特别是 Lemma 3 的 Hoeffding/Chernoff 误用，属于经典错误
- 建立标准化验证流程：白盒审查 → 对抗性测试 → 回归修复 → 自动化验证

**成本收益**：~3 小时审查时间避免了论文发表后可能被发现的错误。决定将其标准化为定理开发的固定环节。

## 决策 15：深水区理论探索结果（2026-06-27）

**决策**：三个深水区理论方向（聚类一致性、Minimax 最优下界、BBP 谱代理）全部完成首轮完整证明和独立审查，全部 reject。决策：**延期至 Paper 1 投稿后**，当前集中精力于 Paper 2 投稿 + Paper 1 GPU 实验。

**尝试了什么**：
- 聚类一致性：完整证明推广 Pollard (1981) k-means 一致性到 SCX 两层架构 → **Reject**：bound 发散（致命代数错误：分母 1-K/m 在 K>m/2 时为负），k-means NP-hard 假设 gap
- Minimax 最优下界：Le Cam 两点法 + 混合-积 TV 界证明 F1 下界最优性 → **Reject**：Slud 不等式在 M=5, p=0.1 时数值上被证伪（1.75 > τ(0.1)），F1 推导中分母出现负值（η > 0.5 时 2-1/(1-η) < 0）
- BBP 谱代理：Gram 矩阵最大特征值替代互信息 δ 的谱诊断 → **Reject**：互信息 ≠ 最大特征值（除非协方差低秩结构完全由 s 决定），高斯假设在 MLIP 数据中不成立

**为什么做这次冲刺**：
- 核心定理（1+2+3）上午全部完成，剩余时间想趁热打铁攻克深水区
- 低估了深水区理论的推导难度——每个方向都是 SCX 耦合结构 + 非标准工具的组合
- 时间压力导致了"赶工式推导"，独立审查发现所有致命错误

**学到了什么**：
1. 深水区理论的推导质量需要系统性提升：推导前先做假设清单 + 数值 sanity check vs 极端参数
2. 独立审查流程有效（发现所有致命错误），应成为深水区推导的标准环节
3. 每个方向的 bug 都是具体的、可修复的——不是原理走不通，是推导质量不够
4. "同时推三个定理 = 一个都推不动"这个早先的判断被完全验证

**为什么延期而不是放弃**：
- 三个方向的学术价值判断不变。聚类一致性的 K* scaling law 对实践仍有极高价值
- 方向确定可行——只需更慢、更系统的推导
- Paper 1 投稿后、等审稿意见的 3-6 个月是恢复深水区理论的最佳时间窗口

- Reviews: `theory/explorations/review_cluster_consistency.md`, `theory/explorations/review_minimax_lower_bound.md`, `theory/explorations/review_bbp_spectral_proxy.md`

## 决策 16：第二轮深水区理论结果 — 全部修复/转向（2026-06-27 收盘前）

**决策**：首轮 reject 后立即启动第二轮修复，收盘前完成。Minimax 下界通过 Hellinger 距离成功复活（非 Slud）；聚类一致性以固定 K、Pollard-style 重写；BBP 谱代理完全放弃谱分析，转向 Bootstrap ARI 聚类稳定性诊断。

**结果**：
- **Minimax 下界** ✅ **Resurrected via Hellinger**：弃用 Slud → Bretagnolle-Huber 不等式（Hellinger 距离版本）。修复 2 个 F1 推导 bug（FP 非对称缩放 + 核泛化界耦合）。核心速率最优性结果：`liminf (-1/M) log(lower_bound) = 2Δ²`。文件：`theory/explorations/minimax_lower_bound_v2.md`（1002+ 行）
- **聚类一致性** ✅ **Rewritten with fixed K**：增长-K 发散 → 固定 K。Pollard-style 4 引理结构。Lloyd's + random restarts（放弃全局最优假设）。749 行。文件：`theory/explorations/cluster_consistency_v2.md`
- **BBP 谱代理** ♻️ **Pivoted to stability diagnostic**：信息-谱桥接断裂确认（互信息 ≠ 最大特征值无通用不等式）。转向 Bootstrap ARI 聚类稳定性诊断——无分布假设，适用于任意特征类型。文件：`theory/explorations/feature_strength_via_stability.md`

**为什么第二轮**：
- v1 审查确认每个方向都有**具体的、可修复的** bug——不是原理走不通
- v1 的独立审查报告提供了明确的修复方向，修复成本可控
- 收盘前仍有时间窗口，判断值得完成修复而不留给未来

**关键教训**：正确的数学工具选择至关重要。
1. **Hellinger > Slud**：对于 F1 的比值形式，Hellinger 距离的乘法结构优于总变差的加法结构
2. **Pollard 固定-K > 增长-K 渐近**：SCX 的耦合结构下高维渐近边界条件复杂，固定-K 经典结果已足够
3. **Bootstrap > 谱分析**：对于实践中的可计算诊断，非参数的 Bootstrap 方法优于需要强分布假设的谱方法

**对项目的影响**：
- 三个深水区方向的 v2 证明均已通过审查，可直接用于 Paper 3 (JMLR) 的写作
- 不需等待 Paper 1 投稿后再修复（原计划 v1 reject 后的延期方案）
- 下一步：数值验证 Bootstrap ARI（可 CPU 跑）+ Paper 3 整合写作
