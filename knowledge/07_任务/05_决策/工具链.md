# 工具链策略

## 核心工具

| 工具 | 用途 | 决策原因 |
|------|------|---------|
| **Claude Code** (Anthropic) | 主 AI 编程助手 | 多 Agent 协作、Worktree 隔离、最强代码生成 |
| **deepseek v4 pro** | 后端模型 | 1M context window，处理大型对话和分析 |
| **VS Code** | IDE | 集成 Claude Code 扩展 |
| **Obsidian** | 知识管理 | 双向链接 + 图谱视图适合思想演化追踪 |
| **GitHub 私密仓库** | 版本控制 + IP 保护 | 私密 = 未公开披露 |

## AI 使用策略

### 用户做什么
- 数学框架设计（定义、命题、证明）
- 架构设计（模块划分、接口定义）
- 设计方向决策（选什么方向、按什么顺序）
- 最终审查和签字

### AI 做什么
- Python 代码实现
- 单元测试编写
- API 文档字符串
- 实验脚本生成
- 分析报告撰写
- 代码审查

### 策略哲学

> 用户是数学家/理论家 + 架构师，AI 是实现工程师。这不是"AI 替我做"，而是"我设计，AI 实现"。

## 超算使用策略

| 超算 | 用途 | 与 SCX 关系 |
|------|------|------------|
| 孝感超算 (cxshu) | EGP DFT 计算 | **仅 EGP Paper 1-3 的 VASP 计算** |
| 个人电脑 | SCX 全部开发 | **SCX 所有理论 + 代码 + 实验** |

SCX 不依赖超算资源。

## 数据安全

- 所有源码存储在 GitHub 私密仓库
- IP 相关文档（IP_NOTE, DEVELOPMENT_LOG, CLEAN_ROOM_CHECK）有 git history 作为时间戳证据
- 超算密码不写入任何仓库文件
