# Agent 讨论：两层 EGP 方案

> 原文：[FINAL_SYNTHESIS.md](G:\仿真数据\egp_archive\agent_discussions\20260614_two_layer_egp\FINAL_SYNTHESIS.md)

## 讨论概要

多 Agent（提案者 + 审查者 + Critic）围绕"两层 EGP"方案进行了深度讨论，形成了强共识。

## Agent 共识

> 材料类别提供可解释的粗粒度专家先验，而误差景观与可微局域环境描述符共同决定连续路由权重。

### 不可谈判的物理约束
- 总能量 = Σ_i Σ_k w_ik(x_i) · e_ik
- 力必须来自混合总能量的自动微分：F = -∂E/∂R
- **禁止** 简单专家力加权平均

### Critic Agent 的核心提醒
> 两层 EGP 最容易失败为"看起来物理，实际只是更复杂模型和数据标签带来的伪提升"。

## 确定的 5-Gate 审批流程

```
Gate 1: 数据盘点 → Gate 2: dry-run MVP → Gate 3: baseline 
→ Gate 4: error landscape → Gate 5: 真实 EGP 训练
```

## 10 条"不要做"

Critic Agent 列出了 10 条当前阶段不应做的事，包括：
- 不要提交真实 HPC 任务
- 不要使用 temperature/pressure 作为 router 输入
- 不要使用 source label / 文件夹名作为 router 输入
- 不要在缺少 GaN/AlGaN DFT 数据时做论文级性能结论

## 意义

这是 SCX 全部思想的**第一个 formal 讨论**。它确立了：
1. 保守推进策略（5-Gate）
2. 物理一致性是不可谈判的
3. 需要 baseline 对比和 held-out 验证
4. 数据标签不能泄漏给 router
