# 专家路由

## 问题

多专家系统中，每个输入 $x$ 应该由哪些专家处理？权重如何分配？

## SCX 专家路由

SCX 将专家路由建模为**状态条件概率**问题：

$$w_m(x) = \frac{SCX_m(s(x)) \cdot \mathbb{1}[x \in \text{Domain}_m]}{\sum_k SCX_k(s(x)) \cdot \mathbb{1}[x \in \text{Domain}_k]}$$

其中 $s(x)$ 是将 $x$ 映射到其状态的函数。

## Governance Protocol（命题 5）

在专家进入蒸馏/生产前，必须通过 5 步协议：

```
1. Gauge Check        → 能量参考系是否对齐？
2. Domain Certificate  → 专家声称的域是否被正式验证？
3. Conflict Resolution → 当多个专家对同一输入给出矛盾预测时如何仲裁？
4. Anchor Verification → 在已知 DFT 标签的 anchor 点上验证
5. Distillation Auth   → 只有通过以上所有步骤的专家才能进入蒸馏
```

协议输出每个 (expert, state) pair 的状态：
- `certified` — 完全可靠
- `conditionally_certified` — 在特定条件下可靠
- `uncertified` — 数据不足，无法认证
- `rejected` — 不可靠

## 独有创新

Expert Governance 是 SCX 在竞争格局中的**独有创新区**——没有已有工作在 MLIP 领域讨论跨专家冲突仲裁和域认证。
