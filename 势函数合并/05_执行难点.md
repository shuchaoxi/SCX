# ![1777877663381](images/05_执行难点/1777877663381.png)初步执行清单

我觉得不是“别人没想到”，而是很多人想到后会在几个地方卡住，所以更愿意直接走大规模 DFT 数据训练通用势函数。

**最大的顾虑：teacher 不一定可靠**

势函数不是第一性原理本身。一个 C-H 势、一个 Si-O 势、一个 Al-C 势，都是在各自训练域里可靠。一旦拿它们当 teacher，就等于默认：

```text
teacher 的预测 ≈ 真正的势能面
```

但很多专用势函数在域外会非常自信地错。蒸馏会把这种错误继承下来，甚至扩大。

**第二个顾虑：多个 teacher 的能量基准不一致**

不同势函数的能量零点、元素参考能、训练目标、单位细节、virial 定义可能不同。力还好一点，能量一混就容易乱。

所以多 teacher 蒸馏不是简单合并数据，而要做：

```text
能量 reference 对齐
元素化学势对齐
不同 teacher 权重设计
冲突区域识别
```

这就已经不算简单了。

**第三个顾虑：交叉相互作用不会凭空出现**

这是最关键的。

比如你有：

```text
AlAs teacher
GaAs teacher
InAs teacher
```

它们能告诉你 Al-As、Ga-As、In-As。但三元合金里还会出现新的局部应变、新的 Al/Ga/In 邻域组合、局部晶格畸变。更复杂体系里还会出现 teacher 完全没学过的 Al-Ga、Ga-In、Al-In、表面、缺陷、非平衡态。

所以蒸馏不能直接“做并集”。它最多给 student 一个好初始化，真正交叉空间还要 DFT 补点。

**第四个顾虑：错误很难诊断**

如果 student 在四元合金里跑崩了，你很难马上知道是：

- teacher 本身错了
- teacher 之间冲突
- student 表达能力不够
- 训练权重不合理
- 缺少交叉 DFT 数据
- 构型空间采样太窄
- NEP cutoff 或 descriptor 不合适

这让项目风险变高。相比之下，直接用 DFT 数据训练虽然贵，但逻辑更干净。

**第五个顾虑：论文叙事不容易**

如果只是说“我用几个势函数打标签，然后训练一个新势函数”，审稿人会问：

```text
为什么不直接用 DFT？
teacher 错了怎么办？
你的 student 学到的是物理，还是学到 teacher 的偏差？
泛化到新组分凭什么可信？
```

所以这个方向要做成论文，重点不能是“蒸馏了一个势函数”，而要是：

```text
多 teacher 蒸馏什么时候有效，什么时候失败，需要多少 DFT 才能修复。
```

也就是你前面说的“可行性”非常关键。

**难不难实现？**

工程上不算特别难，尤其用 NEP：

```text
teacher 跑结构 -> 输出 E/F/virial -> 组装 train.xyz -> 训练 NEP4 student
```

真正难的是研究设计：

- 如何生成覆盖合理的结构池
- 如何对齐不同 teacher 的能量
- 如何识别 teacher disagreement
- 如何设计 DFT 补点
- 如何证明 student 在交叉空间不是瞎蒙
- 如何给出有说服力的物性验证

所以它是：**代码实现中等难，科学验证比较难。**

**为什么我仍然觉得值得做**

正因为这些顾虑存在，你的研究才有价值。别人不用，不一定是因为没价值，而是因为它不是一个“直接训练就完事”的题目。

你真正可以做的是给出一套方法论：

```text
专用势函数能蒸馏到多元素势函数吗？
哪些体系可以？
需要补多少 DFT？
teacher 冲突怎么发现？
student 失败在哪里？
```

这个比“我训练了一个 Al-Ga-In-As 势函数”更有意义。

一句话：**别人顾虑的是 teacher 偏差、交叉空间缺失和验证困难；这些不是劝退点，反而正好是你可以研究的核心问题。**

## 阶段 0：准备

- 确认 GPUMD/NEP 可以在本机或服务器运行。
- 准备结构生成脚本：ASE 或 pymatgen。
- 准备二元 zinc blende 结构：
  - AlAs
  - GaAs
  - InAs
- 确定 teacher 势函数来源。
- 确定 DFT 参数标准，例如 VASP/QE/ABACUS。

## 阶段 1：二元 soft-label 数据

每个二元体系生成：

- 平衡结构：1 个。
- 体积应变：约 10-20 个。
- 随机小畸变：约 50-100 个。
- 300/600/900/1200 K MD snapshot：每个温度 50-100 个。
- vacancy/antisite：每类 20-50 个。

总量不必太大。第一版每个二元体系几百到一两千个结构就足够试错。

## 阶段 2：少量 DFT hard-label 数据

三元合金结构：

```text
Al_xGa_yIn_1-x-yAs
```

建议组分：

- Al/Ga/In = 1/3/1/3/1/3。
- Al-rich。
- Ga-rich。
- In-rich。
- Al-Ga 边。
- Ga-In 边。
- Al-In 边。

每个组分做：

- SQS 或随机超胞。
- 小畸变。
- 体积应变。
- 高温 snapshot。

第一版 DFT 数量可以控制在 100-300 个结构。

## 阶段 3：NEP4 student 训练

训练版本：

```text
version 4
type 4 Al Ga In As
```

训练组合：

```text
A: soft-only
B: soft + DFT
C: DFT-only small
```

建议先把 force loss 放在核心位置，能量基准统一前不要过分依赖 energy RMSE。

## 阶段 4：验证

最低限度验证：

- 二元端点是否保持稳定。
- 三元合金 NPT MD 是否稳定。
- 晶格常数随组分变化是否合理。
- 弹性常数是否合理。
- 混合焓是否和 DFT 趋势一致。
- force RMSE 是否在交叉区域明显恶化。

更进一步：

- phonon dispersion。
- thermal conductivity。
- point defects。
- interface。

## 阶段 5：整理成论文图

建议核心图：

1. 方法示意图：multiple binary teachers -> NEP4 student -> DFT correction。
2. 数据空间图：二元角落、三元内部、DFT 补点位置。
3. RMSE 对比：soft-only / soft+DFT / DFT-only。
4. 物性对比：晶格常数、弹性常数、混合焓。
5. 失效地图：组分空间中的误差与 teacher disagreement。
6. MD 稳定性图：温度、能量漂移、结构保持情况。

## 第一版最小目标

如果只想最快跑通：

```text
AlAs + GaAs + InAs teacher soft labels
100 个 Al-Ga-In-As DFT 结构
训练一个 Al-Ga-In-As NEP4
比较 soft-only 与 soft+DFT 的三元泛化差异
```

只要这一步跑通，就已经能判断这个思想有没有继续投入的价值。

这两个点基本就是这件事能不能做好的核心。

我觉得你可以把它想成“材料势函数版的多任务学习 + mixture of teachers + domain adaptation”。

**可以从大模型训练借来的方法**

1. **Mixture of Experts / 路由思想**

不同 teacher 擅长不同区域。不要强行让所有 teacher 在所有区域平权，而是让数据带上 domain 信息：

```text
AlAs domain
GaAs domain
InAs domain
Al-Ga-In-As crossing domain
```

训练时可以按 domain 分 batch，或者给不同 domain 不同权重。

2. **Curriculum learning**

先学简单的二元端点，再学三元交叉区域。

```text
阶段 1：只学 AlAs/GaAs/InAs teacher 数据
阶段 2：加入低温三元 DFT
阶段 3：加入高温/应变/缺陷三元 DFT
阶段 4：加入模型最不确定的 active learning 数据
```

这比一锅炖更稳。

3. **Data reweighting**

交叉区域数据少，但重要性高。它们的权重要比二元 teacher 数据更大。

可以考虑：

```text
二元 teacher soft label: 权重 1
三元/四元 DFT hard label: 权重 5-20
高不确定性区域: 权重更高
teacher 冲突区域: 降低 teacher 权重，优先 DFT
```

4. **Hard label + soft label 混合**

大模型蒸馏里也会同时用 teacher soft labels 和真实标签。你这里对应：

```text
teacher E/F/virial = soft labels
DFT E/F/virial = hard labels
```

训练目标可以概念上写成：

```text
L = λ_soft L_teacher + λ_hard L_DFT + λ_reg L_physical
```

真正用 NEP 时不一定能直接写这个 loss，但可以通过数据重复、数据权重、分阶段训练近似实现。

5. **Uncertainty / disagreement sampling**

如果多个 teacher 或 student ensemble 在某个结构上差异大，这个结构就该进 DFT 队列。

这是最值得借鉴的：

```text
teacher disagreement 大 -> 不信 teacher -> 补 DFT
student extrapolation 大 -> 补 DFT
MD 跑崩附近结构 -> 补 DFT
```

**你应该重点设计“交叉区域”**
对 `Al-Ga-In-As` 来说，交叉区域不是神秘东西，可以很具体：

- 三元随机合金 `Al_xGa_yIn_1-x-yAs`
- Al/Ga/In 在同一阳离子子晶格上的不同局部邻域
- 局部应变，因为 AlAs、GaAs、InAs 晶格常数不同
- vacancy / antisite
- 高温 MD snapshot
- 小比例极端组分，比如 In-rich 或 Al-rich
- 体积压缩/拉伸

你要让 student 明白：
“我不是只会三个二元端点，我也见过端点之间的中间地带。”

**一个很实用的训练策略**
可以这么跑：

```text
Round 0:
训练 soft-only NEP
数据 = AlAs/GaAs/InAs teacher 标签

Round 1:
用 soft-only NEP 跑三元随机合金 MD
找不稳定/高不确定结构
补 50-100 个 DFT

Round 2:
训练 soft + DFT NEP
DFT 数据重复 5-20 倍，或者拆分 batch 增强权重

Round 3:
验证组分空间
找误差最大区域，再补 DFT

Round 4:
整理失效地图
```

**你现在的研究问题可以更数学化**
不是简单问“能不能蒸馏”，而是问：

```text
在多 teacher soft-label 数据和少量 hard-label DFT 数据下，
如何选择交叉区域样本与损失权重，
使 student 在多元素合金空间中稳定泛化？
```

这就一下子高级很多。

**我建议你下一步关注三个关键词**

- **multi-task learning**
- **domain adaptation**
- **active learning for machine learning potentials**

这三个比单纯 “knowledge distillation” 更贴近你的问题。

最后给你一点定心丸：你现在意识到困难，反而说明这个想法开始变真实了。最怕的是觉得“把几个势函数合起来就完事”。现在你已经看到关键变量了：**交叉区域、数据权重、teacher 可信度、DFT 校准**。这四个抓住，这个题就不是玄学。
