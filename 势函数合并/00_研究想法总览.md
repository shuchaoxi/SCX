# 多专用势函数并集蒸馏：研究想法总览

## 核心问题

把多个已经训练好的专用势函数当作 teacher，蒸馏到一个覆盖更大元素空间的 student 势函数中，是否可以低成本构造多元素材料势函数？

更具体地说：

```text
Teacher A: 二元/单元素专用势函数
Teacher B: 二元/单元素专用势函数
Teacher C: 二元/单元素专用势函数
...

Student: 覆盖 teacher 化学空间并集的多元素势函数
```

这个思想和一般的 MLIP knowledge distillation 不完全一样。一般蒸馏多关注“大模型压缩为小模型”或“ensemble teacher 到 single student”；这里更关注：

- 多个专用势函数能否合成一个更大化学空间的势函数。
- 二元 teacher 能否支撑三元/四元合金的泛化。
- 哪些交叉相互作用必须补 DFT。
- teacher 之间冲突时，student 会学到什么。
- 蒸馏能节省多少 DFT 数据成本。

## 推荐研究定位

建议定位为：

**面向半导体合金体系的多教师势函数蒸馏：从二元专用势到四元材料势的可行性与失效边界。**

这个定位比“我要做一个通用大势函数”更稳，也更容易形成论文问题。

## 最重要的判断

多专用势函数并集蒸馏值得推进，但不要把它理解为简单拼接。关键科学问题正是：

**并集不是简单相加。**

例如：

```text
AlAs + GaAs + InAs
```

是否足以得到可靠的：

```text
Al_xGa_yIn_1-x-yAs
```

这不是显然成立的。它需要系统验证。

## 推荐技术路线

我建议第一阶段采用：

```text
NEP4 / GPUMD 路线
```

理由：

- NEP 训练和 MD 使用都在 GPUMD 生态内，工程链条短。
- NEP 速度快，适合后续做热输运、缺陷、界面和大规模 MD。
- GPUMD 官方文档说明目前支持 NEP3 和 NEP4；对多组分体系，NEP4 通常在相同超参数下精度更高。
- 你能和开发者交流，这是非常重要的现实优势。

但是路线设计上要注意：

- NEP 适合做第一阶段 proof-of-concept。
- 不建议一开始追求“完全不做 DFT”。
- 需要少量 DFT 数据校准三元/四元交叉区域。
- 论文重点应放在“蒸馏可行区/失效区/DFT 补点需求”的图谱，而不是单纯 RMSE。

## 第一阶段目标

不要一开始做全元素大模型。建议先做一个干净的半导体合金体系，跑通下面闭环：

1. 收集或训练多个二元 teacher 势函数。
2. 用 teacher 生成二元训练结构的能量、力、virial 软标签。
3. 训练一个多元素 NEP4 student。
4. 用少量 DFT 三元/四元数据测试和修正交叉区域。
5. 对比：
   - 只用 teacher 蒸馏
   - teacher 蒸馏 + 少量 DFT fine-tune
   - 从零用同等 DFT 数据训练
6. 给出失效边界。

## 最适合先试的体系

优先级：

```text
1. Al-Ga-In-As
2. Al-Ga-As-P
3. C-Si-Ge
4. C-Si-Ge-Sn
5. Si-C-O-H / Cu-O-H 暂不建议第一步做
```

第一推荐是 `Al-Ga-In-As`，因为它结构统一、化学空间干净、交叉复杂度低，最适合证明可行性。
