\subsection{1. Mathematical Domain Attribution}\label{mathematical-domain-attribution}

The Spring self-evolution algorithm does not belong to a single mathematical domain but spans seven classical mathematical domains, synthesizing an entirely new structure at their intersection.

\subsubsection{1.1 Stochastic Approximation}\label{stochastic-approximation}

\textbf{Root}: Robbins \& Monro (1951). ``A stochastic approximation method.'' \emph{Annals of Mathematical Statistics}, 22(3), 400-407.

\textbf{Role in Spring}: Online updates of NEP student parameters \(\theta_t\) follow the Robbins-Monro recursion:

\[\theta_{t+1} = \theta_t - \alpha_t \nabla_\theta \ell(f_{\theta_t}(x_t), y_t)\]

where the learning rate \(\alpha_t\) satisfies the standard RM conditions: \(\sum \alpha_t = \infty\), \(\sum \alpha_t^2 < \infty\).

Spring generalizes standard RM to a \textbf{coupled two-timescale} setting: - \textbf{Fast timescale}: student \(\theta_t\) updates at rate \(\alpha_t\) under a fixed gatekeeper \(S_t\) - \textbf{Slow timescale}: gatekeeper \(S_t\) updates at rate \(\beta_t = o(\alpha_t)\) under student quasi-steady-state

This corresponds to Borkar (2008)'s two-timescale stochastic approximation framework. Condition C6' (\(\beta_t = o(\alpha_t)\)) is the key innovation — it simultaneously guarantees local student convergence and bounded cumulative distribution shift.

\textbf{References}: - Robbins \& Monro (1951) - Kushner \& Yin (2003). \emph{Stochastic Approximation and Recursive Algorithms and Applications} - Borkar (2008). \emph{Stochastic Approximation: A Dynamical Systems Viewpoint} - Polyak \& Juditsky (1992). ``Acceleration of stochastic approximation by averaging''

\subsubsection{1.2 Dynamical Systems / Lyapunov Stability}\label{dynamical-systems-lyapunov-stability}

\textbf{Root}: Lyapunov, A. M. (1892). \emph{The General Problem of the Stability of Motion}. Kharkov Mathematical Society.

\textbf{Role in Spring}: The Spring self-evolution loop is a discrete dynamical system:

\[z_{t+1} = \Phi(z_t), \quad z_t = (S_t, M_t, \theta_t) \in \mathcal{Z}\]

where \(\Phi = (\Phi_S, \Phi_M, \Phi_\theta)\) is the update operator and \(\mathcal{Z} = \mathcal{F} \times \mathcal{M} \times \Theta\) is the evolution state space.

The Lyapunov function candidate is (evaluated on a fixed reference set \(M_0\)):

\[\Psi(S_t, \theta_t) = \frac{1}{|M_0|} \sum_{x \in M_0} (S_t(x, y(x)) - \hat{C}(x))^2 + \lambda \cdot \frac{1}{|V_0|} \sum_{(x,y) \in V_0} \ell(f_{\theta_t}(x), y)\]

The Lyapunov descent property \(\mathbb{E}[\Psi_{t+1} \mid \mathcal{F}_t] \leq \Psi_t\) is fully proved under the reference set replay mechanism via Theorem 12.5. The fixed point \((S^*, \theta^*)\) satisfies self-consistent coupled equations; the \(\omega\)-limit set is a singleton.

Spring uses the complete dynamical systems toolkit: attractors, basins, phase diagrams (four convergence mechanisms), \(\omega\)-limit sets.

\textbf{References}: - Lyapunov (1892). \emph{The General Problem of the Stability of Motion} - Khalil (2002). \emph{Nonlinear Systems} (3rd ed.) - Strogatz (2018). \emph{Nonlinear Dynamics and Chaos} (2nd ed.) - Conley (1978). \emph{Isolated Invariant Sets and the Morse Index}

\subsubsection{1.3 Bayesian Inference / Martingale Theory}\label{bayesian-inference-martingale-theory}

\textbf{Root}: Doob, J. L. (1953). \emph{Stochastic Processes}. Wiley.

\textbf{Role in Spring}: The evolution of the gatekeeper scoring function \(S_t\) is interpreted as a Bayesian posterior mean update:

\[S_t(x,y) = \mathbb{E}_{S \sim P_t}[S(x,y)], \quad P_{t+1}(S) \propto P_t(S) \cdot \mathcal{L}(\mathcal{D}_{t+1} \mid S)\]

Theorem (Doob Martingale Convergence): The sequence \(\{S_t\}_{t \geq 0}\) is a \([0,1]\)-bounded martingale with respect to the filtration \(\mathcal{F}_t = \sigma(\mathcal{M}_1, \dots, \mathcal{M}_t)\). By Doob's martingale convergence theorem:

\[S_t(x,y) \xrightarrow{a.s.} S_\infty(x,y), \quad \forall (x,y)\]

Important limitation: the martingale property requires correct model specification (\(S^*\) in the support of prior \(P_0\)). Under model misspecification, the posterior converges to the KL projection \(S^\dagger\) rather than the true value \(S^*\). Spring honestly notes this limitation.

\textbf{References}: - Doob (1953). \emph{Stochastic Processes} - Doob (1949). ``Application of the theory of martingales'' - Ghosh \& Ramamoorthi (2003). \emph{Bayesian Nonparametrics} - van der Vaart (1998). \emph{Asymptotic Statistics} - Kleijn \& van der Vaart (2012). ``The Bernstein-von Mises theorem under misspecification''

\subsubsection{1.4 Online Learning / Regret Analysis}\label{online-learning-regret-analysis}

\textbf{Root}: Zinkevich, M. (2003). ``Online convex programming and generalized infinitesimal gradient ascent.'' \emph{ICML 2003}.

\textbf{Role in Spring}: The gatekeeper's online gradient descent (OGD) update produces standard regret bounds:

\[\text{Regret}_T \leq 2GR\sqrt{T} \quad \text{(no delay)}\]

Spring generalizes this bound to the \textbf{delayed feedback} setting (NEP student feedback delay \(d_t \leq D_{\max}\)):

\[\text{Regret}_T^{\text{delay}} \leq 2GR\sqrt{T} + G^2 D_{\max} \sqrt{T}\]

Further generalization to the \textbf{growing memory bank} setting — the monotonic growth of memory bank \(M_t\) provides effective sample complexity improvement:

\[\text{Regret}_T^{\text{eff}} \leq \text{Regret}_T^{\text{delay}} + O\left(T \sqrt{\frac{\text{VC}(\mathcal{H})}{N_T}}\right)\]

As \(N_T \to \infty\), the system satisfies \textbf{asymptotic no-regret}: \(\frac{1}{T}\text{Regret}_T^{\text{eff}} \to 0\).

\textbf{References}: - Zinkevich (2003). ``Online convex programming'' - Hazan (2016). ``Introduction to online convex optimization'' - Joulani et al.~(2016). ``Online learning under delayed feedback'' - Shalev-Shwartz (2012). ``Online learning and online convex optimization'' - Cesa-Bianchi \& Lugosi (2006). \emph{Prediction, Learning, and Games}

\subsubsection{1.5 Concentration Inequalities}\label{concentration-inequalities}

\textbf{Root}: - Hoeffding, W. (1963). ``Probability inequalities for sums of bounded random variables.'' \emph{JASA}, 58(301), 13-30. - Chernoff, H. (1952). ``A measure of asymptotic efficiency for tests of a hypothesis.'' \emph{Annals of Mathematical Statistics}, 23(4), 493-507.

\textbf{Role in Spring}: At multiple critical points in Spring:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
 \textbf{Concentration of consensus score}: Hoeffding tail bound for expert consensus \(\hat{C}(x) = \frac{1}{M}\sum_{m=1}^M \mathbf{1}\{\ell(f_m(x), y) > \tau\}\): \[\mathbb{P}(|C - \mathbb{E}[C]| > t) \leq 2\exp(-2M t^2)\] When A2 is violated due to expert correlation, \(M\) is replaced by effective expert count \(M_{\text{eff}} = M/(1 + (M-1)\bar{\rho})\).
\item
 \textbf{Chernoff Information}: The optimal exponential rate of the detection problem is given by Chernoff information \(\kappa = \text{KL}(\theta^*\|p_0) = \text{KL}(\theta^*\|p_1)\), applied in Theorem 4'.
\item
 \textbf{Bahadur-Rao Exact Asymptotics}: The Bernoulli lattice correction factor \((1-e^{-\lambda^*})^{-1}\) replaces the standard Chernoff prefactor, tightening Theorem 4''s exact constant by 5--12\%.
\end{enumerate}

\textbf{References}: - Hoeffding (1963) - Chernoff (1952) - Bahadur \& Rao (1960). ``On deviations of the sample mean'' - Dembo \& Zeitouni (2010). \emph{Large Deviations Techniques and Applications}

\subsubsection{1.6 Information Theory}\label{information-theory}

\textbf{Root}: - Fano, R. M. (1961). \emph{Transmission of Information: A Statistical Theory of Communications}. MIT Press. - Cover, T. M. \& Thomas, J. A. (2006). \emph{Elements of Information Theory} (2nd ed.). Wiley.

\textbf{Role in Spring}:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
 \textbf{Fano's Inequality}: Used in Theorem 2 (Weak Feature Failure) lower bound — when the mutual information between features \(\phi\) and true state \(S\) satisfies \(I(\phi; S) \leq \delta\), any state estimation algorithm has: \[P(\hat{S} \neq S) \geq \frac{H(S) - \delta - \log 2}{\log K}\]
\item
 \textbf{Data Processing Inequality (DPI)}: Used in Theorem 2 proof Step 3 — the Markov chain \(\phi(X) \to \hat{S} \to \hat{z}\) implies TV does not increase.
\item
 \textbf{Pinsker's Inequality}: Converts mutual information bound \(\delta = I(\phi; S)\) to total variation bound \(\text{TV} \leq \sqrt{\delta/2}\).
\item
 \textbf{KL Contraction}: Under correct model specification, posterior convergence \(D_{\text{KL}}(P^* \| P_t) \xrightarrow{a.s.} 0\).
\end{enumerate}

\textbf{References}: - Fano (1961) - Cover \& Thomas (2006) - Pinsker (1964). \emph{Information and Information Stability of Random Variables and Processes}

\subsubsection{1.7 Minimax Theory}\label{minimax-theory}

\textbf{Root}: Le Cam, L. (1973). ``Convergence of estimates under dimensionality restrictions.'' \emph{Annals of Statistics}, 1(1), 38-53.

\textbf{Role in Spring}:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
 \textbf{Le Cam Two-Point Method}: Constructs two difficult-to-distinguish distributions \(P_0 = \text{Bern}(\mu_s)^{\otimes M}\) (clean) and \(P_1 = \text{Bern}(1 - C_{\text{bal}}\mu_s/(K-1))^{\otimes M}\) (noise), using Hellinger tensorization to obtain minimax lower bounds.
\item
 \textbf{Neyman-Pearson Reduction}: Via the Neyman-Pearson lemma, reduces any algorithm to a Bayes test, giving Theorem 4''s optimality.
\item
 \textbf{Hellinger Exact Tensorization}: Under A2 (conditional independence), \(H^2(P_0^{\otimes M}, P_1^{\otimes M}) = 1 - (1 - H^2(P_0, P_1))^M\) — exact equality, no inequality relaxation.
\end{enumerate}

\textbf{References}: - Le Cam (1973) - Le Cam (1986). \emph{Asymptotic Methods in Statistical Decision Theory} - Tsybakov (2009). \emph{Introduction to Nonparametric Estimation}

\subsection{2. Key Ideas and Proof Tools}\label{key-ideas-and-proof-tools}

\subsubsection{\texorpdfstring{2.1 Two-Timescale Separation (\(\beta_t = o(\alpha_t)\))}{2.1 Two-Timescale Separation}}\label{two-timescale-separation}

Spring's most critical architectural innovation. Condition C6' requires the gatekeeper update rate \(\beta_t\) to be a lower-order infinitesimal of the student update rate \(\alpha_t\):

\[\beta_t = o(\alpha_t), \quad \text{i.e.,} \quad \lim_{t\to\infty} \frac{\beta_t}{\alpha_t} = 0\]

\textbf{Purpose}: Between any two gatekeeper updates, the student executes asymptotically infinitely many gradient steps, converging to the local minimum of the current distribution before distribution shift.

\textbf{Canonical schedule}: \(\alpha_t = t^{-a}\), \(\beta_t = t^{-b}\), where \(\frac{1}{2} < a < 1 < b\). For example \(a = 0.6\), \(b = 1.2\): - \(\sum \alpha_t = \infty\) (student exploration), \(\sum \alpha_t^2 < \infty\) (noise control) - \(\sum \beta_t < \infty\) (finite total distribution shift), \(\sum \beta_t^2 < \infty\) (variance control) - \(\beta_t / \alpha_t = t^{-0.6} \to 0\) (timescale separation)

\subsubsection{2.2 Reference Set Replay Breaks Selection Bias}\label{reference-set-replay-breaks-selection-bias}

Spring's deepest proof tool. Theorem 12.2 proves: \textbf{Without reference set replay, Lyapunov descent is asymptotically impossible} — the gatekeeper can reduce its apparent loss by only accepting ``easy'' samples without truly improving discriminative ability.

Solution (Theorem 12.5, full proof): 1. \textbf{Student side}: Importance sampling weights \(w(x) = P_0(x) / P_{S_t}(x)\), making the effective training distribution the reference distribution \(P_0\) 2. \textbf{Gatekeeper side}: Compute SCXUpdate on the fixed reference set \(M_0\), eliminating alignment bias 3. \textbf{Cost}: Importance weight variance inflation factor \(W \leq 1/\varepsilon_{\text{explore}} < \infty\)

\subsubsection{2.3 Lipschitz-Almost-Everywhere Lemma}\label{lipschitz-almost-everywhere-lemma}

Resolves a fundamental mathematical obstacle (Lemma SE-1.0, B9 fix):

\textbf{Problem}: The consensus score \(C(x) = \frac{1}{M}\sum_{m=1}^M \mathbf{1}\{\ell(f_m(x), y) > \tau\}\) is discontinuous at threshold \(\tau\) (indicator function jump).

\textbf{Solution}: Under condition C5 (conditional i.i.d. sampling), the discontinuity set has probability measure zero because: \[\mathbb{P}(\exists m: \ell(f_m(X), Y) = \tau) \leq \sum_{m=1}^M \mathbb{P}(\ell(f_m(X), Y) = \tau) = 0\] for smooth functions \(\ell(f_m(x), y)\) over continuous distributions. Expectations are unaffected by zero-measure set modifications; Lyapunov analysis remains valid.

\subsubsection{2.4 Bayesian Posterior Update as Gatekeeper Evolution}\label{bayesian-posterior-update-as-gatekeeper-evolution}

The gatekeeper's evolution is interpreted as sequential Bayesian updating:

\[P_{t+1}(S) \propto P_t(S) \cdot \prod_{(x,y) \in \mathcal{D}_{t+1}} S(x,y)^{a} (1 - S(x,y))^{1-a}\]

This yields two powerful theoretical properties: 1. \textbf{Martingale property}: \(\mathbb{E}[S_{t+1} \mid \mathcal{F}_t] = S_t\) (tower property), a.s. convergence guaranteed by Doob's theorem 2. \textbf{KL contraction}: Under correct model specification, \(D_{\text{KL}}(P^* \| P_t) \to 0\)

\subsubsection{2.5 Lyapunov Descent with Importance Sampling}\label{lyapunov-descent-with-importance-sampling}

Theorem 12.5's complete proof structure proves \(\mathbb{E}[\Delta\Psi_t \mid \mathcal{F}_t] \leq 0\):

\[\mathbb{E}[\Delta\Psi_t \mid \mathcal{F}_t] \leq -\alpha_t \|\nabla L_0(\theta_t)\|^2 - \beta_t \cdot 2\rho_{\text{ideal}} \cdot \|\Delta_t^{\text{ideal}}\|_{M_0} \cdot \|S_t - \hat{C}\|_{M_0} + O(\alpha_t^2 W + \beta_t^2)\]

Four-term decomposition: - \textbf{(A) Student improvement}: \(\Delta_{\text{student}}\), eliminates \(D_{\text{static}}\) gap via importance sampling (\textbf{proved}) - \textbf{(B) Gatekeeper improvement}: \(\Delta_{\text{gatekeeper}}\), eliminates alignment bias by computing SCXUpdate on \(M_0\) (\textbf{proved}) - \textbf{(C) Distribution shift}: \(\Delta_{\text{selection}} = O(\beta_t) = o(\alpha_t)\), controlled by two-timescale (\textbf{proved}) - \textbf{(D) Cross-coupling}: \(\Delta_{\text{cross}} = 0\), due to reference set separability (\textbf{proved})

\subsubsection{\texorpdfstring{2.6 \(O(t^{-a})\) Convergence Rate and Polyak Averaging}{2.6 Convergence Rate and Polyak Averaging}}\label{convergence-rate-and-polyak-averaging}

Under strong convexity, the student's convergence rate is \(O(t^{-a})\), where \(a \in (0.5, 1]\) is the learning rate exponent.

\textbf{Polyak-Ruppert averaging} (\(\bar{\theta}_t = \frac{1}{t}\sum_{i=1}^t \theta_i\)) boosts the rate to \(O(t^{-1})\) — the optimal rate matching the minimax lower bound \(\Omega(t^{-1})\) — applicable for any \(a \in (0.5, 1)\) without precise tuning.

\subsection{5. Spring's Unique Contributions}\label{springs-unique-contributions}

The following are things Spring accomplishes that \textbf{no prior work simultaneously accomplishes}:

\subsubsection{5.1 Self-Evolving Evaluation Criterion (Not Just Self-Improving Strategy)}\label{self-evolving-evaluation-criterion}

\textbf{Prior limitation}: AlphaZero self-improves its strategy (how to play), but the evaluation criterion (game rules) is a fixed external truth. Active learning and Bayesian optimization have fixed query strategies/acquisition functions.

\textbf{Spring's innovation}: The gatekeeper \(S_t\) \textbf{itself} is the learning object — it defines ``what constitutes good data.'' The system does not optimize strategy under a fixed evaluation criterion but \textbf{simultaneously evolves the evaluation criterion itself}. This is a second-order learning problem: learning how to learn to judge data quality.

\subsubsection{5.2 Monotonic Growing Memory (No Replacement Replay)}\label{monotonic-growing-memory}

\textbf{Prior limitation}: All replay buffer schemes (AlphaZero, DQN, experience replay) replace old memories to some degree — buffers are bounded, oldest/least useful samples are discarded.

\textbf{Spring's innovation}: \(M_t \subseteq M_{t+1}\) — memory only grows. The no-deletion policy means: 1. No catastrophic forgetting — early-learned structures are never discarded 2. Memory monotonicity provides a free structural property (needed by Lemma SE-1.2) 3. Resurrection mechanism becomes possible — sub-threshold old structures remain in the memory bank, waiting for the gatekeeper to mature and re-score them

\subsubsection{5.3 Resurrection Mechanism}\label{resurrection-mechanism}

\textbf{Prior limitation}: No prior algorithm includes the concept of ``resurrection.'' Rejected samples are either discarded (active learning) or not revisited.

\textbf{Spring's innovation}: ``Winter does not kill — it waits for spring.'' Structures below threshold \(\tau_{\text{keep}}\) are not deleted — they enter dormancy. When the gatekeeper evolves in later iterations (\(S_{t+1}\) may give higher scores to the same structures), dormant structures are resurrected and introduced into training:

\begin{Shaded}
\begin{Highlighting}[]

\NormalTok{ new\_score }\OperatorTok{=}\NormalTok{ S\_\{t\}}\OperatorTok{+}\DecValTok{1}\NormalTok{\}(x, y)}
 \ControlFlowTok{if}\NormalTok{ new\_score }\OperatorTok{\textgreater{}=}\NormalTok{ τ\_keep:}
\NormalTok{ RESURRECT(x, y) }\CommentTok{\# Spring resurrects it}
\end{Highlighting}
\end{Shaded}

This is mathematically encoded as: the gatekeeper update can push a sample's score past the threshold at any time, turning rejections into acceptances — impossible in active learning (permanently discarded once rejected) or Bayesian optimization (budget consumed once evaluated).

\subsubsection{5.4 Coupled Dynamical System Analysis (Gatekeeper + Student Co-Evolution)}\label{coupled-dynamical-system-analysis}

\textbf{Prior limitation}: Robbins-Monro analyzes a single recursion. AlphaZero's policy iteration alternates fixing one component to update the other (no coupled dynamical system analysis between policy evaluation and improvement).

\textbf{Spring's innovation}: Complete coupled dynamical system formalization: - State space \(\mathcal{Z} = \mathcal{F} \times \mathcal{M} \times \Theta\) (product space) - Update operator \(\Phi = (\Phi_S, \Phi_M, \Phi_\theta)\) (causal feedforward structure, block-triangular Jacobian) - Two-timescale ODE analysis (fast-slow system of equations) - Lyapunov function explicit descent on the product space - \(\omega\)-limit sets, attractor basins, phase diagrams

No prior algorithm analyzes the co-evolution of two learning components in a formal coupled dynamical system framework.

\subsubsection{5.5 Exact Convergence Rates with Polyak Averaging}\label{exact-convergence-rates-with-polyak-averaging}

\textbf{Prior limitation}: Bayesian optimization gives cumulative regret bounds \(O(\sqrt{T \cdot \gamma_T})\) but does not give parameter convergence rates. Active learning gives label complexity but not learning rates.

\textbf{Spring's innovation}: - Student rate \(O(t^{-a})\) under strong convexity (Theorem 11.1, proved) - Gatekeeper rate \(O(t^{-b})\) under contraction (Theorem 11.2, proved) - Polyak-Ruppert averaging boosts rate to \(O(t^{-1})\) — matching minimax lower bound \(\Omega(t^{-1})\) - Coupled rate dominated by student bottleneck \(O(t^{-a})\) (Theorem 11.3) - Finite-time (non-asymptotic) bounds with explicit constants (Theorem 11.6, proved)

\subsubsection{5.6 Formalized Selection Bias Loop Analysis}\label{formalized-selection-bias-loop-analysis}

\textbf{Prior limitation}: No prior algorithm formally analyzes the mechanism by which its own data selection procedure creates a self-reinforcing bias loop.

\textbf{Spring's innovation}: Theorem 12.2 proves — \textbf{Without reference set replay, Lyapunov descent is asymptotically impossible} — the gatekeeper can reduce its apparent loss by only accepting ``easy'' samples without truly improving. This is a profound limitation on any self-selecting learning system, not formally proven before Spring.

Specifically, the selection bias loop:

\[\text{Gatekeeper } S_t \xrightarrow{\text{selects}} \text{Memory } M_{t+1} \xrightarrow{\text{trains}} \text{NEP } f_{\theta_{t+1}} \xrightarrow{\text{feeds back}} \text{Gatekeeper } S_{t+1}\]

is decomposed, quantified, and proven to be a necessary obstruction to Lyapunov descent — then resolved through the joint mechanism of reference set replay + importance sampling (Theorem 12.5, full proof).

\subsection{Summary}\label{summary}

The Spring self-evolution algorithm operates at the intersection of seven classical mathematical domains: stochastic approximation, dynamical systems, Bayesian inference, online learning, concentration inequalities, information theory, and minimax theory. Its core architectural innovations — two-timescale separation, reference set replay breaking selection bias, monotonic growing memory, resurrection mechanism, coupled dynamical system analysis — are supported by complete proof chains from these domains. The decisive distinction from all prior work is: Spring \textbf{simultaneously} self-evolves both its evaluation criterion and the model being evaluated, formally proves convergence, and maintains monotonic growing memory (never forgetting).
