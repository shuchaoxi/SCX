\subsubsection{2026-06-29: State Crystallization --- The Third Core Algorithm}
\textbf{Naming:} State Crystallization is to SCX what BPE (Byte Pair Encoding) is to tokenization. State Crystallization encodes continuous expert states into discrete ``Status'' tokens.\par
\textbf{Formal definition:} PBE compression maps raw features through clustering to produce a Status label. Unlike BPE, which maps byte sequences to tokens and \textbf{loses information}, State Crystallization maps features to Status labels and \textbf{preserves information} (equality holds).\par
\textbf{SCX Architecture:} State Crystallization is the \textbf{third Core} component alongside Spring (Status gating) and Yajie (Status auditing).\par
\textbf{Documentation:} See \texttt{scx\_method/methods.tex} (Stage 1 ``State Crystallization'') and \texttt{scx\_llm/main.tex} (\S 2.1 State Crystallization vs BPE).

\subsubsection{2026-06-29: State Crystallization vs BPE --- Formal Comparison}
\textbf{Core Conclusion: BPE \(\subseteq\) State Crystallization} (BPE is a special case of State Crystallization). BPE maps byte sequences to token IDs without preserving semantics. State Crystallization maps features to Status labels with preserved semantics. Example: ``sp\(^3\) + C-C bond'' and ``sp\(^3\) + C-N bond'' are different BPE tokens but may share the same Status.\par
\textbf{Honest Strike:} Crystallization is not BPE with a Sam Altman rebrand.
