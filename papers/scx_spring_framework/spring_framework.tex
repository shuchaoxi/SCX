\begin{abstract}
本文提出\Spring{}统一多模态大模型框架——第一个将训练过程本身作为跨模态审计过程的统一智能架构。
\Spring{}不是简单的势函数训练器或单一模态的守门人，而是一个将任意数据模态的
注入、物理建模、自然语言推理和内生审计统一在单一循环中的完整大模型架构。
其核心创新为五层统一：(1) 输入层接受任意模态——DFT计算、分子动力学轨迹、
文本、图像、音频、视频、表格数据——通过统一编码器映射到共享表示空间；
(2) 物理层使用\Spring{}迭代势函数——每一个势函数自身即经过审计，
$M_t$由数据自动生成；(3) 推理层以大语言模型为核心，通过RAG机制检索\Spring{}物理层的输出，
将自然语言查询转化为可验证的物理计算；(4) 审计层嵌入\Yajie{}多专家共识和$M_t$验证机制，
对推理层的每一个输出进行内生审计；(5) 输出层返回答案、\Cercis{}置信度评分和完整审计轨迹。

本文的核心主张是：\Spring{}是\textbf{第一个训练过程即为跨模态审计过程的大模型}。
用户说一句“这个材料有什么化学性质”，\Spring{}将其解析为自然语言查询→\Spring{}势函数计算→
\Yajie{}共识验证→\Cercis{}评分输出的完整链路——一句话知道材料特性，且每一步都可追溯。
现有大模型（GPT、Claude、Gemini）在审计维度上都是$M=1$、UNDECLARED，
而\Spring{}框架是born audited——从诞生之初就内建了审计。

\textbf{关键词：} 统一多模态大模型；Spring框架；训练即审计；势函数学习；Yajie共识；Cercis评分；
跨模态审计；物理层计算；审计内建架构；全模态智能
\end{abstract}

\section{引言：大模型的审计真空}

\subsection{从语言模型到多模态模型：审计的结构性缺失}

过去五年，大语言模型（Large Language Models, LLMs）经历了从GPT-2到GPT-4、Claude、Gemini的
跨越式发展。模型能力从文本扩展到图像、音频、视频，形成了所谓的“多模态大模型”。然而，这
一发展轨迹中存在一个\textbf{结构性的审计真空}：模型训练过程不产生审计信息，模型输出不可
追溯，训练数据质量未经系统验证。

这一审计真空在三个层面显现：

\begin{enumerate}[label=(\arabic*),itemsep=4pt]
    \item \textbf{训练层面的不可审计性：} 现有大模型的训练过程产生一个权重文件，但没有产生
    关于训练过程本身的审计信息。我们不知道哪些训练数据对最终模型的影响最大，不知道模型
    在哪些输入上可能产生幻觉，不知道模型的置信度是否有统计基础。
    
    \item \textbf{推理层面的不可追溯性：} 当GPT-4回答“这个材料的化学性质是什么”时，用户
    收到一个自然语言答案。但这个答案的来源是什么？是基于训练数据的记忆？是真实推理？
    还是纯粹的统计幻觉？用户无从知晓。
    
    \item \textbf{跨模态层面的不可验证性：} 多模态模型可以在图像描述和文本之间建立关联，
    但这些关联的物理真实性不可验证。模型声称“这张电子显微镜图像显示了位错结构”——
    这个声称是来自对真实物理的理解，还是来自对图文配对的统计拟合？
\end{enumerate}

现有的应对方法是事后审计——事实核查、红队测试、基准评估。但这些方法的共同局限是：
它们\textbf{外在于}模型的训练和推理过程。就像在建筑完成后检查裂缝，而非在浇筑时检测
混凝土的质量——事后审计无法修复训练过程中根植的结构性问题。

\subsection{SCX审计理论的三大基石}

\SCX{}理论体系~\\cite{scx_ml_audit,scx_hamiltonian,yajie_protocol}通过三个核心定理揭示了
单模型（$M=1$）在审计维度的根本不可能性：

\begin{enumerate}[label=\textbf{基石 \arabic*},leftmargin=*,itemsep=4pt]
    \item \textbf{自审计禁止定理（SCX定理2）：}
    单个模型（$M=1$）无法对自己的预测进行审计——模型判断“自己何时错误”与“产生预测”
    在信息论上是同一个过程。任何声称“自审计”的$M=1$系统等价于无审计。
    
    \item \textbf{多专家检测定理（SCX定理1）：}
    $M>1$个独立专家提供误差检测能力的指数增长——
    $M$个专家同时遗漏同一错误的概率随$M$指数衰减。
    
    \item \textbf{噪声-信号不可区分定理（SCX定理3）：}
    在不引入额外结构假设的情况下，区分数据噪声和样本固有难度在数学上是不可能的——
    审计必须根植于训练的结构中，而不能事后追加。
\end{enumerate}

这三大定理共同指向一个结论：\textbf{真正的审计必须内建于模型架构之中}。
训练过程必须同时是审计过程，模型必须从诞生之初就携带审计信息。
这就是\Spring{}统一多模态大模型框架的哲学起点。

\subsection{Spring的定位：不再是工具，而是架构}

\Spring{}在此前的SCX工作中被定位为一个势函数训练器——一个专注于分子模拟领域的工具~\\cite{scx_spring_trainer}。
在那项工作中，我们论证了Spring如何通过多专家训练、\Yajie{}共识和$M_t$自动生成实现了势函数学习的
“训练即审计”。

但\Spring{}的深层潜能远不止于此。
\textbf{\Spring{}本身就是一个完整的大模型架构}——不是GPT的竞争对手，
而是对“什么是大模型”这一问题的重新定义。
在这个定义下，大模型不仅包括语言理解和生成，还包括：
\begin{itemize}[nosep]
    \item \textbf{物理建模：} 基于DFT计算和分子动力学的势函数学习
    \item \textbf{科学计算：} 第一性原理计算的自动化执行和结果融合
    \item \textbf{多模态感知：} 文本、图像、音频、视频、表格数据的统一理解
    \item \textbf{内生审计：} 每一个输出都携带共识信息、置信度评分和追溯路径
\end{itemize}

本文的核心革新在于：\textbf{\Spring{}统一多模态大模型框架将上述所有能力统一在一个
“输入→物理→推理→审计→输出”的架构循环中}。在这个循环中，物理层不是外挂
的工具调用，而是模型推理管道的内生组件；审计层不是事后的评估，而是嵌入在每一个
推理步骤中的实时验证。

\begin{definition}[Spring统一多模态大模型 — 完整定义]\label{def:spring_framework}
\Spring{}统一多模态大模型是一个五层架构的自治系统：
\begin{equation}\label{eq:spring_pipeline}
    \boxed{
    \text{Input} \xrightarrow{\Enc} \text{Physics} \xrightarrow{\Spring} \text{Reasoning} \xrightarrow{\Yajie} \text{Audit} \xrightarrow{\Cercis} \text{Output}
    }
\end{equation}
其中每一层的形式定义和交互协议将在后续各节展开。
\Spring{}的特征不是它的任何一个组件——每个组件在文献中都有其前身——
而是\textbf{这些组件在单一架构中的内生统一}。
\end{definition}

\subsection{术语约定}

为清晰起见，本文使用以下术语约定：

\begin{itemize}[nosep]
    \item \textbf{模态（Modality）：} 数据的基本感知类型——文本、图像、音频、视频、DFT数据、
    分子动力学轨迹、表格数据、图结构数据等。形式化地，模态集合记为$\ModalitySet = \{m_1, m_2, \ldots, m_K\}$。
    \item \textbf{物理层（Physics Layer）：} \Spring{}框架中负责科学计算的部分——
    \Spring{}迭代势函数、DFT函数、分子动力学模拟器。
    \item \textbf{推理层（Reasoning Layer）：} 以大语言模型为核心的自然语言理解和生成部分——
    同时具备RAG能力以检索物理层的输出。
    \item \textbf{审计层（Audit Layer）：} \Yajie{}多专家共识、$M_t$验证和\Cercis{}评分的内生审计机制。
    \item \textbf{大模型（Large Model）：} 在整个输入模态空间上操作的统一智能系统——
    不限于语言模型，不限于单模态。
\end{itemize}

\subsection{五层的内生统一原则}

五层管道不是简单的串联，而是遵循以下内生统一原则：

\begin{enumerate}[label=\textbf{原则 \arabic*},leftmargin=*,itemsep=5pt]
    \item \textbf{训练即审计（Training-Is-Audit）：}
    物理层的\Spring{}势函数训练过程本身即是审计过程——第$t$次训练迭代同时产生
    势函数$V^{(t)}_{\text{consensus}}$、\Yajie{}共识评分$\mathbf{s}^{(t)}_{\Yajie}$、
    Lyapunov监控量$\Lyap_t$、自生成$M_t$和\Cercis{}评分$S^{(t)}_{\Cercis}$——
    这五项产出是训练的内生产物，而非事后计算。
    
    \item \textbf{查询即计算（Query-Is-Computation）：}
    用户的自然语言查询不是简单的语义匹配或检索，而是被解析为对物理层的计算请求——
    “这个材料有什么化学性质”被编译为\Spring{}势函数的查询序列，
    其答案来自物理计算而非语言模型的统计外推。
    
    \item \textbf{输出即审计（Output-Is-Audit）：}
    每一个输出都携带完整的审计元数据——哪些专家达成了共识、$M_t$的值、
    \Cercis{}置信度评分。输出与审计不可分离。
    
    \item \textbf{跨模态可验证（Cross-Modality Verifiability）：}
    不同模态之间的声称可以相互验证——文本声称可以通过DFT训练势函数验证，
    图像分类可以通过多专家视觉模型验证。
    
    \item \textbf{数据-参数绑定（Data-Parameter Binding）：}
    $M_t$和$D_{\text{hash}}$之间的密码学绑定确保训练过程和数据的不可篡改性。
\end{enumerate}

\section{输入层：任意模态的统一编码}

\subsection{统一编码器的形式化}

\Spring{}输入层的核心组件是\textbf{统一多模态编码器}$\Enc_\theta: \ModalitySet \to \R^d$，
它将任意模态的数据映射到一个共享的$d$维表示空间。形式化地：

\begin{definition}[统一多模态编码器]\label{def:encoder}
设模态空间为$\ModalitySet = \{\text{DFT}, \text{MD}, \text{Text}, \text{Image}, \text{Audio}, \text{Video}, \text{Tabular}\}$。
统一编码器由模态特定的编码器族和一个共享投影层组成：
\begin{equation}\label{eq:encoder_def}
    \Enc_\theta(x, m) = W_{\text{proj}} \cdot \phi_m(x; \theta_m)
\end{equation}
其中：
\begin{itemize}[nosep]
    \item $\phi_m(\cdot; \theta_m): \X_m \to \R^{d_m}$为模态$m$的特征提取器（$\theta_m$为模态特定参数），
    \item $W_{\text{proj}} \in \R^{d \times d_m}$为共享投影矩阵，将所有模态映射到统一维度$d$，
    \item $\theta = \{\theta_m\}_{m \in \ModalitySet} \cup \{W_{\text{proj}}\}$。
\end{itemize}
\end{definition}

\subsection{各模态的编码策略}

不同模态采用差异化的前端处理，但共享统一的后端表示。我们仅详细描述物理模态的编码——
因为它们在大模型文献中的覆盖度最低，而文本/图像/音频/视频模态已有成熟方案（CLIP、
Whisper、ViT等），我们简要提及这些成熟组件的集成方案。

\begin{enumerate}[label=\textbf{模态 \arabic*},leftmargin=*,itemsep=4pt]
    \item \textbf{DFT计算数据：}
    DFT计算的输出——原子坐标$\{\mathbf{R}_i\}$、总能量$E_{\text{tot}}$、
    力$\{\mathbf{F}_i\}$、应力张量$\sigma$、电子态密度DOS$(E)$——通过以下管道编码：
    \begin{itemize}[nosep]
        \item 原子坐标和物种通过ACE描述符~\\cite{ace}（原子簇展开）转化为旋转不变的特征向量；
        \item 标量量（能量、应力分量）通过可学习的嵌入层映射；
        \item 分布量（态密度）通过核密度估计离散化后编码。
    \end{itemize}
    形式化地：
    \begin{equation}\label{eq:dft_encoder}
        \phi_{\text{DFT}}(\{\mathbf{R}_i, E, \mathbf{F}, \sigma, \text{DOS}\}) = 
        \text{ACE}(\{\mathbf{R}_i\}) \oplus \text{MLP}(E, \sigma) \oplus \text{KDE-Embed}(\text{DOS})
    \end{equation}
    其中$\oplus$表示向量拼接。

    \item \textbf{分子动力学轨迹：}
    MD轨迹$[\mathbf{R}(t_1), \ldots, \mathbf{R}(t_T)]$通过时空编码：
    \begin{equation}\label{eq:md_encoder}
        \phi_{\text{MD}}(\mathbf{R}(t_1:T)) = \text{Transformer}_{\text{temp}}(\text{ACE}(\mathbf{R}(t_1)), \ldots, \text{ACE}(\mathbf{R}(t_T)))
    \end{equation}
    其中$\text{Transformer}_{\text{temp}}$为时域Transformer，捕捉构型的时序演化。

    \item \textbf{文本：}
    \begin{equation}\label{eq:text_encoder}
        \phi_{\text{Text}}(x) = \text{LLM-embed}(x)
    \end{equation}
    使用预训练大语言模型的嵌入层（如BGE、E5等），通过适配器投影到统一维度。

    \item \textbf{图像/视频：}
    \begin{equation}\label{eq:vision_encoder}
        \phi_{\text{Image}}(x) = \text{ViT}(x), \quad 
        \phi_{\text{Video}}(x) = \text{TimeSFormer}(x)
    \end{equation}
    使用视觉Transformer和时间-空间Transformer。

    \item \textbf{音频：}
    \begin{equation}\label{eq:audio_encoder}
        \phi_{\text{Audio}}(x) = \text{Whisper-enc}(x)
    \end{equation}
    使用Whisper编码器或等价的音频基础模型。

    \item \textbf{表格数据：}
    \begin{equation}\label{eq:tabular_encoder}
        \phi_{\text{Tabular}}(x) = \text{FT-Transformer}(x)
    \end{equation}
    使用特征-令牌Transformer（Feature Tokenizer Transformer）。

    \item \textbf{图结构数据（分子图、晶体图等）：}
    \begin{equation}\label{eq:graph_encoder}
        \phi_{\text{Graph}}(G) = \text{GNN}(G)
    \end{equation}
    使用等变图神经网络（如E(3)-等变GNN）。
\end{enumerate}

\subsection{表示空间的对齐与校准}

不同模态的编码器输出需要位于相同的语义空间中。我们采用对比学习实现跨模态对齐：

\begin{definition}[跨模态对比损失]\label{def:contrastive}
设$(x_m, x_{m'})$为模态$m$和模态$m'$的配对样本（例如，DFT计算结果与对应的文本描述）。
跨模态对比损失定义为：
\begin{equation}\label{eq:contrastive_loss}
    \loss_{\text{align}} = -\frac{1}{B}\sum_{i=1}^{B} \log\frac{
        \exp(\Enc_\theta(x_i^m, m)^\top \Enc_\theta(x_i^{m'}, m') / \tau)
    }{
        \sum_{j=1}^{B} \exp(\Enc_\theta(x_i^m, m)^\top \Enc_\theta(x_j^{m'}, m') / \tau)
    }
\end{equation}
其中$B$为批量大小，$\tau$为温度参数。
\end{definition}

关键的是：对于物理模态（DFT、MD），配对样本的“文本描述”由物理层的\Spring{}势函数
自动生成——势函数训练完成后，其输出通过推理层转化为自然语言描述。
这意味着物理模态的跨模态对齐\textit{不}依赖人工标注，而是\Spring{}框架的自动化产物。

\subsection{输入层到物理层的路由}

并非所有模态都需要经过物理层。文本查询可能仅需推理层处理，而DFT数据仅需物理层计算。
输入层通过一个轻量的\textbf{模态路由器}决定数据的下游路径：

\begin{equation}\label{eq:router}
    \text{Route}(x, m) = \begin{cases}
        \text{Physics} \to \text{Reasoning} \to \text{Audit} & m \in \{\text{DFT}, \text{MD}, \text{Tabular\_scientific}\} \\
        \text{Reasoning} \to \text{Audit} & m \in \{\text{Text}, \text{Image}, \text{Audio}, \text{Video}\}
    \end{cases}
\end{equation}

即使对于非物理模态，物理层仍作为\textbf{背景知识库}参与——推理层可以主动查询物理层
以获取科学事实的验证（详见第7节：跨模态审计）。

\section{物理层：Spring迭代势函数作为物理引擎}

\subsection{物理层在大模型中的角色}

现有大模型（GPT-4、Claude、Gemini）的物理和科学知识完全来自训练语料中的文本描述——
模型“知道”量子力学是因为它读过量子力学教材，而非因为它能够求解薛定谔方程。
这种“知识”的根本局限是：\textbf{它不能产生新的物理知识，只能复述已有的文本描述}。

\Spring{}框架中的物理层与此根本不同：\textbf{它是一个真正能够执行物理计算的引擎}。
当\Spring{}回答一个材料科学问题时，它的答案不是来自语料的统计外推，
而是来自\Spring{}迭代势函数对具体原子构型的实际计算。
这种差异是质的差异，而非量的差异——就像一个有物理实验室的科学家和
一个只读过物理教科书的读者之间的差异。

\subsection{\Spring{}势函数训练循环：回顾与扩展}

物理层的核心是\Spring{}自演化势函数训练器~\\cite{scx_spring_trainer}。
我们在此回顾其关键结构，并扩展至多模态语境。

\begin{definition}[\Spring{}势函数训练循环 — 多模态扩展]\label{def:physics_spring}
对于训练数据$\D = \{(\mathbf{R}_i, E_i, \mathbf{F}_i)\}_{i=1}^{N}$（来自DFT计算），
\Spring{}的第$t$次训练迭代执行以下循环：
\begin{enumerate}[label=步骤 \arabic*,leftmargin=*,itemsep=2pt]
    \item \textbf{多专家自举：} 从$\D$中生成$M_t$个统计独立的训练子集$\{\D_m\}_{m=1}^{M_t}$。
    \item \textbf{独立训练：} 在$\D_m$上独立训练专家$f_m(\cdot; \theta_m)$——
    无参数共享，无梯度通信。
    \item \textbf{\Yajie{}共识评分：} 对所有构型计算共识评分$\{s_{\Yajie}(\mathbf{R}_i)\}_{i=1}^{N}$。
    \item \textbf{Lyapunov监控：} 计算$\Lyap_t = \frac{1}{N}\sum_i \Var_m[f_m(\mathbf{R}_i)]$。
    \item \textbf{噪声移除：} 移除$s_{\Yajie}(\mathbf{R}_i) < \tau_{\text{noise}}$的构型。
    \item \textbf{$M_t$更新：} 根据数据复杂度$\Sigma_0(\D)$和收敛进度$\Lyap_t$更新$M_{t+1}$。
    \item \textbf{输出：} 共识势函数$V^{(t)}_{\text{consensus}}$、评分$\mathbf{s}^{(t)}_{\Yajie}$、
    $\Lyap_t$、$\Mauto$、$S^{(t)}_{\Cercis}$。
\end{enumerate}
\end{definition}

\Spring{}势函数训练的数学基础由三个定理保证（详见\cite{scx_spring_trainer}）：

\begin{theorem}[多专家误差检测 — 定理P1]\label{thm:P1}
在$M_t$个独立专家的条件下，所有专家同时错过误差$\Delta$的概率满足：
\begin{equation}
    \Pbb\left(\bigcap_{m=1}^{M_t} |f_m(\mathbf{R}) - E_{\text{ref}}(\mathbf{R})| > \Delta\right) \leq 
    \exp\left(-2 M_t^{\text{eff}} \cdot \frac{\Delta^2}{\bar{\sigma}^2}\right)
\end{equation}
其中$M_t^{\text{eff}} = M_t / (1 + \bar{\rho}_t)$为有效专家数。
\end{theorem}

\begin{theorem}[Yajie共识噪声检测 — 定理P2]\label{thm:P2}
对于含噪标签$\sigma_\epsilon > 0$，共识评分的期望满足：
\begin{equation}
    \E[s_{\Yajie} \mid \sigma_\epsilon > 0] = \left(1 + \frac{\sigma_\epsilon^2}{\bar{\sigma}_{\text{expert}}^2}\right)^{-M_t/2}
\end{equation}
含噪与无噪评分之比随$M_t$指数衰减。
\end{theorem}

\begin{theorem}[Lyapunov收敛 — 定理P3]\label{thm:P3}
在温和条件下，$\{\Lyap_t\}_{t \geq 0}$构成\Spring{}训练循环的Lyapunov函数：
\begin{equation}
    \Lyap_{t+1} \leq \Lyap_t, \quad \lim_{t \to \infty} \Lyap_t = 0
\end{equation}
即专家间分歧单调减小，系统收敛到共识。
\end{theorem}

\subsection{物理层的多模态扩展：从势函数到通用物理计算}

在大模型框架中，物理层的能力不应仅限于势函数训练。我们定义物理层的通用计算接口：

\begin{definition}[物理层计算接口]\label{def:physics_interface}
物理层暴露以下计算原语，供推理层通过RAG调用：
\begin{enumerate}[label=(\roman*),itemsep=2pt]
    \item $\textsc{DFT\_SinglePoint}(\mathbf{R}, \text{settings}) \mapsto (E, \mathbf{F}, \sigma, \text{DOS})$：
    对构型$\mathbf{R}$执行单点DFT计算（当\Spring{}势函数精度不足时回退）。
    \item $\textsc{Potential\_Query}(\mathbf{R}, V_{\text{consensus}}) \mapsto (E_{\text{pred}}, \mathbf{F}_{\text{pred}}, s_{\Yajie})$：
    使用\Spring{}共识势函数$V_{\text{consensus}}$预测构型$\mathbf{R}$的能量和力，并返回共识评分。
    \item $\textsc{MD\_Run}(V_{\text{consensus}}, \mathbf{R}_0, T, \Delta t, N_{\text{steps}}) \mapsto [\mathbf{R}(t)]$：
    使用\Spring{}势函数在恒温$T$下执行$N_{\text{steps}}$步分子动力学模拟。
    \item $\textsc{Property\_Compute}(V_{\text{consensus}}, \text{property\_type}) \mapsto \text{value}$：
    计算材料性质——弹性常数、声子谱、热导率、扩散系数等。
    \item $\textsc{Potential\_Train}(\D_{\text{new}}) \mapsto V_{\text{consensus}}^{\text{new}}$：
    在新数据$\D_{\text{new}}$上训练新的\Spring{}势函数（在线学习）。
\end{enumerate}
\end{definition}

每条计算原语不仅返回结果，还返回审计元数据——\Yajie{}共识评分、使用的$M_t$值、
\Cercis{}评分。物理层产出的每一个数字都是\textbf{可审计的}。

\subsection{物理层作为审计锚点}

在\Spring{}框架中，物理层扮演着一个特殊的角色：\textbf{审计锚点（Audit Anchor）}。
当推理层的自然语言声称可以与物理层的计算结果进行比较时，
物理层提供了\textbf{基础真理的近似}——不是绝对的Ground Truth（DFT本身有近似），
但它是\textit{独立于语言模型统计规律的}物理约束。

这一角色的理论基础是SCX哈密顿量理论~\\cite{scx_hamiltonian}：
物理层的\Spring{}势函数对应着一个明确定义的哈密顿量$H(\theta)$，
其能量景观的几何结构决定了模型的审计性质。
相比之下，纯语言模型没有物理哈密顿量——它们的“能量”只是损失函数的数值，
不具备物理系统的约束性。

\begin{remark}[物理审计锚点的强度]
物理层作为审计锚点的强度取决于两个因素：
\begin{enumerate}[label=(\arabic*),nosep]
    \item \textbf{DFT计算的精度：} 泛函选择、k点采样、赝势近似都会引入系统误差——
    物理审计锚点具有可量化的不确定性，而非绝对真理。
    \item \textbf{\Spring{}势函数的覆盖度：} \Cercis{}评分的覆盖度分量$N_{\text{cov}}$直接反映
    势函数在化学空间中的有效作用域。在覆盖域内，审计是强保证的；
    在覆盖域外（$N_{\text{cov}} < \tau$），物理审计退化为启发式。
\end{enumerate}
\end{remark}

\section{推理层：LLM与RAG在审计约束下的推理}

\subsection{推理层的双重角色}

\Spring{}推理层以大语言模型为核心，但它的角色与独立LLM有本质区别：
在\Spring{}中，LLM不是最终答案的提供者，而是\textbf{物理层计算请求的编译器和审计结果的解释器}。
它的双重角色是：

\begin{enumerate}[label=(\arabic*),itemsep=3pt]
    \item \textbf{查询编译器（Query Compiler）：}
    将用户的自然语言查询编译为物理层计算原语的调用序列。
    “这个材料有什么化学性质”不被解析为一次语义检索，
    而是被编译为$\textsc{Potential\_Query}$和$\textsc{Property\_Compute}$的调用图。
    
    \item \textbf{结果解释器（Result Interpreter）：}
    将物理层返回的数值结果（能量、力、弹性常数、声子谱等）
    翻译为自然语言解释，并附上审计元数据。
\end{enumerate}

\begin{definition}[\Spring{}推理层的查询-响应循环]\label{def:reasoning_cycle}
设用户查询为$q$（自然语言），推理层执行以下循环：
\begin{enumerate}[label=步骤 \arabic*,leftmargin=*,itemsep=2pt]
    \item \textbf{意图解析：} LLM将$q$解析为$(\text{intent}, \text{entities}, \text{constraints})$三元组，
    其中$\text{intent} \in \{\text{PropertyQuery}, \text{SimulationRequest}, \text{ComparisonRequest}, \ldots\}$。
    \item \textbf{物理调用图生成：} 根据意图生成物理层计算原语的调用图$\mathcal{G}_{\text{phys}}$。
    \item \textbf{物理层执行：} 在物理层中执行$\mathcal{G}_{\text{phys}}$，获得计算结果$\mathcal{R}_{\text{phys}}$和审计元数据$\A_{\text{phys}}$。
    \item \textbf{RAG检索：} 在\Spring{}知识库中检索与$q$和$\mathcal{R}_{\text{phys}}$相关的已有结果——
    避免重复计算，获取历史审计信息。
    \item \textbf{结果合成：} LLM将$\mathcal{R}_{\text{phys}}$和RAG检索结果合成为自然语言响应草案。
    \item \textbf{审计层验证：} 审计层对响应草案进行验证（第6节），产生审计结论。
    \item \textbf{输出格式化：} 将响应草案与审计结论打包为最终输出（第8节）。
\end{enumerate}
\end{definition}

\subsection{RAG机制：Spring知识库的结构}

\Spring{}框架中的RAG不是一个通用的文档检索系统，而是一个\textbf{面向物理计算的结构化知识库}。
它存储的内容包括：

\begin{enumerate}[label=(\alph*),itemsep=2pt]
    \item \textbf{已完成的计算：} 历史执行的$\textsc{Potential\_Query}$和$\textsc{Property\_Compute}$
    的结果，附完整的审计元数据。
    \item \textbf{\Spring{}势函数元数据：} 所有已训练的共识势函数的描述——
    覆盖的化学空间、\Cercis{}评分、$M_t$值、训练数据哈希。
    \item \textbf{跨模态关联：} 文本描述与物理计算结果的已验证关联——
    例如，“高硬度”的文本声称与弹性常数$C_{44} > X$的物理计算的对应关系。
    \item \textbf{审计历史：} 历史查询的审计结论——哪些类型的查询容易产生低共识、
    哪些材料体系的\Spring{}势函数覆盖度不足。
\end{enumerate}

RAG检索不仅是语义匹配，更包含\textbf{物理一致性检查}——检索到的历史结果必须
与当前物理层的计算结果在\Cercis{}评分的意义上一致。
不一致的情况触发审计层的深度审计。

\subsection{LLM的审计约束}

与自由推理的LLM不同，\Spring{}推理层中的LLM受到以下审计约束：

\begin{enumerate}[label=\textbf{约束 \arabic*},leftmargin=*,itemsep=3pt]
    \item \textbf{不可独立声称：} LLM不能独立于物理层进行科学声称。
    任何关于材料性质、物理量的声明必须由物理层的计算结果支撑，
    并在输出中显式引用对应的$\textsc{Potential\_Query}$或$\textsc{Property\_Compute}$调用。
    
    \item \textbf{不确定性传播：} 物理层结果的\Yajie{}共识评分和\Cercis{}评分
    必须被传播到自然语言输出中——LLM不能将低置信度的计算结果表述为确定的事实。
    
    \item \textbf{审计轨迹保留：} LLM的推理步骤——从意图解析到物理调用图生成——
    必须保留在审计轨迹中。这确保了推理过程的可追溯性：
    一个错误的答案可以被追溯到是意图解析错误还是物理计算错误。
    
    \item \textbf{不可覆盖物理层：} 如果LLM的训练语料中包含与物理层计算结果矛盾的信息，
    物理层结果优先。LLM的语料知识在\Spring{}中仅作为\textit{辅助建议}，
    而非权威来源。这一约束是“查询即计算”原则的直接推论。
\end{enumerate}

这些约束将LLM从“知识提供者”转变为“科学计算接口”——
它不再是那个声称知道一切的黑箱，而是一个\textbf{受物理约束的可审计推理引擎}。

\subsection{自然语言查询的三个层次}

为说明推理层的工作方式，我们以用户查询“这个材料有什么化学性质”为例，
展示自然语言查询的三个处理层次：

\begin{example}[查询：材料的化学性质]
\textbf{用户输入：} “AlN（氮化铝）有什么化学性质？”
\vspace{4pt}

\textbf{层次1 — 意图解析：}
LLM将查询解析为：
\begin{itemize}[nosep]
    \item $\text{intent} = \text{PropertyQuery}$
    \item $\text{entities} = \{\text{material}: \text{AlN}\}$
    \item $\text{properties\_requested} = \{\text{chemical\_stability}, \text{reactivity}, \text{bond\_type}\}$
\end{itemize}

\textbf{层次2 — 物理调用图生成：}
基于意图，生成以下调用图：
\begin{verbatim}
1. Potential_Query(AlN_equilibrium_structure, V_AlN_consensus)
   → E_coh, F_eq
2. Property_Compute(V_AlN_consensus, "elastic_constants")  
   → C_ij
3. Property_Compute(V_AlN_consensus, "phonon_dispersion")
   → ω(q)
4. RAG_Search("AlN chemical properties", filter: audited)
   → historical_results
\end{verbatim}

\textbf{层次3 — 结果合成与审计：}
物理层返回结果后，LLM合成响应：
\begin{quote}
“根据\Spring{}势函数计算（V\_AlN，$M_t=8$，\Cercis{}评分 $S=1.04$）：
氮化铝的化学性质如下——
(1) 结合能：$E_{\text{coh}} = -11.2$ eV/atom（\Yajie{}共识评分 0.94）；
(2) 弹性稳定性：$C_{44} = 1.97$ GPa，Born稳定性准则满足；
(3) 声子色散无虚频——动力学稳定。
审计轨迹：见附录A，包含完整物理调用日志和专家共识细节。”
\end{quote}

注意：这个响应中的每一条科学声称都直接链接到物理层的计算——
“结合能$E_{\text{coh}}$”来自$\textsc{Potential\_Query}$，
“弹性稳定性”来自$\textsc{Property\_Compute}$，
“声子色散”来自另一条$\textsc{Property\_Compute}$。
没有一条声称来自LLM的统计记忆。
\end{example}

\section{审计层：Yajie共识、$M_t$验证与跨模态审计}

\subsection{审计层在大模型架构中的位置}

审计层是\Spring{}框架的\textbf{中心控制层}——它既接收推理层的输出进行验证，
又向物理层反馈审计结果以改进势函数训练，同时向推理层更新RAG知识库。
审计层实现了“训练即审计”的核心承诺。

\subsection{\Yajie{}多专家共识的形式化}

\Yajie{}协议~\\cite{yajie_protocol}在\Spring{}大模型框架中被推广为
\textbf{跨模态多专家共识机制}。我们在此给出其在多模态语境下的形式化定义。

\begin{definition}[跨模态\Yajie{}共识]\label{def:cross_modal_yajie}
设推理层对用户查询$q$生成了一个声称集合$\mathcal{C} = \{c_1, \ldots, c_K\}$。
对于每个声称$c_k$，存在$M_k$个可验证它的审计模块（专家）。
这些审计模块可能是：
\begin{itemize}[nosep]
    \item \textbf{物理审计模块：} \Spring{}势函数的不同专家（$M_t$个）对$c_k$中涉及的物理量进行独立预测；
    \item \textbf{视觉审计模块：} 多个独立训练的视觉模型对$c_k$中涉及的图像分类进行验证；
    \item \textbf{知识审计模块：} 多个独立的知识库/检索系统对$c_k$中的事实进行交叉验证；
    \item \textbf{逻辑审计模块：} 多个独立的推理引擎对$c_k$中的逻辑推导进行验证。
\end{itemize}

声称$c_k$的\Yajie{}共识评分定义为：
\begin{equation}\label{eq:cross_modal_yajie_score}
    s_{\Yajie}(c_k) = \exp\left(-\frac{1}{M_k}\sum_{m=1}^{M_k}
    \left(\frac{\hat{a}_m(c_k) - \bar{a}(c_k)}{\sigma_{\text{calib}, m}}\right)^2\right)
\end{equation}
其中$\hat{a}_m(c_k) \in [0,1]$为审计模块$m$对$c_k$的“正确性”判断
（1=正确，0=错误），$\bar{a}(c_k)$为共识判断，
$\sigma_{\text{calib}, m}$为模块$m$的校准标准差。
\end{definition}

\subsection{$M_t$验证：从势函数专家到大模型审计专家}

在\Spring{}势函数训练器中，$M_t$指代势函数专家的数量——$M_t$由数据复杂度
$\Sigma_0(\D)$自动生成，并通过密码学哈希与训练数据绑定。
在大模型框架中，$M_t$的概念被推广为\textbf{审计专家池的规模}。

\begin{definition}[审计专家池的$M_t$]\label{def:audit_Mt}
对于用户查询$q$，审计层需要$M_t(q)$个独立审计模块来验证响应。
\begin{equation}\label{eq:audit_Mt_definition}
    \Mauto(q) = \Xi(\text{hash}(q, \mathcal{C}); \Lyap_{\text{prev}}, \mathbf{s}_{\Yajie}^{(\text{prev})}, \Sigma_0(\text{context}))
\end{equation}
其中$\text{hash}(q, \mathcal{C})$为查询和声称集合的密码学哈希——
确保$M_t$与具体的查询-声称对绑定，而非全程统一。
\end{definition}

\textbf{绑定到查询层面}的意义在于：不同复杂度的查询需要不同数量的审计专家。
一个关于简单事实的查询（“水的沸点是多少”）可能仅需$M_t=3$个知识审计模块，
而一个关于DFT计算结果的物理声称可能需要$M_t=12$个物理+知识审计模块。

\subsection{审计层的四级验证机制}

审计层对推理层的每一个输出执行四级递增强度的验证：

\begin{enumerate}[label=\textbf{V\arabic*},leftmargin=*,itemsep=5pt]
    \item \textbf{V1 — 格式合规验证（Format Compliance）：}
    检查输出格式是否满足审计要求：
    \begin{itemize}[nosep]
        \item 每条科学声称是否显式链接到物理层计算或RAG结果？
        \item \Cercis{}评分和审计轨迹是否完整？
        \item $M_t$引用是否正确？
    \end{itemize}
    格式验证是轻量级、高吞吐量的——所有输出必须通过V1。

    \item \textbf{V2 — 共识一致性验证（Consensus Consistency）：}
    检查推理层输出的声称是否与\Yajie{}共识一致：
    \begin{itemize}[nosep]
        \item $s_{\Yajie}(c_k) \geq \tau_{\text{consensus}}$是否对所有声称$c_k$成立？
        \item 若某个声称的共识评分低于阈值，触发专家分歧分析——
        哪些审计模块同意了？哪些不同意？分歧的模式是什么？
    \end{itemize}

    \item \textbf{V3 — 跨模态一致性验证（Cross-Modal Consistency）：}
    检查不同模态的审计信息是否相互一致：
    \begin{itemize}[nosep]
        \item 文本声称的物理量是否与物理层的计算结果一致？
        \item 图像分类结果是否与多专家视觉模型的共识一致？
        \item 表格数据的统计结论是否与底层原始数据的重新计算一致？
    \end{itemize}
    跨模态不一致是幻觉检测的最强信号（详见第7节）。

    \item \textbf{V4 — 历史一致性验证（Historical Consistency）：}
    检查当前输出的声称是否与\Spring{}知识库中的历史审计记录一致：
    \begin{itemize}[nosep]
        \item 对于相同或相似的查询，本次响应是否与历史响应一致？
        \item 若不一致，差异是否由（i）新的物理计算，（ii）更新后的势函数，还是（iii）推理错误引起？
    \end{itemize}
\end{enumerate}

\subsection{审计闭环：训练即审计的完整实现}

审计层不是一个被动的验证层——它主动参与训练过程，形成完整的审计闭环：

\begin{protocol}[审计闭环协议]\label{prot:audit_loop}
对于第$t$次用户交互：
\begin{enumerate}[label=步骤 \arabic*,leftmargin=*,itemsep=3pt]
    \item \textbf{推理层产生响应草案} $\mathcal{R}_{\text{draft}}^{(t)}$ 及其声称集合 $\mathcal{C}^{(t)}$。
    \item \textbf{Audit-V1}：格式合规检查。若不通过，退回推理层重新格式化。
    \item \textbf{Audit-V2}：对$\mathcal{C}^{(t)}$中的每个声称计算\Yajie{}共识评分。
    标记低共识声称$\mathcal{C}_{\text{low}}^{(t)} = \{c_k : s_{\Yajie}(c_k) < \tau_{\text{consensus}}\}$。
    \item \textbf{Audit-V3}：对物理相关的声称执行跨模态验证——
    触发物理层的独立重算（若RAG缓存中无匹配结果）。
    \item \textbf{Audit-V4}：与历史审计记录比较。记录新发现的不一致性。
    \item \textbf{审计结论生成：} 汇总V1-V4的验证结果，生成最终审计结论——
    通过的声称、标记的声称、分歧的审计模块、\Cercis{}评分。
    \item \textbf{反馈传播：}
    \begin{itemize}[nosep]
        \item 向推理层反馈：低共识标记→推理层在后续响应中调整置信度表述。
        \item 向物理层反馈：跨模态不一致→触发物理层的势函数精度评估。
        \item 向知识库反馈：审计结论写入\Spring{}知识库。
    \end{itemize}
    \item \textbf{输出最终响应：} 将$\mathcal{R}_{\text{draft}}^{(t)}$与审计结论打包输出。
\end{enumerate}
\end{protocol}

\section{跨模态审计定理}

\subsection{跨模态审计的形式化}

跨模态审计是\Spring{}框架最具区分性的能力之一。它基于一个简单但强大的原则：
\textbf{不同模态的信息可以通过物理规律相互约束}。
一个文本声称可以通过DFT计算验证，一个图像分类可以通过多专家视觉模型验证。

\begin{definition}[跨模态审计映射]\label{def:cross_modal_mapping}
设$\mathcal{C}_{\text{text}}$为推理层生成的文本声称集合，
$\mathcal{V}_{\text{phys}}$为物理层可计算验证的物理量集合。
跨模态审计映射$\Phi: \mathcal{C}_{\text{text}} \to \mathcal{V}_{\text{phys}}$将文本声称
映射到可计算的物理验证条件：
\begin{equation}\label{eq:cross_modal_map}
    \Phi(c) = v \in \mathcal{V}_{\text{phys}} \text{ 使得 } v \text{ 的真值可判定 } c \text{ 的真值}
\end{equation}
若存在这样的映射，则声称$c$是\textbf{跨模态可审计的}（Cross-Modally Auditable）。
\end{definition}

并非所有文本声称都是跨模态可审计的。“莎士比亚写了《哈姆雷特》”没有直接的物理验证——
这类声称仅由知识审计模块通过文献一致性来验证。但“AlN的带隙约为6.2 eV”
是跨模态可审计的——物理层可以通过DFT计算（或\Spring{}势函数）验证或否定这一数值。

\begin{theorem}[跨模态审计的检测能力 — 定理CM1]\label{thm:CM1}
设文本声称$c$涉及物理量$v$，物理层有$M_t \geq 3$个独立专家可计算$v$。
在物理层专家预测的校准方差为$\bar{\sigma}^2$、声称偏差为$\delta = |\text{claimed}(v) - \text{true}(v)|$的条件下，
跨模态审计检测$\delta > \Delta$的概率满足：
\begin{equation}\label{eq:CM1_bound}
    \Pbb(\text{detect} \mid \delta > \Delta) \geq 1 - \exp\left(-2 M_t^{\text{eff}} \cdot \frac{(\delta - \Delta)^2}{\bar{\sigma}^2}\right)
\end{equation}
\end{theorem}

\begin{proof}
跨模态审计等价于对物理量$v$的$M_t$个独立测量的假设检验：
$H_0: v = v_{\text{claimed}}$ vs $H_1: v \neq v_{\text{claimed}}$。
在$M_t$个独立专家的条件下，每个专家提供$v$的一个独立估计$\hat{v}_m$。
检验统计量$T = \frac{1}{M_t}\sum_m (\hat{v}_m - v_{\text{claimed}})^2 / \bar{\sigma}^2$。
在$H_0$下，$T \sim \chi^2_{M_t^{\text{eff}}}$（经相关性校正）。
在$H_1$下（$\delta > \Delta$），$T$的非中心参数$\lambda = M_t^{\text{eff}} \cdot \delta^2 / \bar{\sigma}^2$。
应用非中心$\chi^2$的尾部界得式\eqref{eq:CM1_bound}。\hfill $\square$
\end{proof}

\begin{corollary}[跨模态审计的指数优势]\label{cor:CM1}
跨模态审计的检测能力随物理层专家数$M_t$指数增长。
当$M_t^{\text{eff}} = 6$，$\delta = 3\bar{\sigma}$时，检测概率$\geq 1 - 10^{-15}$——
即跨模态审计几乎必然检测到显著的物理声称偏差。
\end{corollary}

\subsection{跨模态审计的三种模式}

跨模态审计在实践中以三种模式运行，取决于可用的物理验证资源：

\begin{enumerate}[label=\textbf{模式 \arabic*},leftmargin=*,itemsep=4pt]
    \item \textbf{直接验证模式（Direct Verification）：}
    文本声称$c$直接对应一个可计算的物理量。
    \begin{example}
    声称：“AlN的Wurtzite结构晶格常数$a = 3.11$ \AA”
    验证：$\textsc{Potential\_Query}(\text{AlN\_wurtzite}, V_{\text{consensus}}) \to a_{\text{pred}}$
    若$|a_{\text{pred}} - 3.11| > 3\sigma_{\text{calib}}$ → 声称标记为可疑。
    \end{example}

    \item \textbf{一致性验证模式（Consistency Verification）：}
    文本声称$c$不直接对应可计算量，但可通过物理规律间接验证。
    \begin{example}
    声称：“AlN在高温下不分解”
    验证：$\textsc{MD\_Run}(V_{\text{consensus}}, \mathbf{R}_{\text{AlN}}, T=2000\text{K}, \ldots)$
    若MD模拟显示2000K下结构不稳定 → 声称标记为可疑。
    \end{example}

    \item \textbf{多专家分歧模式（Multi-Expert Disagreement）：}
    文本声称涉及多个物理量之间的关系，通过多专家对关系的一致性判断来审计。
    \begin{example}
    声称：“AlN的弹性模量与SiC相似”
    验证：$\textsc{Property\_Compute}(V_{\text{AlN}}, \text{elastic}) \to C_{ij}^{\text{AlN}}$
    $\textsc{Property\_Compute}(V_{\text{SiC}}, \text{elastic}) \to C_{ij}^{\text{SiC}}$
    比较二者差异是否在声称的“相似”语义范围内。
    \end{example}
\end{enumerate}

\subsection{跨模态审计与幻觉检测}

跨模态审计为大模型幻觉检测提供了迄今最强的理论基础。
其原因在于：\textbf{语言模型的幻觉在本质上是对物理约束的违反}——
模型说了一个在物理上不可能的事情，但因为它只处理文本，它不知道这是不可能的。

跨模态审计通过引入物理约束打破了这一封闭循环：
\begin{itemize}[itemsep=2pt]
    \item 语言模型的文本声称被映射为物理计算请求；
    \item 物理层的独立计算产生与文本声称\textit{独立}的数值结果；
    \item 文本声称与物理结果的比较检测不一致——即幻觉；
    \item 因为物理层的计算不依赖于语言模型的统计规律，
    物理层的“正确”判断不会被语言模型的同一分布偏差所污染。
\end{itemize}

\begin{proposition}[跨模态幻觉检测的独立性]\label{prop:independence}
在\Spring{}框架中，物理层的预测$\hat{v}_{\text{phys}}$与推理层LLM的文本声称$c$
在信息论意义上是独立的：$I(\hat{v}_{\text{phys}}; c \mid q) = 0$。
因此，跨模态审计的假阳性率不受LLM内在偏差的影响。
\end{proposition}

\section{输出层：答案、\Cercis{}评分与审计轨迹的统一交付}

\subsection{输出层的三元组结构}

\Spring{}框架的每一个输出都是以下三元组：
\begin{equation}\label{eq:output_triplet}
    \boxed{\text{Output}(q) \mapsto \Bigl(
        \underbrace{\mathcal{R}_{\text{answer}}}_{\text{自然语言答案}},\ 
        \underbrace{S_{\Cercis}}_{\text{Cercis评分}},\ 
        \underbrace{\mathcal{T}_{\text{audit}}}_{\text{审计轨迹}}
    \Bigr)}
\end{equation}

\begin{definition}[输出三元组的各分量]\label{def:output_components}
\begin{enumerate}[label=(\arabic*),leftmargin=*,itemsep=3pt]
    \item \textbf{自然语言答案 $\mathcal{R}_{\text{answer}}$：}
    推理层合成的自然语言响应。关键特征是：答案中的每一条科学声称都带有审计标签——
    \textcolor{green!50!black}{[已验证]}、\textcolor{orange}{[部分验证]}、
    \textcolor{red}{[未验证]}——以及指向物理层调用或RAG条目的追溯链接。
    
    \item \textbf{\Cercis{}评分 $S_{\Cercis}$：}
    跨架构可比的单一质量度量，定义为：
    \begin{equation}\label{eq:cercis_framework}
        S_{\Cercis}(\mathcal{R}_{\text{answer}}) = Q_{\text{prec}}(\mathcal{R}_{\text{answer}}) + \eta \cdot N_{\text{cov}}(\mathcal{R}_{\text{answer}})
    \end{equation}
    其中：
    \begin{itemize}[nosep]
        \item $Q_{\text{prec}} \in [0,1]$：精度得分——由\Yajie{}共识评分的加权平均决定。
        \item $N_{\text{cov}} \in [0,1]$：覆盖度得分——答案中可审计声称的比例。
        一个声称$c$是可审计的当且仅当存在至少$M_{\min}=3$个审计模块可以独立验证它。
        \item $\eta = 0.2$：认知折扣因子——审计总是具有认知成本。
    \end{itemize}
    
    \item \textbf{审计轨迹 $\mathcal{T}_{\text{audit}}$：}
    完整的审计历史记录，包括：
    \begin{itemize}[nosep]
        \item 查询$q$和响应的密码学哈希；
        \item 推理层的意图解析和物理调用图；
        \item 物理层的所有计算请求及返回结果；
        \item 审计层的V1-V4验证结果——每个声称的共识评分、哪些审计模块同意/不同意；
        \item 使用的$M_t$值及与数据哈希的绑定验证；
        \item 时间戳和审计层签名。
    \end{itemize}
\end{enumerate}
\end{definition}

\subsection{\Cercis{}评分的多模态推广}

原始的\Cercis{}评分~\\cite{scx_ml_audit}是为机器学习模型的审计设计的。
在\Spring{}大模型框架中，我们将其推广到多模态输出场景。

\begin{definition}[多模态\Cercis{}评分]\label{def:multimodal_cercis}
设输出$\mathcal{R}_{\text{answer}}$包含$K$个声称$\{c_1, \ldots, c_K\}$，其中
$K_{\text{auditable}}$个是可审计的（存在$\geq 3$个独立审计模块）。
多模态\Cercis{}评分定义为：
\begin{equation}\label{eq:multimodal_cercis}
    S_{\Cercis} = \underbrace{\frac{1}{K}\sum_{k=1}^{K} s_{\Yajie}(c_k)}_{Q_{\text{prec}}} 
    + \eta \cdot \underbrace{\frac{K_{\text{auditable}}}{K}}_{N_{\text{cov}}}
\end{equation}
\end{definition}

多模态\Cercis{}评分的直观含义是：
\begin{itemize}[nosep]
    \item $Q_{\text{prec}}$高意味着输出中的声称在审计模块之间高度共识——答案内部一致。
    \item $N_{\text{cov}}$高意味着大部分声称有独立的审计支撑——答案外部可验证。
    \item 只有两者都高，$S_{\Cercis}$才高——高度共识但不可验证（$N_{\text{cov}}$低）
    或高度可验证但分歧大（$Q_{\text{prec}}$低）的答案都不被信任。
\end{itemize}

\subsection{审计轨迹的密码学完整性}

审计轨迹$\mathcal{T}_{\text{audit}}$不仅是信息记录，更是\textbf{密码学可验证的证据链}：

\begin{enumerate}[label=(\arabic*),itemsep=2pt]
    \item $\mathcal{T}_{\text{audit}}$的每个条目都经过审计层签名。
    \item 条目之间通过哈希链连接——每个新条目包含前一条目的哈希。
    \item $M_t$与查询数据的绑定通过$D_{\text{hash}}$验证。
\end{enumerate}

这确保了审计轨迹的\textbf{不可篡改性}：任何对历史审计记录的修改都会破坏哈希链，
从而被检测到。审计轨迹是\Spring{}框架实现“输出即审计”的密码学基础。

\section{与现有大模型的系统比较}

\subsection{$M=1$，UNDECLARED：现有大模型的审计状态}

现有代表性大模型——GPT-4、Claude 3、Gemini——在审计维度上处于\textbf{$M=1$，UNDECLARED}
状态。我们逐一分析。

\begin{enumerate}[label=\textbf{模型 \arabic*},leftmargin=*,itemsep=6pt]
    \item \textbf{GPT-4（OpenAI）：}
    GPT-4是一个$M=1$的单体模型——一个神经网络产生所有输出。
    在审计维度上的结构性缺陷：
    \begin{itemize}[nosep]
        \item \textbf{无多专家机制：} GPT-4虽然有MoE（Mixture of Experts）架构，但这些
        “专家”是训练过程中自动形成的路由模式，而非独立训练的审计模块。它们共享
        训练数据和损失函数，不具备独立审计所需的统计独立性。
        \item \textbf{无内生置信度：} GPT-4提供的是一个文本输出，没有形式化的置信度度量。
        “我认为”、“可能”、“大概”等表述是语言模型的统计输出，而非计算出的不确定性。
        \item \textbf{训练数据未经审计：} GPT-4的训练数据（WebText等）的标注质量、事实准确性
        未经系统审计。模型从可能包含错误的数据中学习，并无内建机制检测这些错误。
        \item \textbf{输出不可追溯：} 无法追溯一个具体的声称来自哪些训练数据、哪些推理步骤。
    \end{itemize}
    
    \item \textbf{Claude 3（Anthropic）：}
    Claude 3引入Constitutional AI（宪法AI）进行价值对齐，但在审计维度上仍处于$M=1$状态：
    \begin{itemize}[nosep]
        \item Constitutional AI是\textit{事后}对齐机制——它在预训练完成后附加宪法规则。
        这些规则对训练过程中的数据和模型质量没有审计能力。
        \item 输出中的自信度表述（“我很确定”）仍来自统计模式，而非计算出的不确定性。
        \item 与GPT-4相同，Claude 3没有多专家独立审计机制。
    \end{itemize}
    
    \item \textbf{Gemini（Google）：}
    Gemini在多模态方面最为先进（原生多模态训练），但仍然：
    \begin{itemize}[nosep]
        \item 是一个$M=1$的单体多模态模型——图像、文本、音频在同一个模型中处理，
        但没有独立的跨模态审计机制。
        \item 模型可以“看”图像和“听”音频，但不能验证其多模态声称的物理真实性。
        \item 尽管覆盖了更多模态，但其审计能力与GPT-4处于同一级别——UNDECLARED。
    \end{itemize}
\end{enumerate}

\begin{definition}[UNDECLARED状态]\label{def:undeclared}
一个大模型处于UNDECLARED状态当且仅当以下任一条件成立：
\begin{enumerate}[label=(\roman*),nosep]
    \item 模型训练时的专家数$M$和训练数据质量未被声明；
    \item 模型输出的置信度没有形式化的、可计算的定义；
    \item 模型输出的声称无法追溯到训练数据或推理步骤；
    \item 不存在独立于模型本身的审计机制。
\end{enumerate}
当前所有主要大模型（GPT-4、Claude 3、Gemini）均处于UNDECLARED状态。
\end{definition}

\subsection{Born Audited：Spring框架的审计状态}

与之形成鲜明对比，\Spring{}框架处于\textbf{Born Audited}状态：

\begin{definition}[Born Audited状态]\label{def:born_audited}
一个大模型处于Born Audited状态当且仅当以下所有条件成立：
\begin{enumerate}[label=(\roman*),nosep]
    \item \textbf{训练即审计：} 训练过程的每一次迭代同时产出模型参数和审计信息——
    $M_t$、\Yajie{}共识评分、\Cercis{}评分。
    \item \textbf{输出即审计：} 每一个输出携带完整的审计轨迹——
    \Cercis{}评分、审计模块的共识结果、$M_t$绑定验证。
    \item \textbf{多专家独立审计：} 审计模块在训练数据和随机初始化上独立，
    满足SCX定理1的统计独立性要求。
    \item \textbf{跨模态可验证：} 不同模态的输出可以通过物理规律相互验证。
    \item \textbf{可追溯性：} 任何声称可以追溯到其审计来源——
    哪些审计模块同意了、$M_t$是什么、共识评分是多少。
\end{enumerate}
\end{definition}

\subsection{Born Audited vs. UNDECLARED：不仅是信任问题}

从UNDECLARED到Born Audited的转变不仅是“更透明”或“更值得信任”的问题——
它改变了模型能力的基本性质。

\begin{enumerate}[label=(\arabic*),itemsep=3pt]
    \item \textbf{从统计到物理：} UNDECLARED模型的科学知识是统计性的——
    它来自训练语料中的频率和共现模式。
    Born Audited模型的科学知识是物理性的——它来自对物理系统的实际计算。
    
    \item \textbf{从不可知到可知：} UNDECLARED模型对自己何时错误是不可知的——
    SCX定理2证明了$M=1$系统无法自我审计。
    Born Audited模型通过$M_t>1$的多专家机制突破了这一限制。
    
    \item \textbf{从一次性到持续性：} UNDECLARED模型的训练是一次性的——
    训练完成后模型不再学习（除非微调，带来新的审计真空）。
    Born Audited模型的物理层通过\Spring{}自演化持续改进——每一次新的
    物理计算都更新势函数和审计信息。
    
    \item \textbf{从不可验证到可验证：} UNDECLARED模型的声称无法独立验证——
    用户只能信任或怀疑。
    Born Audited模型的声称可以直接验证——审计轨迹提供了完整的验证路径。
\end{enumerate}

\section{讨论}

\subsection{Spring框架的适用边界}

\Spring{}统一多模态大模型框架并非适用于所有场景。我们明确定义其适用边界：

\begin{enumerate}[label=(\arabic*),itemsep=3pt]
    \item \textbf{适用场景：}
    \begin{itemize}[nosep]
        \item \textbf{科学计算密集型任务：} 材料设计、药物发现、催化研究——
        这些领域的核心需求是物理计算的准确性，而非语言流畅性。
        \item \textbf{安全关键决策：} 工程材料选择、安全生产条件评估——
        输出需要可审计、可追溯。
        \item \textbf{知识密集型研究：} 跨学科文献综合、假设生成——
        需要将不同模态的信息（文本、DFT数据、实验数据）统一处理。
        \item \textbf{需要物理验证的AI应用：} 科学教育、技术写作、实验设计建议——
        AI的声称需要物理验证。
    \end{itemize}

    \item \textbf{不适用场景：}
    \begin{itemize}[nosep]
        \item \textbf{纯创意任务：} 文学创作、艺术生成、娱乐对话——
        这些任务的输出没有物理真值，跨模态审计失去意义。
        \item \textbf{实时低延迟场景：} 对话机器人、游戏NPC——
        \Spring{}的审计闭环引入了可感知的延迟。
        \item \textbf{纯模式识别任务：} 图像分类（无物理约束）、语言翻译——
        这些任务的输出类型单一，不需要跨模态审计。
    \end{itemize}

    \item \textbf{中间地带：}
    \begin{itemize}[nosep]
        \item \textbf{轻量审计模式：} 对于延迟敏感但仍需审计的场景，
        \Spring{}可配置为“轻量模式”——固定$M_t=3$，仅执行V1+V2审计。
        \item \textbf{离线审计模式：} 对历史输出进行事后审计——
        审计轨迹允许回放和重新验证。
    \end{itemize}
\end{enumerate}

\subsection{计算成本分析}

\Spring{}框架引入的额外计算成本来自三个层面：

\begin{enumerate}[label=(\arabic*),itemsep=3pt]
    \item \textbf{物理层 —— \Spring{}势函数训练的$M_t$倍成本：}
    如\cite{scx_spring_trainer}中所分析，$M_t$个专家的独立训练引入了$\mathcal{O}(M_t)$的
    FLOP开销。但因为各专家在互斥数据子集上训练，且完全可并行，实际墙钟时间
    接近单专家训练时间。这是审计的信息论代价——不可消除，但可优化。

    \item \textbf{推理层 —— RAG检索和物理计算的执行成本：}
    推理层的RAG检索和物理层计算调用的延迟。对于高频查询，
    \Spring{}知识库的缓存机制使重复查询的延迟接近纯LLM推理。

    \item \textbf{审计层 —— V1-V4验证的计算成本：}
    V1（格式验证）成本可忽略。V2（共识验证）的成本取决于审计模块的数量和类型——
    物理审计模块的实际计算已在推理阶段完成（物理层调用），V2仅是共识评分的聚合。
    V3（跨模态验证）仅在物理声称不匹配时触发完全重算（罕见事件）。
    V4（历史一致性）是轻量级的RAG查询。
\end{enumerate}

总体而言，\Spring{}框架的计算成本比单次GPT-4推理高约$2\times$至$5\times$
（取决于缓存命中率和审计深度），但这一成本应被视为\textbf{审计基础设施的运营成本}，
而非可选的额外开销。在安全关键领域，获得审计保证的成本远低于未审计错误的代价。

\subsection{诚实暴击：Spring的局限与未解决问题}

\honestcrit{\Spring{}统一多模态大模型框架是一个雄心勃勃的概念，而非一个已经所有问题
解决的技术方案。以下是我们诚实面对的限制和开放问题。}

\begin{enumerate}[label=\textbf{H\arabic*},leftmargin=*,itemsep=5pt]
    \item \textbf{物理模态的覆盖面有限：}
    \Spring{}物理层目前仅支持\Spring{}势函数训练的分子动力学体系。
    扩展到电子结构计算（DFT的实时执行）、量子化学、流体力学、电磁学——
    每一个新物理领域都需要开发新的\Spring{}审计适配器。
    这不是理论上的障碍，而是巨大的工程挑战。

    \item \textbf{跨模态审计映射的不完备性：}
    定义\ref{def:cross_modal_mapping}中的映射$\Phi$在实际中是不完备的——
    许多有意义的科学声称无法被映射到可计算的物理验证条件。
    “AlN比SiC更适合高温应用”这样的比较性、语境性声称，
    其验证需要复杂的多准则决策，而非单一的物理量计算。

    \item \textbf{LLM推理与物理计算的语义鸿沟：}
    推理层的LLM需要将自然语言查询编译为物理计算调用图（定义\ref{def:reasoning_cycle}）。
    这一编译过程的正确性目前无法被形式化保证——
    LLM可能错误地解析意图，导致正确的物理计算回答了错误的问题。
    这是\Spring{}架构中最脆弱的环节。

    \item \textbf{$M_t$自动生成的跨模态泛化：}
    定理P3中的$M_t$自动生成机制是为单一模态（势函数训练）设计的。
    将其推广到多模态审计场景——如何为文本查询确定所需的审计模块数量？——
    需要一个跨模态数据复杂度的统一定义。这一概念目前尚不存在。

    \item \textbf{审计专家池的构建与维护：}
    \Spring{}框架需要维护一个异质的审计专家池——包括不同架构的物理模型、
    视觉模型、知识库系统。这些模块的校准、更新、版本管理引入了
    实质性的运维复杂度。

    \item \textbf{\Cercis{}评分的认知折扣因子：}
    $\eta=0.2$的选择来自通用ML审计的校准~\\cite{scx_ml_audit}。
    在多模态科学计算场景中，$\eta$的最优值可能是领域相关的——
    物理计算的精度-覆盖度权衡与文本事实检查的权衡具有不同的认知成本。

    \item \textbf{审计轨迹的存储与隐私：}
    完整的审计轨迹——包括所有物理计算请求、\Yajie{}共识评分、RAG检索条目——
    可能构成大量的存储开销。在高频部署场景下，审计轨迹的存储和管理
    需要专用的基础设施。
\end{enumerate}

\subsection{与SCX理论体系的内在关系}

\Spring{}统一多模态大模型框架是SCX理论体系在大模型架构层面的系统化应用。
它与SCX其他组件的关系如下：

\begin{itemize}[itemsep=3pt]
    \item \textbf{SCX定理1（多专家检测）：} 审计层多专家机制的理论基础——
    $M_t$个独立审计模块提供误差检测的指数保证。
    \item \textbf{SCX定理2（自审计禁止）：} 解释了现有大模型（$M=1$）在审计维度的根本不可能性。
    \item \textbf{SCX定理3（噪声-信号不可区分）：} 支持\Yajie{}共识作为噪声检测的唯一可靠机制。
    \item \textbf{\Yajie{}协议~\\cite{yajie_protocol}：} 审计层核心算法的理论基础。
    \item \textbf{SCX哈密顿量理论~\\cite{scx_hamiltonian}：} 物理层\Spring{}势函数的Lyapunov收敛性的统计力学基础。
    \item \textbf{\Spring{}势函数训练器~\\cite{scx_spring_trainer}：} 物理层核心组件的完整理论和实现。
    \item \textbf{\Cercis{}评分~\\cite{scx_ml_audit}：} 跨模态质量度量的形式化基础。
    \item \textbf{Galois-SCX~\\cite{scx_galois}：} 审计专家群的群论结构理论——解释了审计模块的不可约分解。
    \item \textbf{SCX因果关系~\\cite{scx_causal}：} 跨模态审计中因果推断的理论基础。
\end{itemize}

\section{结论}

本文提出了\Spring{}统一多模态大模型框架——第一个将训练过程作为跨模态审计过程的
统一智能架构。我们的核心贡献可以总结为以下七点：

\begin{enumerate}[label=(\arabic*),itemsep=4pt]
    \item \textbf{从工具到架构的范式转换：}
    \Spring{}不是另一个大模型工具，而是对大模型架构本身的重新定义——
    一个将多模态感知、物理建模、自然语言理解和内生审计统一在单一架构循环中的完整系统。

    \item \textbf{五层统一架构：}
    输入层（任意模态编码）→物理层（\Spring{}迭代势函数）→推理层（LLM + RAG）
    →审计层（\Yajie{}共识、$M_t$验证）→输出层（答案+\Cercis{}评分+审计轨迹）——
    五层管道遵循“训练即审计”、“查询即计算”、“输出即审计”的内生统一原则。

    \item \textbf{物理层作为审计锚点：}
    \Spring{}物理层不是外挂的工具调用，而是模型推理管道的内生组件。
    它提供独立于语言模型统计规律的物理约束——这是跨模态审计和幻觉检测的物理基础。

    \item \textbf{跨模态审计定理（定理CM1）：}
    我们证明了跨模态审计的检测能力随物理层专家数$M_t$指数增长——
    这是首次将跨模态验证能力以数学定理的形式严格表征。

    \item \textbf{Born Audited vs. UNDECLARED：}
    我们给出了大模型审计状态的形式化定义——
    Born Audited（\Spring{}）vs. UNDECLARED（GPT-4、Claude 3、Gemini）。
    从UNDECLARED到Born Audited的转变不是透明度的改善，而是模型能力的质性变化。

    \item \textbf{输出三元组：}
    \Spring{}框架的每一个输出都是答案+\Cercis{}评分+审计轨迹的不可分离三元组——
    输出与审计不可分离。审计轨迹的密码学完整性确保了长期的可追溯性和不可篡改性。

    \item \textbf{诚实暴击：}
    我们诚实地指出了\Spring{}框架的七个关键限制——
    物理模态覆盖的有限性、跨模态映射的不完备性、LLM编译的语义鸿沟、
    $M_t$自动生成的跨模态泛化、审计专家池的运维复杂度、
    $\eta$折扣因子的领域敏感性、以及审计轨迹的存储成本。
    这些不是对\Spring{}的否定，而是对后续工作的诚实指引。
\end{enumerate}

\Spring{}统一多模态大模型框架的核心主张可以浓缩为一句话：
\textbf{大模型的未来不是更大的单一模型，而是内建审计的统一架构——}
模型在训练中审计，在推理中验证，在输出中追溯。
“一句话知道材料特性”不是营销口号，而是\Spring{}框架的形式化保证：
自然语言查询→物理层势函数计算→\Yajie{}共识验证→\Cercis{}评分输出——
每一个环节都是可审计的，每一个输出都是可追溯的。

这不是一个容易实现的愿景。诚实暴击中列出的挑战是真实的、深刻的。
但我们相信，\Spring{}框架指向的方向——\textbf{Born Audited, not UNDECLARED}——
是大模型发展的必然下一阶段。
