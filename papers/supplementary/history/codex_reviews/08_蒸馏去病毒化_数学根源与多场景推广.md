### 一、问题定义：蒸馏为什么会被''中毒''？<!-- label: ux4e00ux95eeux9898ux5b9aux4e49ux84b8ux998fux4e3aux4ec0ux4e48ux4f1aux88abux4e2dux6bd2 -->

#### 1.1 标准知识蒸馏（Knowledge
Distillation）回顾<!-- label: ux6807ux51c6ux77e5ux8bc6ux84b8ux998fknowledge-distillationux56deux987e -->

Hinton et al.~(2015) 的框架：

**Teacher** f\_T: X → Y，在训练集 D\_train = \{(x\_i, y\_i*)\}
上训练得到。

**蒸馏**：Teacher 对一组无标签数据 \{x\_i\^{}distill\}
生成软标签或硬标签：

\begin{verbatim}
y_i^distill = f_T(x_i^distill)
\end{verbatim}

**Student** f\_S: X → Y，在 D\_distill = \{(x\_i\^{}distill,
y\_i\^{}distill)\} 上训练。

#### 1.2
问题的数学本质<!-- label: ux95eeux9898ux7684ux6570ux5b66ux672cux8d28 -->

核心问题是：**Teacher 不是完美的**。定义 Teacher 的状态条件风险：

\begin{verbatim}
R_T(s) = E_{x~P(·|s)} [ ℓ(f_T(x), f*(x)) ]
\end{verbatim}

其中 f*(x) 是真实标签（未知），s 是状态。

**关键观察**：R\_T(s) 随 s 变化很大。在 AlN v3 训练中：

\begin{longtable}[]{@{}ll@{}}

\endhead
\bottomrule\noalign{}

EOS 近平衡区 & 0.015 

Elastic 变形区 & 0.011

**MLMD stress10 区** & **0.036** 

\end{longtable}

Teacher 在 thermal 区的误差是 phonon 区的 **9.6 倍**。

**蒸馏中毒机制**：

\begin{verbatim}
Teacher 在 thermal 1800K 区生成蒸馏数据
    → 蒸馏标签 y_distill = f_T(x) = f*(x) + δ_T(s)  [δ_T 是 teacher 在状态 s 的偏差]
    → Student 拟合: f_S(x) ≈ y_distill = f*(x) + δ_T(s)
    → Student 在 thermal 区的误差 ≥ δ_T(s)
    → Student 从 Teacher 那里"继承"了系统偏差
\end{verbatim}

这和 VASP 数据中毒的不同在于：

\begin{verbatim}
VASP 中毒：f*(x) 本身就错了（DFT 标签有问题）
蒸馏中毒：f*(x) 是对的（DFT 标签正确），但 Teacher 没有正确学会 f*(x)
\end{verbatim}

**两者的共同点**：都是状态条件的------中毒只在特定构型区域发生。

### 二、现有方法的数学分析<!-- label: ux4e8cux73b0ux6709ux65b9ux6cd5ux7684ux6570ux5b66ux5206ux6790 -->

#### 2.1 Hinton
KD（2015）------不提纯，全量蒸馏<!-- label: hinton-kd2015ux4e0dux63d0ux7eafux5168ux91cfux84b8ux998f -->

**做法**：T → S，所有蒸馏样本一视同仁。

**隐式假设**：Teacher 在所有输入区域同样可靠。

**数学上**：这等价于假设
`Var{[}ℓ(f\_T(x),\ f*(x)){]}\ ≈\ 0`，即 teacher error
的方差可以忽略。但在 MLIP 场景下，Teacher
在不同构型区域的误差差异可达一个数量级------这个假设明显不成立。

#### 2.2 置信度过滤（Confidence-based
Filtering）<!-- label: ux7f6eux4fe1ux5ea6ux8fc7ux6ee4confidence-based-filtering -->

**做法**：用 Teacher 的输出熵 H(f\_T(x)) 或最大 softmax
值作为''置信度''，过滤低置信度的蒸馏样本。

**数学形式**：

\begin{verbatim}
D_clean = {x_i | max_j f_T^{(j)}(x_i) > τ_confidence}
\end{verbatim}

**在 MLIP 场景的失败原因**：

Teacher（ACE 线性模型）输出的是能量 E 和力
F，不是概率分布。没有''置信度''概念------E 和 F
是确定性输出。**置信度过滤直接不适用。**

即使在分类场景下： - 高置信度 ≠
低误差。一个模型可以对错误预测高度自信（overconfidence） - 置信度不反映
state-conditioned 的可靠性结构

#### 2.3 集成不一致度过滤（Ensemble
Disagreement）<!-- label: ux96c6ux6210ux4e0dux4e00ux81f4ux5ea6ux8fc7ux6ee4ensemble-disagreement -->

**做法**：训练 M 个
Teacher（不同初始化/数据子集），用它们输出的方差作为不确定性：

\begin{verbatim}
σ²_ens(x) = Var[f_T1(x), f_T2(x), ..., f_TM(x)]
D_clean = {x_i | σ²_ens(x_i) < τ_ens}
\end{verbatim}

**优点**：比置信度更可靠，捕捉到了模型在不同区域的不确定性。

**局限**： - 需要训练 M 个 Teacher → 成本高（MLIP 场景下 M 个 ACE
训练 = M × 40 epochs） - 集成方差反映的是''多个 Teacher
之间的一致程度''，不是''Teacher 和真实标签之间的误差'' - 如果所有
Teacher
在同一个区域都犯同样的系统偏差（这是常见情况------架构限制导致的），集成方差会很低，但实际误差很大

#### 2.4 Data Shapley / LOO 值<!-- label: data-shapley-loo-ux503c -->

**做法**：每个训练样本算 Shapley 值或 LOO
误差，高价值样本保留，低价值丢弃。

**数学上**：Shapley 值在第 i
个样本上计算的是''这个样本对模型性能的边际贡献''。但这是**全局量**------它不区分在不同状态上的贡献。

**局限**：和置信度一样，Shapley 值不包含状态条件信息。一个对 phonon
区有贡献的样本和一个对 thermal 区有贡献的样本，Shapley 值可能相同，但
Teacher 在 thermal 区的可靠性完全不一样。

### 三、SCX
蒸馏去病毒化的数学框架<!-- label: ux4e09scx-ux84b8ux998fux53bbux75c5ux6bd2ux5316ux7684ux6570ux5b66ux6846ux67b6 -->

#### 3.1 核心思想<!-- label: ux6838ux5fc3ux601dux60f3 -->

**不是逐样本判断蒸馏数据的质量，而是在状态层面判断 Teacher
的可靠性。**

直觉： - 逐样本判断：这个蒸馏帧的力标签对吗？→ 无法回答，因为没有 ground
truth - 状态级判断：Teacher 在''thermal 1800K
构型区域''的整体可靠性是多少？→ 可以回答，因为 Teacher 在原始训练集的
thermal 测试帧上已经表现出高误差

#### 3.2 形式化定义<!-- label: ux5f62ux5f0fux5316ux5b9aux4e49 -->

**已知**（来自 Teacher 训练）： - D\_train = \{(x\_i, y\_i*)\} ---
原始训练集，有 DFT 标签 - Encoder φ: X → R\^{}d --- 状态编码器 - Teacher
f\_T 已训练

**可计算**（无需额外 DFT）： - Teacher 在 D\_train 上每个样本的
residual：r\_i = ℓ(f\_T(x\_i), y\_i*) - 状态发现：用 φ 聚类 → 状态划分 S
= \{s\_1, ..., s\_K\} - 每个状态的 Teacher reliability：

\begin{verbatim}
R_T(s_k) = (1/|s_k|) Σ_{x_i∈s_k} ℓ(f_T(x_i), y_i*)
\end{verbatim}

这就是 **Teacher Reliability Map**------本质上是一张''Teacher
在哪些构型区域不可靠''的地图。

**蒸馏数据评估**： - 蒸馏数据 D\_distill = \{(x\_j\^{}distill,
f\_T(x\_j\^{}distill))\} - 对每个 x\_j\^{}distill，编码
φ(x\_j\^{}distill)，分配到最近状态 s\_k - 如果 R\_T(s\_k) \textgreater{}
τ，标记 x\_j\^{}distill 为''病毒帧''

**过滤后蒸馏**：

\begin{verbatim}
D_clean = {x_j^distill | R_T(state(x_j^distill)) ≤ τ}
\end{verbatim}

训练 Student f\_S 在 D\_clean 上。

#### 3.3
数学保证（启发式）<!-- label: ux6570ux5b66ux4fddux8bc1ux542fux53d1ux5f0f -->

**Claim**：设 Teacher 在状态 s 上的偏差为 δ\_T(s) =
| E{[}f\_T(x) - f*(x) |{} x∈s{]}|。Student
训练在过滤后的蒸馏集上，其在状态 s 上的期望偏差满足：

\begin{verbatim}
E[|f_S(x) - f*(x)| | x∈s] ≤ δ_S_clean(s) + λ · max(0, δ_T(s) - τ)
\end{verbatim}

其中 δ\_S\_clean(s) 是 Student 在清洁数据上的本征学习误差，第二项是
teacher 偏差的泄漏。

**解读**：通过设置 τ 过滤高偏差状态，可以控制 Teacher 系统误差向
Student 的传播。过滤越严格（τ 越小），泄漏越少，但蒸馏数据越少。

**权衡**：

\begin{verbatim}
V_s(τ) = |D_clean(τ)| / |D_distill|  — 保留比例
Q_s(τ) = 1 - (δ_T(s) - τ)_+ / δ_T(s)  — 质量提升比例
\end{verbatim}

最优 τ 的选择取决于：Teacher
在不同状态的可靠性分布、蒸馏数据的覆盖需求、以及 Student
对数据量的敏感度。

#### 3.4
为什么两层描述符在这里特别关键<!-- label: ux4e3aux4ec0ux4e48ux4e24ux5c42ux63cfux8ff0ux7b26ux5728ux8fd9ux91ccux7279ux522bux5173ux952e -->

单层 Encoder（如 MLIPEncoder 12-dim）： - 粗粒度，50\% 帧挤在一个状态 -
无法区分''EOS 近平衡帧''和''thermal 1800K
帧''（它们可能被分到同一个粗状态） - → 一个状态内混杂着 Teacher
可靠和不可靠的子区域 - → 过滤时要么''错杀''可靠帧，要么''漏网''不可靠帧

两层 Encoder（Layer 1 + ErrorDrivenEncoder）： - Layer 1
粗聚：大区域划分 - Layer 2 误差驱动细聚类：在每个 Layer 1 簇内，用与
Teacher 误差相关的特征维度再聚类 - → 把 Teacher
不可靠的子区域从可靠区域中分离出来 - → 精确过滤：只丢弃 Teacher
确实不可靠的子状态

**这就是两层方法在蒸馏去病毒化中的独特价值------它能把''Teacher
的可靠区''和''Teacher 的不可靠区''在同一个粗区域内分开。**

### 四、不同场景的蒸馏去病毒化<!-- label: ux56dbux4e0dux540cux573aux666fux7684ux84b8ux998fux53bbux75c5ux6bd2ux5316 -->

#### 4.1 场景 1：MLIP 势函数蒸馏（ACE →
ACE/NEP）<!-- label: ux573aux666f-1mlip-ux52bfux51fdux6570ux84b8ux998face-acenep -->

**蒸馏方式**：Teacher ACE 势函数对大量构型做单点能+力计算 → 生成
(构型, 能量, 力) 数据 → 训练 Student NEP/轻量 ACE。

**中毒风险**：Teacher 在 thermal 1800K/MLMD stress10
等极端区域的高误差传播给 Student。

**SCX 方案**： 1. Teacher 在 D\_train (534 帧) 上训练 2. 用
D\_train 计算 Teacher per-state 力 RMSE → Reliability Map 3. 蒸馏生成
5000 帧（Teacher 对随机扰动构型做单点计算） 4. SCX Encoder 编码蒸馏帧 →
映射到状态 5. 过滤 thermal 1800K 区、MLMD stress10 区的蒸馏帧 6.
用清洁蒸馏集训练 Student

**Encoder 选择**：`MLIPEncoder`（12-dim
手工描述符），用两层方法细化。

#### 4.2 场景 2：图像模型蒸馏（ResNet-50 →
MobileNet）<!-- label: ux573aux666f-2ux56feux50cfux6a21ux578bux84b8ux998fresnet-50-mobilenet -->

**蒸馏方式**：Teacher ResNet-50 对无标签图像做推理 → (图像, soft
label) → 训练 MobileNet。

**中毒风险**：Teacher
在特定图像子空间（如低光照、遮挡、模糊）的识别误差传播给 Student。

**SCX 方案**： 1. Teacher 在 CIFAR/ImageNet 训练集上训练 2.
用验证集计算 Teacher per-state 分类误差 → Reliability Map 3.
蒸馏大量无标签图像 4. `VisionEncoder`（CNN backbone）编码 →
映射到状态 5. 过滤 Teacher 高误差状态（如模糊图像区）的蒸馏帧 6.
用清洁蒸馏集训练 Student

**Encoder 选择**：`VisionEncoder`（simple\_cnn/resnet
backbone）。

**和置信度过滤的关键区别**：置信度过滤只看 Teacher 的 softmax
输出（``Teacher 自己有没有把握''），SCX 看 Teacher
在相似图像上的实际预测误差（``Teacher 在这个区域真的错得多不多''）。

#### 4.3 场景 3：LLM 蒸馏（GPT →
小模型）<!-- label: ux573aux666f-3llm-ux84b8ux998fgpt-ux5c0fux6a21ux578b -->

**蒸馏方式**：Teacher LLM 生成大量 QA 对 → (prompt, response) →
训练 Student 小模型。

**中毒风险**：Teacher
在特定语义子空间（如推理类问题、代码生成）的幻觉/错误被蒸馏数据放大传播。

**SCX 方案**： 1. Teacher 在 benchmark 数据集上评估 2.
用评估结果建立 Teacher per-topic 可靠性 → Reliability Map 3.
蒸馏生成大量 QA 对 4. `LLMEncoder`（BERT/GPT embedding）编码
prompt → 映射到语义状态 5. 过滤 Teacher
高误差语义状态（如多步推理）的蒸馏 QA 对 6. 用清洁蒸馏集训练 Student

**Encoder 选择**：需要实现新的 `LLMEncoder`------用 sentence
embedding（如 BERT）将 prompt 编码为向量，在语义空间中聚类。

**独特价值**：这是置信度过滤也难以做好的场景------LLM
经常对错误答案高度自信（hallucination with high confidence）。SCX 不依赖
Teacher 的自我评估，而是用外部 benchmark 上的实际表现来刻画可靠性。

#### 4.4 场景 4：仿真保真度蒸馏（Fine-mesh FEM → Coarse-mesh +
Surrogate）<!-- label: ux573aux666f-4ux4effux771fux4fddux771fux5ea6ux84b8ux998ffine-mesh-fem-coarse-mesh-surrogate -->

**蒸馏方式**：精细网格 FEM 在参数空间采样 → (参数, 响应) →
训练降阶模型/粗网格修正项。

**中毒风险**：精细网格 FEM
在特定参数区域（如近共振、近分叉）的数值精度下降时，蒸馏数据带着数值误差。

**SCX 方案**： 1. 精细 FEM 在验证点上评估 2. 用验证误差建立 FEM
per-region 可靠性 → Reliability Map 3. FEM 在参数空间密集采样 4.
`TabularEncoder`（仿真参数向量）编码 → 映射到参数状态 5. 过滤 FEM
高误差参数区域（如近分叉区）的仿真点 6. 用清洁仿真集训练 Student
降阶模型

**Encoder 选择**：`TabularEncoder`（仿真参数向量，如 Re, Ma,
载荷幅值...）。

**Confidence-based** & Balan et al.~(2022), ``Confidence-Aware KD''

**Ensemble-based** & Malinin et al.~(2020), ``Ensemble

#### 关键空白<!-- label: ux5173ux952eux7a7aux767d -->

所有现有 KD 质量控制的共同假设：**Teacher
的可靠性是可以在样本级（per-sample）判断的**。SCX
的独特贡献是指出这个假设不成立------可靠性是状态级的（per-state），因为
Teacher 的误差是系统性集中在特定构型/特征区域的。

**SCX 是第一个将''状态条件专家可靠性''概念引入知识蒸馏的工作。**
