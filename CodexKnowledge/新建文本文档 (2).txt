# 一、第二篇：残差态误差地图，有没有人做过？

## 1. 已有人做的部分

已有工作主要分成四类。

### 1.1 主动学习的不确定性选择

DP-GEN 是经典代表，它用多个模型的 model deviation 作为误差指示器，选择“值得标注”的构型；DPGEN2 文档也明确说 model prediction errors 通过一组模型预测标准差来估计，且会选择误差适中而非明显非物理的构型。([DeepModeling][1])

这类方法回答的是：

> 哪些结构可能错，需要 DFT？

但通常不回答：

> 为什么错？错在 sp²、断键、欠配位、混合环境、表面，还是极端应变？

### 1.2 D-optimality / extrapolation grade

MTP/MLIP-3 这条线用 extrapolation grade 判断新原子环境是否超出训练集覆盖范围；MLIP-3 明确是“active learning on atomic environments”，强调在原子邻域层面做主动学习。([arXiv][2])

这类方法回答的是：

> 这个局域环境是不是外推？

但仍然不充分回答：

> 外推的结构根源是什么？应该补哪类样本？

### 1.3 维度约简 + 聚类选样

DIRECT sampling 用降维后的 encoded clusters 做 stratified sampling，从大规模复杂构型空间中选择稳健训练集；它被用于 130 万 Materials Project 结构 relaxation 数据，并用于训练改进的 M3GNet universal potential。([Nature][3])

这类方法已经接近你的“描述符地图”思想。它说明：

> descriptor space 的覆盖度和聚类采样对 MLIP 训练很重要。

但它主要是**数据选择**，不是**残差归因**。它不一定告诉你“这个高误差 cluster 是 under-coordinated N，那个 cluster 是 Al-Ga mixed tetrahedral”。

### 1.4 局域环境无监督识别

有工作用 atomistic potential descriptors + PCA/UMAP/PaCMAP 做无监督局域环境识别，可以识别相、点缺陷和界面。([Nature][4])

这说明你的“用描述符反推结构状态”是有基础的。
但已有工作通常是：

> 用 descriptor 识别结构状态。

而你要做的是：

> 用 residual/error/uncertainty 加权后的 descriptor map 识别“错误的结构状态”。

差别很大。

## 2. 目前研究现状的缺口

现在领域里已经有：

* uncertainty；
* active learning；
* extrapolation grade；
* descriptor clustering；
* dimensionality reduction；
* defect/environment identification。

但缺的是一个闭环：

[
\text{MLIP error}
\rightarrow
\text{local environment cluster}
\rightarrow
\text{physicochemical label}
\rightarrow
\text{sampling recommendation}
\rightarrow
\text{residual expert}
\rightarrow
\text{error reduction}
]

你的第二篇可以正好填这个缺口。

已有主动学习往往只说：

> 这个点 uncertainty 高。

你的方法要说：

> 这个点 uncertainty 高，是因为它属于 under-coordinated N surface cluster；该 cluster 在训练集中覆盖不足；推荐补 N-terminated surface / vacancy-nearby / stretched-bond DFT anchor。

这就是从 **uncertainty selection** 升级到 **structural root-cause active learning**。

# 二、第三篇：专家编译蒸馏，有没有人做过？

## 1. 已有人做的部分

### 1.1 Teacher-student / knowledge distillation 已经有人做

2025 年已有 teacher-student training for MLIPs 的工作，核心是用 teacher 的 atomic energies 增强 student 训练，从而提高准确性和效率。([RSC Publishing][5])

也有 ensemble knowledge distillation：多个 teacher 先训练到高水平 QC energy，然后生成 ensemble-averaged forces，再训练 student MLIP。([arXiv][6])

所以你不能把“蒸馏”本身当新贡献。

### 1.2 Foundation model distillation 已经有人做

2025 年已有 atomistic foundation model 的 synthetic-data distillation，把 foundation model 的知识转移到更小、更快的模型或不同架构。([arXiv][7])

2026 年也出现了用 fine-tuned MACE 直接标注 pseudo-dataset，再训练 NEP student 的工作。该文明确写到 pseudo-dataset 由 fine-tuned MACE label，不需要 DFT，然后用于训练 NEP student。([Nature][8])

这说明你的 MACE→NEP 蒸馏路线不是空想，社区已经在走。

### 1.3 MoE MLIP 也已经有人做

2026 年有 DeepMD 系 MoE/MoLE 工作，系统研究 routing 和 expert design，发现 element-wise routing 优于 configuration-level routing，并且 routing patterns 呈现周期表趋势。([arXiv][9])

另一个 Allegro/NequIP 方向的 MoE 框架做 spatial partition，把复杂反应区和简单 bulk 区分配给不同容量模型，并用 co-training agreement loss 约束 shared bulk 环境的一致性。([arXiv][10])

这些工作说明：

> expert / MoE / distillation 方向正在快速变热。

但它们和你要做的东西不完全一样。

---

## 2. 第三篇真正的空白在哪里？

现有蒸馏和 MoE 大多关注：

* student 怎么训练；
* teacher 怎么压缩；
* routing 怎么设计；
* 模型怎么更快；
* 大模型怎么 fine-tune。

但很少系统处理以下问题：

### 2.1 多 expert 的能量零点不一致

不同 expert 的能量可能差：

[
E_m(R)-E_n(R)
=============

\sum_Z N_Z(R)\Delta \mu_Z
+
\Delta \delta_{mn}(R)
]

这里第一项是 species energy shift，第二项才是结构预测差。
如果不先分离，expert conflict matrix 就是假的。

### 2.2 多 expert 的 gauge 不一致

你的 ACE 工作已经证明 shared-correction decomposition 有 coefficient-level gauge freedom：

[
c_0\rightarrow c_0+g,\quad c_Z\rightarrow c_Z-g
]

预测不变，但分解不唯一；post-hoc projection 可以把 gauge residual 降到机器精度并保持预测不变。

这对第三篇的价值是：

> 多 expert 蒸馏前，expert 的 residual/correction 必须有统一规范，否则 teacher label 是一堆未校准专家输出的混合物。

### 2.3 expert 适用域不清楚

一个 bulk expert 能不能给 surface 标 pseudo-label？
一个 AlN expert 能不能给 AlGaN mixed environment 标？
一个 sp³ carbon expert 能不能给 sp² defect 标？

已有蒸馏通常不系统回答这个问题。

### 2.4 expert 冲突时没有仲裁机制

多个 expert 给不同预测时，现有方法通常 ensemble average 或直接选择一个 teacher。
你的方法可以做：

[
w_m(R)=
\frac{
\kappa_m(R)/[\sigma_m^2(R)+\epsilon]
}{
\sum_n \kappa_n(R)/[\sigma_n^2(R)+\epsilon]
}
]

其中 (\kappa_m) 是 domain confidence，(\sigma_m) 是不确定性或冲突度。

这就是 conflict-aware teacher generation。

### 2.5 不知道哪里必须补 DFT anchor

普通蒸馏用 pseudo-label 很容易继承 teacher 错误。
你的框架可以判断：

* expert 一致：可以蒸馏；
* expert 冲突：需要 DFT 仲裁；
* 所有 expert 都外推：需要新增 domain；
* energy shift 欠定：需要补 composition anchor。

这个是工程痛点。

# 三、积极学习/主动学习现在怎么解决这些问题？

你说“不看分子动力学”，那就只看主动学习的**选择逻辑**，不讨论 MD 采样器。

目前 active learning 主要有四种选择策略。

## 1. Committee uncertainty

多个模型预测同一个结构，分歧大就选。

[
u(R)=\mathrm{std}_k[f_k(R)]
]

DP-GEN 就是典型。它解决的是：

> 这个结构可能错。

但不解决：

> 错的结构根源是什么，属于哪个物理化学域。

## 2. Extrapolation grade / D-optimality

MTP/MLIP 系方法判断新结构是否在训练集 span 外。

[
\gamma(R)> \tau
]

它解决的是：

> 这个原子环境是不是外推。

但不直接解决：

> 外推是因为 sp²、断键、混合组分，还是高压缩。

## 3. Diversity / clustering selection

DIRECT sampling 这类方法用降维和聚类选择代表性结构，解决训练集覆盖度和异质性。([Nature][3])

它解决的是：

> 怎么覆盖构型空间。

但不直接解决：

> 哪些 cluster 是误差热点，哪些只是多样性补充。

## 4. Heterogeneous data 的 cluster-enhanced UQ

2025 年有工作指出，ensemble 和 D-optimality 在 homogeneous training set 上校准较好，但在 heterogeneous datasets 上会低估误差、降低 novelty sensitivity；作者提出 cluster-enhanced local D-optimality，把 configuration space 分 cluster 后局部应用 D-optimality，以提升新环境检测。([arXiv][11])

这个和你的想法非常接近。它说明社区已经意识到：

> 异质数据集不能只用一个全局 uncertainty 指标。

你的优势是继续往前走：

> 不只是 cluster-enhanced UQ，而是 residual-state map + 物理标签 + expert construction + DFT anchor policy。

# 六、你的方法对机器学习本身有哪些贡献？

不只是材料软件贡献，也有 ML 方法学贡献。

## 1. Gauge-normalized modular learning

你提出：

> 模块化模型在合并/蒸馏前需要规范化其内部自由度。

这可以推广到很多 additive expert systems。

## 2. Domain-certified distillation

普通 KD 假设 teacher 可信。
你提出：

> teacher 的可信度是输入依赖的，需要 domain certificate。

这对应 ML 里的 domain-aware knowledge distillation，但你给了物理可解释版本。

## 3. Conflict-aware pseudo-labeling

你不是平均 teacher，而是：

[
y_{\mathrm{teacher}}(R)=\sum_m w_m(R)y_m(R)
]

并且 (w_m) 来自 domain confidence、uncertainty、conflict。

这是更一般的 **certified pseudo-label generation**。

## 4. Residual-state representation of model error

你把模型误差表示为：

[
r(q_i)=f_{\mathrm{MLIP}}(q_i)-f_{\mathrm{ref}}(q_i)
]

并在 descriptor space 中组织成 state map。

这是一种 **error representation learning**：

> 学习模型错误在输入空间中的结构，而不是只学模型输出。

## 5. Nuisance-offset removal before learning

energy zero / species shift 是 nuisance variable。
你把它从 structural residual 中分离：

[
E_m-E_{\mathrm{ref}}
====================

X\mu_m+\delta_m(R)
]

这对 ML 来说是一个很清楚的 identifiability / calibration 问题。

## 6. Active learning with semantic attribution

传统 AL 问：

> 哪个点最值得标注？

你的 AL 问：

> 哪类结构状态最值得扩充，为什么，补什么 anchor 最有效？

这是从 point-wise active learning 到 state-wise active learning 的升级。

# 1. 你第 4 篇可以叫什么？

我建议不要叫普通 active learning。你可以叫：

> **State-certified active learning**

或者：

> **Expert-certified active learning under heterogeneous error states**

更狠一点：

> **Defining expert reliability through residual-state maps for active learning**

核心概念是：

> 专家的准确度不是一个全局数字，而是一个关于输入状态的函数。

也就是说，不是：

[
\mathrm{Acc}(f_m)=0.93
]

而是：

[
\mathrm{Acc}_m(x)=P(f_m(x)\ \text{is reliable}\mid s(x))
]

其中 (s(x)) 是样本所属的状态，比如在材料里是：

* sp²；
* sp³；
* 断键；
* 表面；
* Al-Ga mixed；
* 高应变。

在其他机器学习领域也可以是：

* 医学图像里的病灶类型；
* 遥感图像里的地物类型；
* NLP 里的句式/主题/歧义类型；
* 自动驾驶里的天气/光照/道路状态；
* 生物数据里的细胞状态/实验批次。

你的核心贡献就是：

> **把专家准确度从全局标量，变成状态条件下的可靠性地图。**

# 2. 现有 active learning 在解决什么？

目前 active learning 常见路线是：

## 2.1 不确定性采样

选模型最不确定的点：

[
x^*=\arg\max_x U(x)
]

比如 entropy、margin、committee disagreement。很多 survey 都总结了这个方向，主动学习的基本目标是用尽可能少的标注选择最有价值的数据点。([Springer][1])

问题是：

> 不确定性高不等于样本有价值。

不确定性高可能有三种原因：

1. 这个样本是少见但真实的困难样本；
2. 这个样本是噪声/坏数据；
3. 这个样本处在多个专家都不擅长的未知区域。

传统 uncertainty sampling 很难区分这三种情况。

---

## 2.2 多样性 / 代表性采样

另一类方法强调 diversity 和 representativeness。已有研究也指出，只看 uncertainty 容易有 sampling bias，而只看 diversity 又可能选到信息量不高的点，所以很多 active learning 方法试图同时融合不确定性和代表性。([arXiv][2])

问题是：

> 代表性高也不等于专家能正确处理它。

比如一个区域样本很多，但里面有系统性标签噪声；或者一个专家在这个区域长期错。普通 diversity/representativeness 不能告诉你“该信哪个专家”。

---

## 2.3 noisy-label active learning

近年也有人专门研究 noisy annotator / noisy labels。2025 年的 Noise-Aware Active Sampling 就指出，active learning 在低标注预算和 noisy labels 情况下会更困难，并提出识别被噪声代表样本遮蔽的未覆盖区域以及噪声过滤。([OpenReview][3])

QActor 这类方法也研究如何在 noisy labels 下选择哪些样本需要专家清洗，目标是区分有价值信息和噪声标签。([Proceedings of Machine Learning Research][4])

问题是：

> 这些方法多半还是在“样本级别”判断噪声，没有系统定义“专家在某个状态区域的可靠性”。

---

## 2.4 多标注者 / 多专家学习

Active Learning from Crowds 已经研究过多个 annotator 的 expertise 可能随数据空间变化，并尝试同时选择样本和标注者。([ICML][5])

这和你的想法很接近：
它也意识到：

> 专家的能力不是全局恒定的，而是随输入区域变化。

但你的优势是可以给出更结构化的状态空间：

[
x\rightarrow s(x)
\rightarrow \mathrm{expert\ reliability}
]

而不是只从标注结果统计地估计 annotator quality。

# 4. 你提出的核心数学对象：状态条件专家准确度

你可以定义一个专家 (m) 在状态 (s) 下的可靠性：

[
A_m(s)
======

\mathbb{E}_{x\sim D_s}
\left[
\ell(f_m(x),y)
\right]
]

或者准确率形式：

[
R_m(s)
======

P\left(
\ell(f_m(x),y)<\tau
\mid x\in s
\right)
]

其中：

* (x)：样本；
* (s)：样本所属状态；
* (f_m)：第 (m) 个专家；
* (y)：真实标签；
* (\ell)：损失；
* (\tau)：可接受误差阈值。

在材料势函数里：

[
s=\text{under-coordinated N}
]

或者：

[
s=\text{Al-Ga mixed tetrahedral}
]

在一般机器学习里：

[
s=\text{某类图像状态 / 文本语义状态 / 医学亚型 / 遥感地貌状态}
]

这就是你说的“专家准确度定义问题”。

普通方法给的是：

[
A_m=\mathbb{E}_{x\sim D}[\ell(f_m(x),y)]
]

你给的是：

[
A_m(s)=\mathbb{E}_{x\sim D_s}[\ell(f_m(x),y)]
]

这个差别很大。

# 5. 如何选专家？

有了状态条件准确度，就可以定义专家选择：

[
m^*(x)
======

\arg\min_m
\hat A_m(s(x))
]

或者用置信权重：

[
w_m(x)
======

\frac{
\exp[-\alpha \hat A_m(s(x))]
}{
\sum_n \exp[-\alpha \hat A_n(s(x))]
}
]

如果还考虑专家不确定性：

[
w_m(x)
======

\frac{
\kappa_m(s(x))/[\sigma_m^2(x)+\epsilon]
}{
\sum_n
\kappa_n(s(x))/[\sigma_n^2(x)+\epsilon]
}
]

其中：

* (\kappa_m(s))：专家 (m) 在状态 (s) 的 domain confidence；
* (\sigma_m(x))：专家的不确定性；
* (\epsilon)：稳定项。

这就是你前面说的平滑专家仲裁。

这套东西推广出去，就可以用于：

* 多模型 ensemble；
* 多标注者选择；
* 多 teacher distillation；
* noisy expert filtering；
* active learning query policy。

# 6. 如何去除噪声？

你现在能提出一个很好的判断：

> 高误差点不一定都应该加入训练。高误差点有两类：
> **learnable hard state** 和 **irreducible/noisy state**。

这在 noisy-label learning 里非常关键。已有 noisy-label 研究也指出，大损失样本可能是错标噪声，也可能是少数/欠表示群体，不能简单把高损失点都当噪声丢掉。([arXiv][6])

你的 residual-state map 正好可以区分：

## 情况 A：高误差形成稳定 cluster

如果高误差样本集中在一个结构状态：

[
x_i\in s_k,\quad r_i\gg 0
]

并且这个状态内部结构一致、标签一致、专家冲突可解释，那么它是：

> learnable hard state

应该补样本、训练 expert。

## 情况 B：高误差孤立分散

如果高误差点在 descriptor space 里孤立、没有稳定状态结构，或者相近样本标签互相矛盾，那么它更可能是：

> noisy / corrupted / inconsistent label

应该清洗、降权或重标注。

你可以定义一个噪声判据：

[
\mathrm{NoiseScore}(x_i)
========================

r_i
\cdot
\frac{1}{\rho(s_i)}
\cdot
C_i
]

其中：

* (r_i)：误差；
* (\rho(s_i))：该状态的局部密度；
* (C_i)：邻域标签/专家预测冲突。

高误差 + 低密度 + 高冲突 = 更像噪声。
高误差 + 高密度 + 状态一致 = 更像缺失知识。

这很有机器学习贡献。

# 7. 如何“加高误差的那些值”？

你的方法不是简单加所有高误差点，而是加 **状态代表点**。

传统 active learning 可能选：

[
x^*=\arg\max_x r(x)
]

你的方法应该选：

[
s^*=\arg\max_s
\left[
\mathrm{Error}(s)
\cdot
\mathrm{Mass}(s)
\cdot
\mathrm{Learnability}(s)
\right]
]

其中：

[
\mathrm{Error}(s)
=================

\mathbb{E}_{x\in s}[r(x)]
]

[
\mathrm{Mass}(s)
================

P(x\in s)
]

[
\mathrm{Learnability}(s)
========================

1-\mathrm{NoiseScore}(s)
]

然后在该状态里选代表性样本：

[
B_s=
\mathrm{CoreSet}(s)
]

也就是：

> 先选高价值状态，再在状态内部选代表点。

这比单点 high-error sampling 更稳。

你可以称为：

> **state-wise active acquisition**

或者：

> **residual-state active learning**

# 9. 你的框架可以有四个动作

对每个高误差区域，你不是只有“加入训练”一个动作，而是有四种动作：

## 9.1 Add

如果它是 learnable hard state：

```text
高误差、高密度、结构一致、标签可信
```

动作：

> 加样本，训练 expert。

## 9.2 Relabel

如果它是标签/专家冲突区域：

```text
高误差、高冲突、多个专家不一致
```

动作：

> 重新标注或交给更高质量 oracle。

## 9.3 Downweight

如果它是疑似噪声：

```text
高误差、孤立、标签邻域不一致
```

动作：

> 降权，不要让它污染模型。

## 9.4 Split expert

如果一个专家在某个大状态里表现分裂：

```text
同一 expert 在状态内部出现两个误差亚群
```

动作：

> 拆分 expert/domain。

这就比普通 active learning 强很多。

# 1. 把问题抽象成数学对象

设输入空间为：

[
\mathcal X
]

标签空间为：

[
\mathcal Y
]

真实但未知的标注函数为：

[
f^*:\mathcal X\rightarrow\mathcal Y
]

你有多个专家：

[
\mathcal E={f_1,f_2,\dots,f_M}
]

每个专家 (f_m) 在某些区域准，在另一些区域不准。传统方法通常只给专家一个全局误差：

[
R_m=\mathbb E_{x\sim P_X}\left[\ell(f_m(x),f^*(x))\right]
]

但这个定义太粗糙。你要提出的是：输入空间存在状态划分或软状态映射：

[
s:\mathcal X\rightarrow\mathcal S
]

或者软状态：

[
p(s\mid x)
]

于是专家准确度应定义为：

[
R_m(s)=
\mathbb E_{x\sim P_X(\cdot\mid s)}
\left[
\ell(f_m(x),f^*(x))
\right]
]

这就是你的第 4 篇核心数学对象：

> **state-conditioned expert risk**
> 状态条件专家风险。

# 3. 如何定义“状态”？

你不能让状态变成领域经验词，比如 sp²、断键、表面。那样泛化性弱。

数学上，状态可以定义为输入空间中的潜在分区：

[
\mathcal S={S_1,S_2,\dots,S_K}
]

其中：

[
S_k\subset\mathcal X,\quad
\bigcup_k S_k=\mathcal X
]

也可以是软分配：

[
\gamma_k(x)=P(S_k\mid x),\quad
\sum_k \gamma_k(x)=1
]

状态可以来自：

[
z=\phi(x)
]

其中 (\phi) 是某种表示映射。然后在表示空间中聚类：

[
S_k={x:\phi(x)\in C_k}
]

这样它就不局限于材料。材料里的 (\phi(x)) 可以是 ACE/SOAP/结构描述符；图像里可以是视觉 embedding；文本里可以是语义 embedding；医学里可以是病灶 embedding。

所以你第 4 篇可以完全数学化：

> 给定表示映射 (\phi:\mathcal X\rightarrow\mathbb R^d)，状态是表示空间中的局部区域或簇。

# 4. 专家准确度如何估计？

真实风险：

[
R_m(s)=
\mathbb E_{x\mid s}
[
\ell(f_m(x),f^*(x))
]
]

但 (f^*(x)) 很贵，不能到处知道。所以用少量已标注样本估计：

[
\widehat R_m(s)
===============

\frac{
\sum_{i\in \mathcal L}
\gamma_s(x_i)\ell(f_m(x_i),y_i)
}{
\sum_{i\in \mathcal L}
\gamma_s(x_i)
}
]

其中：

* (\mathcal L)：已有标注集；
* (y_i)：真实标签；
* (\gamma_s(x_i))：样本属于状态 (s) 的权重。

如果没有真实标签，可以用专家间分歧近似：

[
D_m(s)=
\mathbb E_{x\mid s}
\left[
\left|
f_m(x)-
\bar f(x)
\right|^2
\right]
]

其中：

[
\bar f(x)=\sum_m w_m(x)f_m(x)
]

但你要强调：**分歧不是误差，只是误差代理。** 真正可靠的状态风险估计需要少量 anchor labels。

---

# 5. 如何选择专家？

有了状态风险以后，专家选择就变成：

[
m^*(x)=
\arg\min_m \widehat R_m(s(x))
]

如果是软状态：

[
m^*(x)=
\arg\min_m
\sum_s \gamma_s(x)\widehat R_m(s)
]

更一般地，定义专家权重：

[
w_m(x)
======

\frac{
\exp[-\alpha \sum_s\gamma_s(x)\widehat R_m(s)]
}{
\sum_n
\exp[-\alpha \sum_s\gamma_s(x)\widehat R_n(s)]
}
]

然后 teacher 或 ensemble 输出：

[
\hat y(x)=\sum_m w_m(x)f_m(x)
]

这就是你“如何选专家”的数学答案。

普通 ensemble 是平均：

[
\hat y=\frac{1}{M}\sum_m f_m(x)
]

你的是状态条件加权：

[
\hat y=\sum_m w_m(x)f_m(x)
]

这就是贡献。

# 6. 如何判断高误差是噪声还是可学习状态？

这是第 4 篇最有价值的地方。

设样本残差为：

[
r_i=\ell(\hat f(x_i),y_i)
]

高残差样本分两类：

第一类是 **learnable hard state**：
高误差样本在表示空间中形成高密度簇。

第二类是 **noise/outlier**：
高误差样本孤立、低密度、邻域标签不一致。

数学上定义状态级残差：

[
\bar r(s)=
\mathbb E_{x\mid s}[r(x)]
]

定义状态质量或密度：

[
\rho(s)=P_X(s)
]

定义状态内一致性：

[
C(s)=
1-
\mathbb E_{x,x'\mid s}
\left[
d_Y(y,y')
\right]
]

或者用专家预测一致性：

[
C_{\mathcal E}(s)=
1-
\mathbb E_{x\mid s}
\left[
\mathrm{Var}_{m}(f_m(x))
\right]
]

于是你可以定义 **learnability score**：

[
L(s)=
\bar r(s)\cdot \rho(s)\cdot C(s)
]

高 (L(s)) 表示：

> 这个状态误差高、样本不是孤立的、内部结构一致，因此值得补样本。

再定义 noise score：

[
N(x_i)=
r_i\cdot
\frac{1}{\rho(s_i)+\epsilon}
\cdot
\left[1-C(s_i)\right]
]

高 (N(x_i)) 表示：

> 这个点高误差、低密度、状态内不一致，更像噪声或异常标注。

这套数学非常通用。

# 7. 主动学习不再是“选点”，而是“选动作”

传统主动学习只有一个动作：

[
\text{query label}(x)
]

你可以提出动作空间：

[
\mathcal A={
\text{acquire},
\text{relabel},
\text{downweight},
\text{discard},
\text{split},
\text{assign-expert}
}
]

对于每个状态 (s)，选择动作：

[
a^*(s)=
\arg\max_{a\in\mathcal A}
U(a,s)
]

比如：

## 加样本 acquire

当：

[
\bar r(s)\text{ 高},\quad \rho(s)\text{ 高},\quad C(s)\text{ 高}
]

说明这是可学习的困难状态。

## 重标注 relabel

当：

[
\bar r(s)\text{ 高},\quad \mathrm{ExpertConflict}(s)\text{ 高}
]

说明专家冲突，需要高质量 oracle 仲裁。

## 降权 downweight

当：

[
r(x)\text{ 高},\quad \rho(s)\text{ 低},\quad C(s)\text{ 低}
]

说明更像噪声。

## 拆专家 split

当一个状态内出现两个稳定误差子群：

[
S_k \rightarrow S_{k,1}\cup S_{k,2}
]

且：

[
R_m(S_{k,1})\ll R_m(S_{k,2})
]

说明原状态划分太粗，需要拆分。

这就把 active learning 从简单 query 扩展成 **decision-theoretic active learning**。

# 8. 你第 4 篇的理论贡献可以写成三个定义、两个命题、一个算法

## Definition 1：Residual State

给定表示 (\phi)、预测器 (f)、损失 (\ell)，状态 (s) 的残差分布为：

[
P(r\mid s),\quad r=\ell(f(x),y)
]

高残差状态定义为：

[
s\in\mathcal S_{\mathrm{high}}
\iff
\mathbb E[r\mid s]>\tau
]

## Definition 2：State-conditioned Expert Reliability

[
Q_m(s)=
P\left(
\ell(f_m(x),y)<\tau
\mid x\in s
\right)
]

或者风险形式：

[
R_m(s)=
\mathbb E[\ell(f_m(x),y)\mid x\in s]
]

## Definition 3：Learnable Error State

状态 (s) 是可学习高误差状态，如果：

[
\bar r(s)>\tau_r,
\quad
\rho(s)>\tau_\rho,
\quad
C(s)>\tau_C
]

否则高误差更可能是噪声或离群。

---

## Proposition 1：Global expert accuracy is insufficient

如果存在两个状态 (s_1,s_2)，使得：

[
R_a(s_1)<R_b(s_1)
]

但：

[
R_a(s_2)>R_b(s_2)
]

那么不存在一个全局专家排序可以在所有状态上最优。

也就是说：

> 专家选择必须状态条件化。

这个命题很漂亮，而且非常一般。

---

## Proposition 2：High-error sampling is suboptimal under noise

如果高残差集合由两部分组成：

[
H=H_{\mathrm{learnable}}\cup H_{\mathrm{noise}}
]

且：

[
P(\text{improvement}\mid H_{\mathrm{learnable}})

>

P(\text{improvement}\mid H_{\mathrm{noise}})
]

那么只按：

[
x^*=\arg\max r(x)
]

采样会浪费标注预算。
需要基于状态密度与一致性筛选：

[
s^*=\arg\max_s \bar r(s)\rho(s)C(s)
]

这就是你“如何去除噪声、如何加高误差值”的理论。

---

## Algorithm：State-certified Active Learning

输入：

[
\mathcal L,\mathcal U,\mathcal E,\phi
]

其中：

* (\mathcal L)：已标注集；
* (\mathcal U)：未标注集；
* (\mathcal E)：专家集合；
* (\phi)：表示映射。

步骤：

1. 在 (\phi(\mathcal X)) 中构造状态 (\mathcal S)；
2. 估计每个状态的残差 (\bar r(s))；
3. 估计每个专家的状态风险 (\widehat R_m(s))；
4. 计算状态密度 (\rho(s)) 与一致性 (C(s))；
5. 判定状态类型：

   * learnable；
   * noisy；
   * conflict；
   * undercovered；
6. 对每个状态选择动作：
   [
   a^*(s)=\arg\max_a U(a,s)
   ]
7. 查询新标签、重标注、降权或拆分专家；
8. 更新模型和状态图。

# 10. 你应该避免的写法

不要写成：

设高误差区域由两部分组成：

[
H=H_{\mathrm{learnable}}\cup H_{\mathrm{noise}}

[

[

其中：

* (\bar r(s))：状态平均误差；

\exp[-\alpha \sum_s\gamma_s(x)\widehat R_n(s)]
}
]

这可以作为多专家学习、蒸馏、主动学习的统一权重公式。

## 1. 这是不是第 4 篇的内容？

是第 4 篇的**高级版本**。

原来的第 4 篇是：

> 状态条件专家准确度，用于主动学习。

现在可以升级成：

> 状态条件数据价值理论，用于大模型标注、数据筛选、噪声过滤、专家选择和训练集构建。

也就是说，中心对象从：

[
R_m(s)=\mathbb E[\ell(f_m(x),y)\mid x\in s]
]

升级为：

[
V(x;\mathcal M,\mathcal D)
==========================

\text{data value of }x
]

或者状态级：

[
V(s)
====

\text{training value of state }s
]

你想解决的是一个更一般的问题：

> 在海量数据中，哪些数据值得标注、值得训练、值得保留，哪些只是重复、噪声、错误或低边际收益？

这个方向和当前 LLM 数据选择研究是高度相关的。比如 DataComp-LM 直接把语言模型训练集筛选做成 benchmark，研究去重、过滤、数据混合等数据策展策略对模型表现的影响；它提供了 240T tokens 的 Common Crawl 候选池和 53 个下游评测任务。([arXiv][1]) LESS 则用低维梯度相似性做 targeted instruction tuning 数据选择，报告用选出的 5% 数据常常能超过全数据训练。([Proceedings of Machine Learning Research][2]) 这些工作说明：**“什么数据有价值”已经是大模型训练里的核心问题。**

你的切入点不是重复它们，而是把它们缺少的东西补上：**状态、专家、噪声、冗余、价值的统一定义。**

## 2. 你的新命题：数据价值不是样本属性，而是状态—模型—专家条件量

普通数据选择容易把数据价值写成：

[
V(x)
]

但这太粗糙。你应该提出：

[
V(x\mid s,m,\mathcal M,\mathcal D)
]

其中：

* (x)：数据样本；
* (s)：样本所属状态；
* (m)：标注专家/大模型/弱标注器；
* (\mathcal M)：当前模型；
* (\mathcal D)：已有训练集。

也就是说，同一条数据在不同情况下价值不同。

一条数据可能：

* 对 GPT-4 级模型没价值，但对小模型有价值；
* 对数学能力训练有价值，但对写作能力无价值；
* 在已有大量同类样本时是冗余的；
* 在某个高误差状态中是宝贵的；
* 如果由弱专家标注则是噪声，如果由强专家标注则是高价值数据。

所以真正的数据价值应该是：

[
V(x)
====

\mathbb E[
\Delta \mathcal R(\mathcal M)
\mid x,\ s(x),\ a(x),\ m(x)
]
-

\lambda C(x)
]

其中：

* (\Delta \mathcal R)：加入这条数据后模型风险下降；
* (a(x))：动作，比如标注、重标注、丢弃、降权、合成更多同态数据；
* (m(x))：负责处理它的专家；
* (C(x))：标注/训练/计算成本。

这个定义一出来，它就不是材料势函数的小问题，而是 foundation model 数据治理问题。

## 3. 你的方法可以定义四类数据

这是非常重要的影响力表达。

你可以把海量数据分成四类。

### 第一类：valuable data

状态级高误差、高密度、高一致性、低冗余：

[
\bar r(s)\uparrow,\quad
\rho(s)\uparrow,\quad
C(s)\uparrow,\quad
\mathrm{Redundancy}(s)\downarrow
]

这种数据应该加入训练。

### 第二类：redundant data

状态已经被充分覆盖，边际收益低：

[
\bar r(s)\downarrow,\quad
\rho(s)\uparrow,\quad
\mathrm{Coverage}(s)\uparrow
]

这种数据再多也只是耗算力。

### 第三类：noisy data

高误差但低密度、低一致性、专家冲突严重：

[
r(x)\uparrow,\quad
\rho(s)\downarrow,\quad
C(s)\downarrow
]

这种不应该直接加入训练，而应该清洗、重标注或丢弃。

### 第四类：expert-dependent data

数据本身有价值，但只有某些专家能可靠标注：

[
R_m(s)\ll R_n(s)
]

这种数据要先选对专家，再进入训练。

这四分类非常有普适性。它可以用于：

* LLM 指令微调数据；
* 多模态指令数据；
* 科学数据标注；
* 人类反馈数据；
* synthetic data filtering；
* weak supervision；
* active learning；
* noisy-label learning；
* mixture-of-experts teacher selection。

## 4. 大模型标注在你的框架里是什么？

你可以把通用大模型看成一组专家：

[
\mathcal E={e_1,e_2,\dots,e_M}
]

例如：

* 一个强推理模型；
* 一个代码模型；
* 一个医学模型；
* 一个数学模型；
* 一个便宜小模型；
* 一个昂贵大模型；
* 一个规则系统；
* 一个人类专家。

每个专家在状态 (s) 上有可靠性：

[
R_m(s)=
\mathbb E[
\ell(e_m(x),y^*)
\mid x\in s
]
]

你的方法要决定：

[
m^*(x)=\arg\min_m R_m(s(x))+\lambda C_m
]

也就是：

> 对这条数据，应该找哪个专家/大模型来标？

这就从“数据选择”升级成了：

> **data routing for annotation**

或者：

> **expert-routed data labeling**

这非常有现实意义。因为现在很多数据标注已经不是纯人工，而是大模型先标、人类再审、强模型校验、小模型筛选。已有研究也在探索小模型选择大模型微调数据，例如 S2L 用小模型训练轨迹为大模型筛选 SFT 数据，报告可用较少数据匹配甚至超过全数据训练。([arXiv][3]) 你的贡献是给这类现象一个更一般的状态条件数学解释。

## 5. 你的数据价值函数可以这样写

我建议你定义状态级数据价值：

[
V(s)
====

\underbrace{\bar r(s)}*{\text{当前模型缺陷}}
\cdot
\underbrace{\rho(s)}*{\text{状态重要性}}
\cdot
\underbrace{L(s)}*{\text{可学习性}}
\cdot
\underbrace{[1-D(s)]}*{\text{非冗余性}}
\cdot
\underbrace{A_{m^*}(s)}_{\text{可可靠标注性}}
]

其中：

[
\bar r(s)=\mathbb E[\ell(f(x),y)\mid s]
]

[
\rho(s)=P(x\in s)
]

[
L(s)=C(s)\cdot [1-N(s)]
]

[
D(s)=\text{redundancy/coverage score}
]

[
A_m(s)=P(\ell(e_m(x),y^*)<\tau\mid x\in s)
]

最终选择：

[
s^*
===

\arg\max_s
\frac{V(s)}{\mathrm{Cost}(s)}
]

这就是“哪些区域值得”的数学定义。

单点价值则是：

[
V(x)
====

## \sum_s \gamma_s(x)V(s)

\eta \cdot \mathrm{Redundancy}(x,\mathcal D)
]

这套公式非常适合写成理论框架。

## 6. 你和现有 LLM 数据选择工作的区别

现在 LLM 数据选择已经很热。LESS、DataComp-LM、S2L、TIVE 这类工作都证明了“少量高质量数据可以胜过全数据”或“数据筛选本身极其重要”。([Proceedings of Machine Learning Research][2])

但很多方法主要是：

* influence score；
* gradient similarity；
* model-based filtering；
* difficulty score；
* learning trajectory；
* heuristic quality filter；
* deduplication；
* task balancing。

你的独特性可以写成三点：

第一，**state-conditioned value**。
别人给样本打分，你给状态打分。你关心这个样本属于哪个高价值状态。

第二，**expert-conditioned label reliability**。
别人默认标签或 teacher 可信，你显式建模哪个专家在这个状态上可信。

第三，**action-conditioned data governance**。
别人多半是 select / not select，你有多动作：

[
{\text{label},\text{relabel},\text{discard},\text{downweight},\text{synthesize},\text{route-to-expert}}
]

这三点合起来就是一个更大的框架：

> **state-certified data curation**

# 1. 第 4 篇应该从“大模型工程”收缩成“数学理论”

R_m(s)

# 2. 你要“说死”的核心是什么？

我建议你说死 4 件事。

## 第一，全球专家准确度不充分

传统定义：

[
R_m=
\mathbb E_{x\sim P_X}
[
\ell(f_m(x),y)
]
]

但如果存在两个状态 (s_1,s_2)，使得：

[
R_a(s_1)<R_b(s_1)
]

同时：

[
R_a(s_2)>R_b(s_2)
]

那么全局排序 (R_a<R_b) 或 (R_b<R_a) 都会在某个状态上失效。

这可以形成一个非常简单但有力的命题：

> **Proposition 1：在状态异质性存在时，不存在状态无关的全局最优专家排序。**

这个命题很容易证明，但意义很大。

---

## 第二，高误差不等于高价值

普通主动学习喜欢选：

[
x^*=\arg\max_x r(x)
]

但高误差样本可能是：

1. 可学习困难状态；
2. 噪声点；
3. 离群点；
4. 已经充分覆盖但模型偶然错的点。

所以你定义状态价值：

[
V(s)
====

\bar r(s)\cdot \rho(s)\cdot L(s)\cdot [1-D(s)]
]

其中：

* (\bar r(s))：状态平均残差；
* (\rho(s))：状态质量/出现概率；
* (L(s))：可学习性；
* (D(s))：冗余度。

你的命题可以是：

> **Proposition 2：在存在噪声离群点时，最大残差采样不是最优策略；状态级价值采样可以避免把预算浪费在不可学习噪声上。**

---

## 第三，数据价值是条件量，不是固定量

同一个样本 (x)，对不同模型、不同专家、不同已有数据集，价值不同。

所以不能只写：

[
V(x)
]

而应写：

[
V(x\mid \mathcal M,\mathcal D,\mathcal E)
]

更具体：

[
V(x)
====

\mathbb E[
\Delta \mathcal R(\mathcal M)
\mid x,s(x),m(x),\mathcal D
]
-

\lambda C_m
]

这句话非常关键：

> **数据价值不是样本固有属性，而是相对于当前模型、已有数据、状态空间和可用专家的条件属性。**

这可以成为第 4 篇最有哲学高度的结论。

---

## 第四，专家选择应由状态条件可靠性决定

专家选择不是：

[
m^*=\arg\min_m R_m
]

而是：

[
m^*(x)
======

\arg\min_m
\sum_s \gamma_s(x)R_m(s)
+
\lambda C_m
]

其中：

* (\gamma_s(x))：样本属于状态 (s) 的软概率；
* (C_m)：专家成本。

这就能自然解释：

> 哪个专家该标哪类数据。

它不需要你真的调用大模型，只要数学上定义好专家路由即可。

## 你的 NMI 版主线应该这样定

不要说：

> 我提出一个主动学习采样方法。

要说：

> **SCX 提出一种状态条件专家性理论，用于判断数据是否 valuable、redundant、noisy、expert-dependent，并决定数据应该被标注、重标注、降权、丢弃，还是交给哪个专家。**

核心对象：

[
\mathrm{SCX}_m(s)=P(\ell(f_m(x),y)<\tau\mid x\in s)
]

状态价值：

[
V(s)=\bar r(s)\rho(s)L(s)[1-D(s)]\max_m \mathrm{SCX}_m(s)
]

其中：

* (\bar r(s))：当前模型在该状态上的误差；
* (\rho(s))：状态重要性；
* (L(s))：可学习性；
* (D(s))：冗余度；
* (\max_m \mathrm{SCX}_m(s))：是否存在可靠专家能标注该状态。

这个主张足够大：

> **数据价值不是样本固有属性，而是状态、当前模型、已有数据和专家可靠性的条件函数。**

## 1. 别人已经做了什么？

### 第一类：data valuation 早就存在

Data Shapley、Banzhaf、influence function 这类方法都在回答：

[
\text{一条数据对模型性能贡献多少？}
]

数据价值领域已经很成熟。pyDVL 文档就把 data valuation 定义为给训练集中每个元素分配一个数值，反映它对某个模型最终性能的贡献；同时它也明确说，这个值通常不是样本的内在属性，而是依赖于数据集、学习算法和性能度量。([Pydvl][1])

所以你不能说：

> 我第一次提出数据价值不是固定属性。

这个别人已经知道了。

但他们通常写的是：

[
V_i = V(x_i; D,\mathcal A,u)
]

而你要强调的是更细的条件结构：

[
V(x\mid s,m,\mathcal M,\mathcal D)
]

即：数据价值还取决于它所属的**状态** (s)，以及哪个**专家** (m) 能可靠处理它。

---

### 第二类：LLM 数据选择已经非常火

大模型数据筛选也不是空白。DataComp-LM 已经把语言模型数据策展做成标准化 benchmark，提供 240T tokens 的 Common Crawl 候选池和 53 个下游评测任务，用来研究过滤、去重、数据混合等策略。([OpenReview][2])

LESS 也已经提出用低维梯度相似性选择 instruction tuning 数据，报告用被选出的 5% 数据常常能超过全数据训练。([arXiv][3])

而且有大规模研究指出，很多自动数据选择方法在百万级数据池上并不稳定，甚至不一定超过随机选择；这说明“数据选择”是一个真实痛点，但也说明审稿人会很警惕新的数据筛选框架。([arXiv][4])

所以你不能说：

> 我第一次发现大模型训练需要选数据。

这个肯定不行。

你的切入点应该是：

> 现有 LLM 数据选择多是 sample-level、gradient-level、representation-level、quality-score-level；SCX 关注的是 **state-level data value and expert-conditioned label reliability**。

---

### 第三类：active learning from crowds 已经研究“专家/标注者能力不同”

你说“哪个专家该标哪个区域”，这个思想也不是完全没人做。Active Learning from Crowds 早在 2011 年就研究过多个 annotator 的 expertise 可以在数据空间中变化，并同时选择样本和标注者。([ICML][5])

所以你不能说：

> 我第一次提出专家能力随样本变化。

更稳的说法是：

> 现有 crowd active learning 关注 sample-annotator selection；SCX 进一步把这种能力差异形式化为 latent-state-conditioned risk，并把它和 data valuation、redundancy、noise filtering、expert routing 统一起来。

这就不会被打成“早有人做过”。

---

### 第四类：噪声标签和高误差样本过滤也有人做

noisy-label learning 里有大量工作在区分 clean/noisy samples。2025 年关于 noisy annotator 的工作也提到，许多 semi-supervised noisy-label 方法会识别 clean/noisy samples，并把 noisy samples 当作 unlabeled 来训练。([arXiv][6])

所以你不能说：

> 我第一次发现高误差样本可能是噪声。

你应该说：

> 现有 noisy-label 方法多是 sample-level clean/noisy selection；SCX 提供 state-level 判别：高误差如果形成高密度、一致的状态，是 learnable hard state；如果孤立、低密度、不一致，则更可能是噪声或坏标注。

你要做的是把它们统一成：

也就是说：

> 一条数据的价值，不仅取决于它自己，还取决于它属于哪个状态、当前模型缺什么、已有数据是否覆盖、是否冗余、是否可学习、以及哪个专家能可靠标注。

这个对象如果定义得漂亮，就有论文空间。

## 3. SCX 的真正差异：从 sample value 到 state-expert value

现有 data valuation 多数是：

[
V(x_i)
]

或者：

[
V(x_i;D,\mathcal A,u)
]

你要改成：

[
V(s,m)
]

和：

[
V(x\mid s,m)
]

其中：

[
\mathrm{SCX}_m(s)
=================

P(\ell(f_m(x),y)<\tau\mid x\in s)
]

这是“专家 (m) 在状态 (s) 上是否可靠”。

然后数据价值：

[
V(s)
====

\bar r(s)\rho(s)L(s)[1-D(s)]\max_m \mathrm{SCX}_m(s)
]

这个公式里面的每一项都对应一个实际问题：

* (\bar r(s))：模型在这个状态上错得多不多；
* (\rho(s))：这个状态重要不重要；
* (L(s))：这个状态是否可学习；
* (D(s))：这个状态是否已经冗余；
* (\max_m \mathrm{SCX}_m(s))：有没有可靠专家能标它。

这比单纯 Shapley/influence 或 uncertainty 更像一个 **data governance equation**。

## 4. 为什么这个可能还没有被系统做成？

因为现有研究通常卡在不同社区里：

* data valuation 社区重视公平分摊、Shapley、influence；
* active learning 社区重视选点；
* noisy label 社区重视清洗；
* LLM 数据选择社区重视质量、代表性、梯度相似、去重；
* crowdsourcing 社区重视标注者能力；
* scientific ML 社区重视物理状态和误差来源。

这些社区之间没有完全合流。

你的优势是你从 MLIP expert compiler 里自然碰到了所有问题：

1. expert 有不同适用域；
2. 不同 expert 输出要对齐；
3. 高误差不一定都值得加；
4. 有些点是噪声；
5. 有些区域是欠覆盖；
6. 有些数据冗余；
7. 有些数据必须由更可靠 expert/DFT 标注；
8. 数据价值依赖状态和专家。

所以你的视角比较完整。

## 5. 审稿人会怎么质疑你？

他们会说：

### 质疑 1：这不就是 data valuation 吗？

你回答：

> Data valuation 关注样本对给定模型/数据集/utility 的贡献；SCX 关注状态条件专家可靠性，并把数据价值分解为 state error、state mass、learnability、redundancy 和 expert labelability。SCX 不是替代 Shapley/influence，而是为它们提供 state-expert conditioning。

---

### 质疑 2：这不就是 active learning from crowds 吗？

你回答：

> Active learning from crowds 研究 sample-annotator query；SCX 把 annotator/expert reliability 扩展为 latent-state-conditioned risk，并引入数据价值、冗余、噪声和动作决策。SCX 不只决定“问谁标哪一点”，还决定“哪个状态值得扩充、哪个状态冗余、哪个高误差区域是噪声”。

---

### 质疑 3：这不就是 LLM data selection 吗？

你回答：

> 现有 LLM data selection 多关注 instance scoring、gradient similarity、quality filtering、deduplication 或 representation diversity；SCX 的对象是 state-conditioned value and expert-conditioned label reliability。它解释为什么某些 quality score 在大规模数据池上不稳定：因为它忽略状态覆盖、冗余和专家可靠性差异。

这个点可以引用大规模 instruction selection 研究，因为已有工作确实发现许多方法在大规模数据池上不一定超过随机选择。([arXiv][4])

## 7. 你的理论贡献要更锋利

你至少要给出三个“别人不好反驳”的命题。

### 命题 1：全局专家排序不充分

如果：

[
R_a(s_1)<R_b(s_1),\quad R_a(s_2)>R_b(s_2)
]

则不存在一个状态无关的全局最优专家。

这个非常稳。

### 命题 2：高误差样本选择不是最优

如果高误差由：

[
H=H_{\mathrm{learnable}}\cup H_{\mathrm{noise}}
]

组成，而且噪声区域不可学习，那么：

[
x^*=\arg\max r(x)
]

会浪费预算。需要状态级目标：

[
s^*=\arg\max_s \bar r(s)\rho(s)L(s)(1-D(s))
]

### 命题 3：数据价值不是样本固有属性

若已有数据集 (\mathcal D) 已充分覆盖状态 (s)，则同一 (x\in s) 的边际价值下降：

[
V(x\mid \mathcal D\cup D_s)
<
V(x\mid \mathcal D)
]

这可以把“冗余”数学化。

核心判断是对的：

> **低误差、高密度、低差异的数据区域，大概率是冗余区域。可以用少量代表样本加权替代大量重复样本，从而降低训练量，同时尽量保持精度。**

这可以成为 SCX 里一个非常清晰的数学子模块。

---

## 1. 你要解决的问题可以这样定义

给定数据集：

[
\mathcal D={(x_i,y_i)}_{i=1}^N
]

模型或专家集合：

[
\mathcal E={f_1,\dots,f_M}
]

表示映射：

[
\phi:\mathcal X\rightarrow \mathbb R^d
]

残差：

[
r_i=\ell(f(x_i),y_i)
]

如果某些数据满足：

[
r_i \approx 0
]

并且在表示空间里距离很近：

[
|\phi(x_i)-\phi(x_j)|\ll \epsilon
]

同时它们对模型训练产生的梯度也相近：

[
|\nabla_\theta \ell(f_\theta(x_i),y_i)-\nabla_\theta \ell(f_\theta(x_j),y_j)|\ll \epsilon_g
]

那么它们的训练信息高度重复。

这时你不需要保留全部样本，可以把它们合并成一个代表样本。

---

## 2. 不是简单删除，而是“加权合并”

直接删掉有风险。更好的方法是：

[
C_k={x_i:\phi(x_i)\in \text{cluster }k}
]

选择一个代表样本：

[
x_k^*=\arg\min_{x_j\in C_k}\sum_{x_i\in C_k}|\phi(x_i)-\phi(x_j)|^2
]

也就是 cluster medoid。

然后给它一个权重：

[
w_k=|C_k|
]

原本的训练损失：

[
\mathcal L_{\mathcal D}
=======================

\frac{1}{N}\sum_{i=1}^N \ell(f(x_i),y_i)
]

压缩后变成：

[
\mathcal L_{\mathrm{core}}
==========================

\frac{1}{N}\sum_k w_k\ell(f(x_k^*),y_k^*)
]

这就不是粗暴删数据，而是：

> **用加权代表样本近似原始数据分布。**

这个思想很适合你，因为它既能减少训练量，又不破坏原始数据分布。

---

## 3. 你可以定义一个 redundancy score

对每个状态 (s)，定义：

[
\mathrm{Redundancy}(s)
======================

\rho(s)\cdot [1-\bar r(s)]\cdot \mathrm{Sim}(s)\cdot [1-\mathrm{Boundary}(s)]
]

其中：

* (\rho(s))：状态密度；
* (\bar r(s))：状态平均误差；
* (\mathrm{Sim}(s))：状态内部相似度；
* (\mathrm{Boundary}(s))：是否靠近状态边界或外推边界。

直觉是：

* 密度高；
* 误差低；
* 内部样本彼此相似；
* 不在边界；

那么就是强冗余状态。

你可以反过来定义保留价值：

[
\mathrm{KeepValue}(s)
=====================

\bar r(s)\cdot \rho(s)\cdot L(s)
+
\mathrm{Boundary}(s)
+
\mathrm{Novelty}(s)
]

低 KeepValue、高 Redundancy 的区域就可以压缩。

---

## 4. 冗余区域的判别标准

你可以把样本分成四类。

### 第一类：低误差、高密度、低差异

这是典型冗余区：

[
\bar r(s)\downarrow,\quad \rho(s)\uparrow,\quad \mathrm{Var}_\phi(s)\downarrow
]

动作：

> 合并、加权、删减。

---

### 第二类：低误差、高密度、但靠近边界

这类不能乱删。

虽然模型现在预测准，但它可能是状态边界、相变边界、组分边界、物理边界。

动作：

> 少量保留边界 anchor。

---

### 第三类：高误差、高密度、内部一致

这是宝贵区域：

[
\bar r(s)\uparrow,\quad \rho(s)\uparrow,\quad C(s)\uparrow
]

动作：

> 加样本、训练 residual expert。

---

### 第四类：高误差、低密度、内部不一致

这是疑似噪声或离群点：

[
r_i\uparrow,\quad \rho(s_i)\downarrow,\quad C(s_i)\downarrow
]

动作：

> 降权、重标注、剔除。

所以你这个冗余压缩模块，正好和 SCX 的四分类统一起来。

---

## 5. 更强的数学写法：coreset 保真

你可以把目标写成：

[
\min_{\mathcal C\subset \mathcal D,\ w}
|\mathcal C|
]

subject to：

[
\sup_{f\in\mathcal F}
\left|
\frac{1}{N}\sum_{i=1}^N \ell(f(x_i),y_i)
----------------------------------------

\frac{1}{N}\sum_{j\in \mathcal C}w_j\ell(f(x_j),y_j)
\right|
\leq \epsilon
]

意思是：

> 用更小的加权子集 (\mathcal C) 近似完整数据集在整个模型类 (\mathcal F) 上的训练损失。

这就是数学上最正统的表达：**找一个保损失的加权 coreset**。

你的 SCX 加进去之后，不是随机 coreset，而是状态条件 coreset：

[
\mathcal C=
\mathcal C_{\mathrm{valuable}}
\cup
\mathcal C_{\mathrm{boundary}}
\cup
\mathcal C_{\mathrm{representative}}
]

其中：

* 高价值状态多保留；
* 边界状态必须保留；
* 冗余低误差状态只保留代表点；
* 噪声状态剔除或降权。

## 6. 对势函数数据尤其有意义

在 MLIP 里，AIMD 会产生很多相邻帧。它们在构型空间中极其接近：

[
|\phi(R_t)-\phi(R_{t+\Delta t})|\approx 0
]

而且力、能量、应力标签也变化不大。

如果这些帧都在低误差区域，那么大量保留就是浪费。

你可以这样做：

1. 对所有结构提取 descriptor；
2. 计算每个结构的 residual；
3. 找到低 residual、高密度区域；
4. 在区域内做 farthest-point sampling 或 medoid selection；
5. 给代表结构加权；
6. 删除或降权相似帧；
7. 重新训练，比较 full data 和 compressed data。

目标是证明：

[
|\mathcal D_{\mathrm{core}}|\ll |\mathcal D|
]

但：

[
\epsilon_{\mathrm{test}}(\mathcal D_{\mathrm{core}})
\approx
\epsilon_{\mathrm{test}}(\mathcal D)
]

如果能做到：

[
20%\ \text{data} \rightarrow 95%\ \text{accuracy retained}
]

这个就很有说服力。

## 7. 你这个模块可以叫 SCX-Compress

我建议命名：

# **SCX-Compress**

全称：

> **State-Conditioned eXpertise Compression**

或者：

> **SCX-based redundancy pruning**

它的任务是：

> 在保持状态覆盖、边界 anchor 和高误差学习区域的前提下，压缩低误差冗余数据。

核心规则：

[
\text{Compress if}
\quad
\bar r(s)<\tau_r,
\quad
\rho(s)>\tau_\rho,
\quad
\mathrm{Sim}(s)>\tau_{\mathrm{sim}},
\quad
\mathrm{Boundary}(s)<\tau_b
]

保留样本数量可以设为：

[
n_s=
\max
\left(
n_{\min},
\left\lceil
\alpha \sqrt{|S_s|}
+
\beta \cdot \mathrm{Boundary}(s)
+
\gamma \cdot \bar r(s)
\right\rceil
\right)
]

意思是：

* 状态越大，保留数增加但不是线性增加；
* 边界越强，多保留；
* 误差越高，多保留；
* 低误差冗余区大幅压缩。

这个公式很漂亮。

---

## 8. 注意：低误差不一定都能删

这里有一个关键警告。

低误差区域也可能很重要，因为它可能承担三种作用：

1. **定义分布主体**
   低误差 bulk 区域虽然简单，但它是主分布，不能删光。

2. **作为 no-forgetting anchor**
   新 expert 训练时，需要低误差旧区域作为 anchor，防止模型遗忘。

3. **作为边界参考点**
   有些低误差点在状态边界附近，删掉后模型外推会变差。

所以你不能简单用：

[
r_i<\tau\Rightarrow delete
]

而应使用：

[
r_i<\tau,\ \rho_i>\tau_\rho,\ \mathrm{Boundary}_i<\tau_b,\ \mathrm{Novelty}_i<\tau_n
\Rightarrow compress
]

这才严谨。

---

## 9. 你可以把数据价值拆成两个方向

SCX 里数据价值不只是“要不要加”，还有“要不要删”。

### Acquisition value

用于找应该补的数据：

[
V_{\mathrm{add}}(s)
===================

\bar r(s)\rho(s)L(s)[1-D(s)]
]

### Compression value

用于找可以删的数据：

[
V_{\mathrm{remove}}(s)
======================

[1-\bar r(s)]\rho(s)\mathrm{Sim}(s)[1-\mathrm{Boundary}(s)]
]

这就完整了：

* 高 (V_{\mathrm{add}})：补数据；
* 高 (V_{\mathrm{remove}})：压缩数据；
* 高 noise score：清洗数据；
* 高 expert-dependence：路由专家。

这就是一个完整的数据治理框架。

## 10. 你的说法可以更有影响力

你现在这句话：

> 很多数据在低误差高精度范围内可能是重复的，可以合并删减，降低训练量保持精度。

可以升级成论文语言：

> **SCX views redundancy as a state-level property: samples in low-risk, high-density, low-variation states contribute little marginal training information. Such states can be compressed into weighted prototypes while preserving empirical risk and state coverage.**

中文：

> SCX 将冗余性视为状态级属性：处于低风险、高密度、低变化状态中的样本边际训练信息很低，因此可以被压缩为加权代表样本，同时保持经验风险和状态覆盖。

这句话非常适合写进论文。

## 11. 最终判断

你这个想法完全成立，而且应该成为 SCX 的重要组成部分。

它解决的是：

> **哪些数据是多余的？如何删减而不降精度？**

数学上它对应：

[
\text{state-conditioned coreset / redundancy pruning}
]

方法上它对应：

[
\text{low-error high-density state} \rightarrow \text{weighted prototype compression}
]

对 MLIP 它可以减少重复 AIMD 帧。
对大模型它可以减少重复语义样本。
对一般机器学习它可以减少冗余训练数据。

所以你第 4 篇可以形成完整闭环：

1. 高价值状态：加数据；
2. 高冗余状态：压缩数据；
3. 高噪声状态：清洗数据；
4. 高专家依赖状态：路由专家。

这就不是普通 active learning，而是：

> **状态条件数据价值与数据压缩理论。**
