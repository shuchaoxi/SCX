## 一、SCX 总体框架定义

### 1.1 两层架构

GPT 明确建议 SCX 走 **"论文确权 + 最小开源 + 核心工程保留 + 可商业授权"** 的路线，即两层架构：

#### 第一层：SCX-Research（开源层）

| 维度 | 说明 |
|------|------|
| **定位** | 学术层、开源核心 |
| **目标用户** | 学术界、开源社区、审稿人、潜在合作者 |
| **许可证** | 偏宽松（便于学术传播） |
| **开放程度** | 完全开源 |
| **目的** | 让别人能复现论文、建立学术影响力、抢概念命名权 |

内容清单：
- SCX 数学定义与核心公式
- 命题与证明
- 算法伪代码
- Synthetic experiments（合成实验）
- Toy data valuation demo
- State-conditioned expert reliability estimator（基础版）
- Simple redundancy compression（基础版）
- Paper figures reproduction
- 一个小型 MLIP / 通用 ML demo

#### 第二层：SCX-Pro / SCX-Compiler（闭源/商业层）

| 维度 | 说明 |
|------|------|
| **定位** | 商业执行层、工程实现 |
| **目标用户** | 企业客户、产业联盟成员 |
| **开放程度** | 闭源，通过授权/年费/项目费交付 |
| **目的** | 形成商业壁垒，服务付费客户 |

内容清单：
- 大规模数据管线（large-scale data pipeline）
- MLIP expert compiler（多势函数编译/蒸馏）
- LLM data curation module（大模型数据治理）
- 企业可视化 dashboard
- 自动报告生成（automated report generation）
- 专家库管理（expert registry）
- 企业私有部署 / API 服务
- 私有 benchmark
- 工业级自动阈值选择/状态划分策略
- 专家冲突仲裁完整工程细节

#### 第三层（远期）：SCX Consortium

产业联盟层：
- 多方公司交会员费参与共建
- 核心标准由 SCX 维护方控制
- 非独占授权，不卖断
- 形成多方认可的中立标准

### 1.2 核心原则

> **理论开放，基础代码开放，工程系统保留。**

- 论文保护学术优先权
- 开源保护生态扩散
- 专利保护商业排他性

> **开源到"足以复现论文"，不要开源到"足以替代商业产品"。**

### 1.3 IP 保护建议

顺序链：
1. 先做 IP 评估/学校成果披露
2. 有专利价值先提交申请
3. 再发论文/arXiv
4. 再开源最小代码
5. 最后做商业版

## 二、模块划分

### 2.1 核心模块（SCX-Core）

这些是 SCX 的数学/算法基础，属于开源层。

| 模块 | 功能定义 | 输入 | 输出 |
|------|----------|------|------|
| **State Discovery（状态发现）** | 提取 embedding，聚类形成状态空间 | 数据集 D = {(x_i, y_i)}，表示函数 phi(x) | 状态标签 s_i，状态边界 |
| **Expert Reliability（专家可靠性估计）** | 估计每个专家在每个状态上的可靠性 | 多专家输出 f_m(x)，gold labels y* 或代理标签 | SCX_m(s) = P(loss < tau | x in s) |
| **Data Value Function（数据价值函数）** | 计算样本/状态的数据价值 | 状态统计量：mean residual, density, similarity, consistency | V(s) 价值分数 |
| **Redundancy Compression（冗余压缩）** | 识别低误差高密度状态并压缩 | 状态统计量、embedding | 代表样本子集 + 权重 |
| **Noise Risk Estimation（噪声风险评估）** | 估计噪声风险，区分可学习困难状态和噪声 | residual, density, consistency, anchor support | N(x) 噪声风险分数 |
| **Expert Routing（专家路由）** | 按状态选择最优专家 | 各专家可靠性 R_m(s)，成本 C_m | m*(x) = argmin[R_m(s) + lambda*C_m] |
| **Action Policy（数据动作策略）** | 输出最终数据操作决策 | 上述所有模块输出 | keep/drop/downweight/relabel/route |

### 2.2 领域适配模块（Domain Adapters）

这些是 SCX-Core 在各行业的具体实现。

#### A. SCX-MLIP（材料势函数 — 主战场/根据地）

| 维度 | 说明 |
|------|------|
| **状态表示** | ACE descriptor / atomic local environment |
| **参考标签** | DFT (energy, force, stress) |
| **专家类型** | ACE, NEP, MACE, DeepMD, GAP, NequIP 等势函数 |
| **核心功能** | descriptor residual map, gauge alignment, energy-zero alignment, expert merging, DFT budget-aware active learning |
| **数据动作** | 补 DFT / 合并专家 / 蒸馏学生 / 压缩冗余 AIMD 帧 |
| **商业价值** | 势函数数据审计、多专家编译蒸馏 |

#### B. SCX-Sim / SCX-FEM（工程仿真 — 商业主入口）

| 维度 | 说明 |
|------|------|
| **状态表示** | 几何/网格/响应 embedding, 输入参数 |
| **参考标签** | 细网格/高保真 FEM / 实验数据 |
| **专家类型** | 粗网格 FEM, 细网格 FEM, 高阶求解器, ROM, neural surrogate |
| **核心功能** | 识别高价值工况、压缩冗余仿真、识别数值异常、多保真专家路由 |
| **数据动作** | 高保真重算 / 保留 / 压缩 / 异常标记 |
| **商业价值** | 仿真数据审计报告、surrogate 训练集选择、多保真计算调度 |

#### C. SCX-Health（医学/可穿戴健康 — 开源影响力）

| 维度 | 说明 |
|------|------|
| **状态表示** | 图像 embedding (DINO/CLIP/ViT)，临床特征 |
| **参考标签** | 医生标签 / 多医生一致性 / 金标准 |
| **专家类型** | 不同模型、不同医生、不同影像设备 |
| **核心功能** | 冗余正常病例压缩、噪声标签识别、长尾高价值病例、专家复核优先排序 |
| **数据动作** | 保留 / 压缩 / 降权 / 专家复核 / 补标注 |
| **商业价值** | 第三方数据价值审计报告、可穿戴健康数据治理 |

#### D. SCX-Semi（半导体仿真/制造 — 远期战略）

| 维度 | 说明 |
|------|------|
| **子模块** | SCX-CMP, SCX-TCAD, SCX-VM (Virtual Metrology), SCX-OPC, SCX-Yield |
| **状态表示** | layout density map, pattern clip embedding, device geometry, process parameters |
| **专家类型** | Rule-based OPC, Model-based OPC, ILT, ML-OPC, 粗网格/细网格 TCAD, 简化/工业 CMP 模型 |
| **核心功能** | hotspot 可信度校准、多保真调度、候选 mask 排序、layout 修改优先排序、VM 预测可信度评估 |
| **数据动作** | 普通 OPC / ILT / 高保真仿真 / 量测 / 重算 |
| **商业价值** | EDA/TCAD 上层质量判断层 |

#### E. SCX-Vision（工业/通用视觉）

| 维度 | 说明 |
|------|------|
| **状态表示** | DINO/CLIP/ViT embedding, defect morphology |
| **参考标签** | 人工标注 / 质检标签 |
| **核心功能** | 冗余正常样本压缩、高价值长尾缺陷、标注噪声识别、专家复核优先级 |
| **商业价值** | 工业质检数据治理、视觉标注成本降低 |

#### F. SCX-Music（文化创意 — 探索性/演示用）

| 维度 | 说明 |
|------|------|
| **定位** | 有趣 demo，展示 SCX 泛化能力，非 Nature 主线 |
| **状态表示** | 音频特征 + 歌词特征 + 结构特征 |
| **参考标签** | 流媒体表现 (stream/save/share/skip) |
| **核心功能** | 歌曲状态分类 + 热歌条件规律 + demo 质量诊断 |
| **商业价值** | demo 评分器、A&R 筛歌系统、热歌状态地图 |
| **注意** | 音乐标签极脏，受市场/平台/歌手/运气等多因素干扰 |

#### G. SCX-LLM / SCX-Judge（大模型数据治理 — 远期外延）

| 维度 | 说明 |
|------|------|
| **定位** | 评价器编排框架，非主战场 |
| **状态表示** | task embedding / 领域 embedding / judge disagreement |
| **专家类型** | LLM judge, reward model, verifier, influence score, loss dynamics, DSIR |
| **核心功能** | 多评价器状态条件加权、judge 可靠性校准、高分冗余/低分高价值判断 |
| **数据动作** | keep/drop/downweight/relabel/route to stronger judge/verify with tool |

## 四、四篇论文的重新分布

### 4.1 GPT 建议的论文序列

GPT 强烈建议不要走 "一个 SCX 方法换几个数据集的 salami slicing" 路线，而应形成论文谱系：

| 顺序 | 论文 | 核心问题 | 目标期刊 | 独立贡献 |
|------|------|----------|----------|----------|
| **1** | **SCX-MLIP** (Gauge-normalized residual expert merging) | 多个 MLIP expert 在 ACE/PACE 下如何规范化、对齐、合并，避免 gauge ambiguity 和能量零点冲突？ | npj Computational Materials / Nature Communications / Computer Physics Communications | **材料势函数专门方法**：gauge fixing, energy alignment, expert merge, descriptor residual maps, AlGaN |
| **2** | **SCX-Theory** (State-Conditioned eXpertise) | 专家可靠性和数据价值应该被如何数学定义？什么时候噪声不可识别？什么时候全局专家排序失效？ | TMLR / SIMODS / JMLR / arXiv 先占坑 | **数学地基**：identifiability, state-conditioned risk, conditional data value, action policy |
| **3** | **SCX-Sim** (Scientific & Engineering Simulation) | 在高成本仿真中，如何根据状态选择粗模型/细模型/高保真求解器/实验验证，降低成本并保持精度？ | Nature Computational Science / Nature Communications / Nature Machine Intelligence | **跨领域范式**：多保真调度、求解器可信度、数值异常、expensive oracle budget |
| **4** | **SCX-Health** (Medical Data Valuation) | 在医学数据中如何根据状态区分冗余样本、长尾高价值病例、疑似错标和专家依赖样本？ | npj Digital Medicine / Medical Image Analysis / Nature Biomedical Engineering | **医学/视觉专有问题**：弱标签、multi-rater reliability、long-tail pathology、expert review |

### 4.2 当前论文定位是否需要调整？

原有四篇可能混合了 MLIP、医学、通用数学等内容。GPT 建议严格切割：

**每篇论文的独立贡献对象必须不同：**

| 论文 | Loss | Expert | State | Action |
|------|------|--------|-------|--------|
| MLIP | energy/force/stress residual | ACE/NEP/MACE/DFT | atomic local environment | add DFT / merge expert |
| Theory | abstract loss | abstract experts | measurable state partition | acquire/relabel/downweight/route |
| Sim | FEM/PDE/CMP/TCAD error | coarse/fine solver/surrogate | geometry/process/numerical state | high-fidelity solve / rerun / route |
| Health | classification/noise/disagreement | model/doctor/annotator | visual phenotype/domain state | review/prune/retain/route |

### 4.3 Paper 4 (SCX-Health) 是否需要拆分？

**不需要拆分为 ML 理论论文 + 应用论文。** 但需注意：

- SCX-Health **不应该**与 SCX-Sim 同时硬冲 Nature Communications。如果只是 MedMNIST + HAM10000 + CheXpert 小实验，不适合冲 NC。
- SCX-Health 更合理的定位是：开源影响力项目 + npj Digital Medicine 目标。
- 如果拿到华为/小米/医学合作真实数据，可以升级为 Nature Biomedical Engineering 目标。

## 五、LLM 专项

### 5.1 GPT 对 LLM 的总体判断

> **你的主战场不是大模型，而是科学计算与工程仿真。**

核心逻辑：
1. **大模型通用数据评价，大厂内部已经在做**，而且投入巨大
2. **SCX 不具备与 GPT/Claude/DeepSeek 正面对打 judge 的资源**
3. **LLM 数据缺少明确 y***（无 gold label），SCX 的 claim 必须降级
4. 大厂强项是通用模型和工程规模；你的机会在垂直领域

### 5.2 LLM 在 SCX 中的正确定位

LLM 在 SCX 中应作为：

> **讨论 / extension / 远期外延，不是核心实验主战场。**

SCX 在 LLM 中的正确切口不是"又一个 judge"，而是：

> **状态条件评价器编排框架（State-conditioned judge orchestration framework）**

核心公式：
```
Q_SCX(x) = sum_j w_j(s(x)) * Q_j(x)
w_j(s) propto exp(alpha * A_j(s) - lambda * C_j)
```

其中：
- Q_j(x)：第 j 个评价器的打分
- A_j(s)：该评价器在状态 s 上的可靠性
- C_j：评价成本

### 5.3 SCX 与现有 9 类 LLM 数据筛选方法的区别

| 方法 | 核心回答 | SCX 的补充 |
|------|----------|------------|
| 规则清洗 (FineWeb) | 是不是明显垃圾？ | 状态条件质量判断 |
| 质量分类器 (FinerWeb) | 看起来是不是高质量？ | 多状态质量评估器，而非全局 |
| Perplexity/Loss (Rho-1) | 模型觉得难不难？ | state value + token value 结合 |
| 目标分布匹配 (DSIR) | 像不像目标分布？ | 状态条件版 DSIR |
| Influence/LESS | 对目标能力有没有梯度贡献？ | 只在高风险/高价值状态调用 |
| LLM-as-a-Judge | 强模型觉得好不好？ | 多 judge 状态条件加权 |
| 去重/多样性 | 是否重复？ | 质量 * (1-冗余) * 薄弱覆盖 |
| Verifier | 是否可验证正确？ | 设为高可靠专家，权重大 |
| DataComp-LM | 训练后效果如何？ | 作为最终验证闭环 |

### 5.4 SCX 的差异点

SCX 的核心差异不是"又一个质量打分算法"，而是：

> **在当前状态 (s)、当前模型 (M)、当前数据集 (D)、可用评价器集合 (J) 下，这个样本应该采取什么动作？**

动作空间：keep, drop, downweight, deduplicate, relabel, route to stronger judge, verify with tool, use for SFT, use for preference data, use for eval, use for curriculum later

### 5.5 LLM 实验建议

**不要一上来做大规模预训练。** 最小实验设计：

- **数据**：公开 instruction dataset（几千到几万条）
- **模型**：0.5B–3B 小模型，LoRA 微调
- **SCX 表示**：sentence embedding 或模型 hidden states
- **误差代理**：初始模型 loss 或多模型 disagreement
- **验证**：比较 full data vs random 20% vs diversity 20% vs high-loss 20% vs SCX 20%

### 5.6 SCX-LLM 的商业定位

- 称为 **SCX Data Quality Calibration System**（数据质量校准系统）
- 不是"一个万能 judge"，而是"评价器的评价器"
- 类比：最牛 GPT 是医生，SCX 是医院质控体系
- 企业入口：数据打分 API、数据审计报告、私有 SCX-Judge 部署

**主要领域应聚焦垂直 judge（而非通用）：**
- SCX-MedJudge：医学问答/影像数据
- SCX-CodeJudge：代码数据
- SCX-STEMJudge：数学/物理/材料科学

## 六、附录：GPT 讨论中提到的关键结论

### 6.1 SCX 的核心哲学

> **评价质量决定数据质量，数据质量决定模型质量；但评价本身也是状态条件的，因此必须被校准、路由和审计。**

### 6.2 SCX 的适用条件（领域适配评估表）

| 条件 | ACE/MLIP | 医学图像 | 大模型文本 |
|------|:--------:|:--------:|:--------:|
| 有可靠标签/oracle | 强 | 中到强 | 弱到中 |
| 表示空间有语义 | 强 | 中强 | 中 |
| 状态可解释 | 强 | 中强 | 中弱 |
| 噪声可验证 | 中强 | 中 | 弱 |
| 冗余可识别 | 强 | 中强 | 中 |
| 专家可靠性可估计 | 强 | 中强 | 中 |

### 6.3 噪声识别的理论边界（必须在论文中说明）

> **没有额外假设或参考信息时，高误差不能被唯一识别为噪声。**

SCX 估计的是 noise risk（噪声风险），不是确定性噪声标签。论文中应包含 "Identifiability limits of noise detection" 小节。

### 6.4 SCX 的四种核心分类

| 类别 | 数学特征 | 建议动作 |
|------|----------|----------|
| Valuable（高价值） | r↑, ρ↑, C↑, A↑ | acquire / add data / train expert |
| Noisy（疑似噪声） | r↑, ρ↓, C↓, A↓ | relabel / downweight / inspect |
| Redundant（冗余低价值） | r↓, ρ↑, Sim↑ | compress / weighted prototype / prune |
| Expert-dependent（专家依赖） | R_a(s) << R_b(s) | route to best expert |

### 6.5 SCX 通用架构

```
x --[phi_domain]--> z --[Pi]--> s --> {value, redundancy, risk, expert, budget, model choice}
```

- phi_domain：领域专用状态编码器（Domain-Specific State Encoder / Gauge）
- domain-invariant decision policy：通用数据动作策略

### 6.6 最终战略定位

> **SCX 不是替代求解器，不是替代医生，不是替代大模型；SCX 判断哪个专家、哪个仿真器、哪个数据样本在当前状态下可信和值得投入成本。**

> **SCX 不需要是万能方法；它是面向科学与工程仿真、医学 AI、高质量标注数据的状态条件质量判断与多保真调度框架。**
